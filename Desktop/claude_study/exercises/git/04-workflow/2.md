## Задание

1. Создай 5 Issues на GitHub для проекта:
   - "Добавить навигацию" (label: feature)
   - "Исправить стили кнопки" (label: bug)
   - "Написать README" (label: documentation)
   - "Добавить тёмную тему" (label: enhancement)
   - "Настроить CI/CD" (label: devops)

2. Для Issue "Добавить навигацию":
   - Создай ветку: `git switch -c feature/navigation`
   - Сделай изменения + commit: `git commit -m "feat: добавить навигацию. Closes #1"`
   - Запуши + создай PR
   - В описании PR: "Closes #1"
   - Слей PR — Issue закроется автоматически

3. Объясни GitHub Flow:
   ```
   1. Создать Issue
   2. Создать ветку
   3. Работать + коммитить
   4. Открыть PR
   5. Code Review
   6. Merge + Issue закрывается
   ```

4. Конвенция коммитов:
   - feat: новая функция
   - fix: исправление бага
   - docs: документация
   - style: форматирование
   - refactor: рефакторинг
