# Задание 2: Разрешение конфликтов
**Уровень:** 🟡 Средний
**Тема:** Merge conflicts, разрешение конфликтов
**Ссылка:** notes/git/02-branches.md — "Конфликты"

---

## Задание

Создай конфликт и разреши его:

1. В main измени строку в index.html (например, заголовок).
   Закоммить.

2. Создай ветку feature/title.
   Измени ТУ ЖЕ строку по-другому.
   Закоммить.

3. Переключись на main и попробуй merge:
   ```bash
   git switch main
   git merge feature/title
   ```
   Результат: CONFLICT!

4. Открой файл — увидишь маркеры конфликта:
   ```
   <<<<<<< HEAD
   Текст из main
   =======
   Текст из feature/title
   >>>>>>> feature/title
   ```

5. Разреши конфликт:
   - Удали маркеры
   - Оставь нужный вариант (или объедини оба)
   - `git add index.html`
   - `git commit` (без -m — Git создаст merge commit)

6. Объясни: почему конфликт возник? Когда Git может merge автоматически?
