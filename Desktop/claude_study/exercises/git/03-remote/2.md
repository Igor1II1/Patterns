# Задание 2: SSH и fetch
**Уровень:** 🟡 Средний
**Тема:** SSH ключи, fetch vs pull
**Ссылка:** notes/git/03-remote.md — "SSH", "Fetch"

---

## Задание

1. Сгенерируй SSH-ключ:
   ```bash
   ssh-keygen -t ed25519 -C "your@email.com"
   ```

2. Добавь публичный ключ на GitHub:
   Settings → SSH and GPG Keys → New SSH Key
   ```bash
   cat ~/.ssh/id_ed25519.pub
   ```

3. Проверь подключение:
   ```bash
   ssh -T git@github.com
   ```

4. Переключи remote на SSH:
   ```bash
   git remote set-url origin git@github.com:username/repo.git
   ```

5. Разница fetch и pull:
   ```bash
   git fetch origin      # Скачать изменения (не применять)
   git log origin/main   # Посмотреть что нового
   git merge origin/main # Применить
   # git pull = git fetch + git merge
   ```

6. Объясни: зачем SSH вместо HTTPS? (Не нужно вводить пароль каждый раз)
