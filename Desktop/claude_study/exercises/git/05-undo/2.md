## Задание

1. git revert — безопасная отмена:
   ```bash
   # Создай "коммит-отмену" (не удаляет историю!)
   git revert HEAD
   ```
   Сделай 3 коммита → отмени средний через revert.
   Проверь `git log` — появился новый коммит.

2. Когда revert, а когда reset?
   - revert: если коммит уже запушен (не меняет историю)
   - reset: если коммит только локальный (меняет историю)

3. git stash — временно спрятать изменения:
   ```bash
   # Работаешь над feature, но нужно срочно переключиться
   git stash                    # Спрятать изменения
   git switch main              # Переключиться
   # ... сделать срочное дело ...
   git switch feature/my-work
   git stash pop                # Достать изменения обратно
   ```

4. Практика:
   - Измени файл → `git stash` → проверь status (чисто!)
   - Переключись на другую ветку → вернись → `git stash pop`

5. Команды stash:
   - `git stash list` — список спрятанного
   - `git stash show` — что внутри
   - `git stash drop` — удалить последний stash
