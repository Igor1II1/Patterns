## Задание

1. Спроектируй страницы админ-панели:
   - /admin — дашборд (статистика: пользователи, доски, задачи)
   - /admin/users — управление пользователями (список, блокировка)
   - /admin/boards — все доски (модерация)

2. Защита:
   - middleware для /admin/* — только role === "ADMIN"
   - Server Component с проверкой сессии и роли

3. Напиши API эндпоинты для админки:
   - GET /api/admin/stats
   - GET /api/admin/users?page=&search=
   - PUT /api/admin/users/:id/block
   - DELETE /api/admin/boards/:id

4. Все админские эндпоинты проверяют роль ADMIN.

5. Объясни: почему админ-панель — отдельный route group (admin)?
