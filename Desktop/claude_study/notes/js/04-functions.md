## Зачем эта тема существует?

Когда программа становится больше 20 строк, код начинает повторяться. Одна и та же логика копируется в разные места — а потом при изменении приходится искать и исправлять все копии. Функции решают эту проблему: ты описываешь действие один раз и вызываешь его по имени сколько угодно раз.

Функции появились ещё в математике (f(x) = x + 1), а в программировании стали ключевым инструментом организации кода. В JavaScript функции — "граждане первого класса" (first-class citizens): их можно присваивать переменным, передавать как аргументы другим функциям и возвращать из функций. Это делает JS особенно гибким.

## Зачем нужны функции — три причины

### 1. Избежать дублирования кода (DRY — Don't Repeat Yourself)

```js
// ❌ Без функции — дублирование, ошибки трудно исправлять
const price1 = 1000 + 1000 * 0.2; // добавляем НДС 20%
const price2 = 500  + 500  * 0.2;
const price3 = 2500 + 2500 * 0.2;
// Если НДС изменится с 20% на 25% — меняешь в трёх местах

// ✅ С функцией — меняешь в одном месте
function addVAT(price) {
  return price + price * 0.2;
}
const total1 = addVAT(1000);  // 1200
const total2 = addVAT(500);   // 600
const total3 = addVAT(2500);  // 3000
```

### 2. Разбить сложную задачу на простые части

<!-- добавлено: 2026-03-14 -->
> 💡 `order` здесь — это объект (набор связанных данных, например имя покупателя и сумма). Объекты подробно — в главе 7. Пока важна идея: функция принимает "что-то" и работает с этим.

```js
// Каждая функция делает одну вещь — читается как список действий
function processOrder(order) {
  validateOrder(order);     // проверить заказ
  calculateTotal(order);    // посчитать сумму
  sendConfirmation(order);  // отправить подтверждение
}
```

### 3. Дать смысловое имя куску кода

```js
// ❌ Непонятно что проверяется
if (age >= 18 && verified && !banned) { /* ... */ }

// ✅ С функцией — сразу понятно
if (canAccess(age, verified, banned)) { /* ... */ }

function canAccess(age, verified, banned) {
  return age >= 18 && verified && !banned;
}
```

## Три способа объявить функцию

### 1. Function Declaration — объявление функции

#### Откуда взялось и зачем

Это классический способ объявления функции, унаследованный из языка C. Самый простой и понятный.

#### Синтаксис

```js
function greet(name) {
  return `Привет, ${name}!`;
}

console.log(greet("Игорь")); // "Привет, Игорь!"
```

#### Особенность — Hoisting (поднятие)

До выполнения кода JavaScript "поднимает" Function Declaration в начало области видимости. Функцию можно вызвать **до** строки где она написана:

```js
greet("Игорь"); // "Привет, Игорь!" — работает, хотя функция написана ниже

function greet(name) {
  return `Привет, ${name}!`;
}
```

**Когда использовать:** для основных функций программы, которые нужны везде в файле.

> **Механика: как работает hoisting функций**
>
> JavaScript выполняет код в два прохода:
> ```
> Проход 1 — КОМПИЛЯЦИЯ (до запуска кода):
>   Движок сканирует весь файл и находит все function declaration.
>   Для каждой — создаёт функцию в памяти и привязывает к имени.
>
> Проход 2 — ВЫПОЛНЕНИЕ (построчно):
>   Строка 1: greet("Игорь")  → движок ищет greet → уже есть в памяти → вызов работает
>   Строка 3: function greet() {...}  → пропускается, уже обработано на проходе 1
> ```
> Именно поэтому `const f = function()` НЕ поднимается: на проходе 1 движок видит
> `const f` (переменную, а не функцию) и создаёт её как неинициализированную.
> Попытка вызвать `f()` до строки `const f = ...` даст ReferenceError.

### 2. Function Expression — функциональное выражение

#### Откуда взялось и зачем

Поскольку функции в JS — это значения, их можно присвоить переменной. Это даёт больше контроля: функция создаётся только когда выполнение дойдёт до этой строки.

#### Синтаксис

```js
const greet = function(name) {
  return `Привет, ${name}!`;
};  // точка с запятой — это конец инструкции присваивания

console.log(greet("Игорь")); // "Привет, Игорь!"
```

#### Особенность — нет Hoisting

```js
greet("Игорь"); // ReferenceError — переменная ещё не инициализирована

const greet = function(name) {
  return `Привет, ${name}!`;
};
```

**Когда использовать:** когда функцию нужно передать как аргумент, присвоить условно, или когда hoisting нежелателен.

### 3. Стрелочная функция (Arrow Function)

#### Откуда взялось и зачем

Добавлена в ES6 (2015). Более краткий синтаксис, особенно удобный для коротких функций и коллбэков. Главное техническое отличие — не имеет своего `this` (подробно в гл. 8).

<!-- добавлено: 2026-03-14 -->
> 💡 **Что такое коллбэк (callback)?**
>
> Коллбэк — это функция, которую ты **передаёшь как аргумент** в другую функцию. Та вызовет её когда нужно — не ты сам.
>
> **Аналогия:** ты оставляешь номер телефона в ресторане. Когда столик освободится — тебе **перезвонят** (call back). Ты не ждёшь, а занимаешься своими делами.
>
> ```js
> // setTimeout принимает функцию (коллбэк) и вызывает её через 2 секунды:
> setTimeout(function() {
>   console.log("прошло 2 секунды");
> }, 2000);
> // Ты не вызываешь функцию сам — setTimeout вызовет её за тебя
> ```
>
> Подробнее — в разделе "Коллбэки" ниже в этой главе.

#### Синтаксис — четыре формы записи

```js
// 1. Полная запись — с фигурными скобками и return
const greet = (name) => {
  return `Привет, ${name}!`;
};

// 2. Краткая: если одно выражение — убираем {} и return
const greet2 = (name) => `Привет, ${name}!`;

// 3. Один параметр — убираем скобки
const double = n => n * 2;

// 4. Без параметров — скобки обязательны
const sayHello = () => "Привет всем!";
```

**Возврат объекта — ловушка:**

> **Забегая вперёд:** `{ name: "Игорь", age: 25 }` — объектный литерал (гл. 7). Это способ записать несколько связанных данных в одной переменной: `{ ключ: значение, ключ: значение }`. Фигурные скобки объекта конфликтуют с фигурными скобками тела функции — поэтому нужны круглые скобки.

```js
// ❌ Ошибка: JS думает что {} — это тело функции, а не объект
const getUser = () => { name: "Игорь", age: 25 }; // SyntaxError!

// ✅ Правильно: оборачиваем объект в ()
const getUser = () => ({ name: "Игорь", age: 25 });
```

**Когда использовать:** коллбэки, короткие преобразования, везде где нужна краткость.

> <!-- добавлено: 2026-03-14 -->
> 💡 **Что такое `this`? (кратко, подробно в гл. 8)**
>
> `this` — это ссылка на объект, **от имени которого** вызвана функция. Определяется тем, что стоит **перед точкой** при вызове.
>
> **Что такое метод?** Метод — это функция, записанная как свойство объекта. В примере ниже `sayHi` — свойство объекта `user`, значением которого является функция:
> ```js
> const user = { name: "Игорь", sayHi: function() { console.log(this.name); } };
> user.sayHi(); // "Игорь" — перед точкой user, значит this = user
> ```
> **Аналогия:** местоимение "я" — зависит от того, кто говорит. `this` зависит от того, кто вызвал функцию.

> <!-- добавлено: 2026-03-14 -->
> **Этот блок использует объекты (гл. 7) и this (гл. 8).** Если пока непонятно — пропусти и вернись после изучения этих тем. Здесь важна только идея: стрелочная функция не создаёт свой this.

> **Механика: как стрелочная функция обрабатывает this**
>
> Обычная функция при создании получает **свой** `this` (определяется в момент вызова).
> Стрелочная функция **не создаёт** свой `this` вообще. Вместо этого:
> ```
> const user = {
>   name: "Игорь",
>   greet: function() {          // обычная → свой this = user
>     const inner = () => {      // стрелочная → НЕТ своего this
>       console.log(this.name);  // берёт this из greet → user
>     };
>     inner();
>   }
> };
> user.greet(); // "Игорь"
> ```
> Стрелочная функция «замораживает» this из того места, где она **написана** (лексически),
> а не где вызвана. Это не наследование — это отсутствие собственного this.

### Сравнение трёх видов

| Характеристика | Function Declaration | Function Expression | Arrow Function |
|----------------|---------------------|--------------------|-|
| Hoisting | Да | Нет | Нет |
| Свой `this` | Да | Да | Нет (берёт внешний) |
| Свой `arguments` | Да | Да | Нет |
| Краткость | Средняя | Средняя | Максимальная |
| Использовать как конструктор (`new`) | Да | Да | Нет |

<!-- добавлено: 2026-03-14 -->
> 💡 `arguments` — специальная переменная внутри обычных функций, содержащая все переданные аргументы. Устаревший механизм — вместо него используй rest-параметры (`...args`), о них ниже.
>
> **Конструктор** — функция, вызываемая через `new`, которая создаёт новый объект. Подробно в блоке "ООП и классы" (гл. 1.1). Пока запомни: стрелочные функции нельзя использовать с `new`.

> ❌ **Частые заблуждения:**
> - "Стрелочная функция — просто короткая запись обычной" — нет. У неё **нет своего `this`** и `arguments`. Это не сокращение — это другой механизм. Подробно в главе 8.
> - "Function Expression и Arrow Function — одно и то же" — поведение с `this` отличается принципиально.
> - "Всегда лучше использовать стрелочные функции" — не всегда. Для методов объектов (функций внутри объекта, гл. 7) нужен свой `this` → обычная функция.

> 🔍 **Мини-проверка:** что выведет этот код? Подумай до запуска.
> ```js
> sayHi();
> sayBye();
>
> function sayHi() { console.log("Привет"); }
> const sayBye = () => console.log("Пока");
> ```
> <details><summary>Ответ</summary>
>
> `sayHi()` выведет `"Привет"` — Function Declaration поднимается (hoisting).
> `sayBye()` выбросит `ReferenceError` — стрелочная функция через `const` не поднимается. Код упадёт на второй строке, до определения `sayBye`.
> </details>

## Параметры и аргументы

### Что это такое

**Параметры** — имена переменных в скобках при объявлении функции. Это "входные слоты".

**Аргументы** — реальные значения которые передаёшь при вызове.

```js
function sum(a, b) {  // a и b — ПАРАМЕТРЫ (имена-заглушки)
  return a + b;
}

sum(3, 5); // 3 и 5 — АРГУМЕНТЫ (реальные значения)
```

<!-- добавлено: 2026-03-14 -->
> **Аналогия:** параметр — это бейдж с надписью "Имя гостя". Аргумент — это конкретный гость "Игорь", который пришёл и надел этот бейдж.

### Что если передать меньше аргументов?

```js
function greet(name, greeting) {
  console.log(`${greeting}, ${name}!`);
}

greet("Игорь"); // "undefined, Игорь!" — greeting не передан, стал undefined
```

JavaScript не выдаёт ошибку — недостающие параметры получают значение `undefined`.

> **Механика: как аргументы передаются в функцию**
>
> При вызове `sum(3, 5)` движок создаёт **новые локальные переменные** `a` и `b`:
> ```
> Вызов: sum(3, 5)
> Шаг 1: создать локальную переменную a, скопировать значение 3 → a = 3
> Шаг 2: создать локальную переменную b, скопировать значение 5 → b = 5
> ```
> Для **примитивов** (числа, строки, boolean) копируется ЗНАЧЕНИЕ — изменение внутри
> функции не влияет на оригинал:
>
> ```js
> let age = 25;
> function change(x) {   // x — копия значения, на ней тоже 25
>   x = 35;              // меняем КОПИЮ — оригинал не затронут
> }
> change(age);
> console.log(age);      // 25 — оригинал не изменился
> ```
>
> <!-- добавлено: 2026-03-14 -->
> 💡 **Объект** — это набор связанных данных в формате `{ ключ: значение }`. Точка (`.`) — способ обратиться к конкретному значению: `user.name` → `"Игорь"`. Подробно в главе 7.
>
> Для **объектов** работает иначе. Переменная с объектом хранит не сам объект, а **ключ от комнаты** (адрес в памяти). При передаче в функцию создаётся **дубликат ключа** — но комната одна и та же:
>
> ```js
> const user = { name: "Игорь" };  // создали комнату, положили табличку "Игорь"
>
> function addAge(obj) {   // obj — дубликат ключа от ТОЙ ЖЕ комнаты
>   obj.age = 25;          // зашёл в комнату, повесил табличку "age: 25"
> }
>
> addAge(user);
> console.log(user.age);   // 25 — заходим в ту же комнату, видим новую табличку
> ```
>
> ```
> Память:
>                     ┌─────────────────────┐
>   user ───────────► │ { name: "Игорь" }   │ ◄─────────── obj
>   (ключ от комнаты) │                     │   (дубликат ключа)
>                     └─────────────────────┘
>                     Одна комната, два ключа от неё
> ```
>
> **А если внутри функции написать `obj = {}`?** Это как выбросить свой дубликат ключа и получить ключ от **другой** комнаты. Оригинальная комната не пострадала:
>
> ```js
> function replace(obj) {
>   obj = { name: "Костя" };  // выбросил ключ от комнаты "Игорь",
>                              // получил ключ от новой комнаты "Костя"
> }
> const user = { name: "Игорь" };
> replace(user);
> console.log(user.name);  // "Игорь" — твой ключ всё ещё от старой комнаты
> ```
>
> **Итого — три правила:**
> | Что передаём | Что получает функция | Может изменить оригинал? |
> |---|---|---|
> | Число/строку | Копию значения | Нет |
> | Объект | Дубликат ключа от комнаты | Да, если меняет содержимое (`obj.age = 25`) |
> | Объект | Дубликат ключа | Нет, если заменяет весь объект (`obj = {}`) |

### Что если передать больше аргументов?

```js
function sum(a, b) {
  return a + b;
}

sum(1, 2, 3, 4); // 3 — лишние аргументы (3 и 4) просто игнорируются
```

## Значения по умолчанию (Default Parameters)

### Откуда взялось и зачем

До ES6 для значений по умолчанию приходилось писать проверки внутри функции. ES6 добавил чистый синтаксис прямо в параметрах.

### Синтаксис

```js
function greet(name, greeting = "Привет") {
  console.log(`${greeting}, ${name}!`);
}

greet("Игорь");             // "Привет, Игорь!" — взялось значение по умолчанию
greet("Игорь", "Хай");      // "Хай, Игорь!" — передали своё значение
greet("Игорь", undefined);  // "Привет, Игорь!" — undefined запускает значение по умолчанию
```

### Плохой пример → Хороший пример

```js
// ❌ Плохо: ручная проверка на undefined (старый способ)
function greet(name, greeting) {
  if (greeting === undefined) {
    greeting = "Привет";
  }
  console.log(`${greeting}, ${name}!`);
}

// ✅ Хорошо: значение по умолчанию в параметре
function greet(name, greeting = "Привет") {
  console.log(`${greeting}, ${name}!`);
}
```

> **Механика: когда именно срабатывает значение по умолчанию**
>
> Значение по умолчанию — это НЕ предварительное присвоение. Это условная проверка
> при каждом вызове:
> ```
> function greet(name, greeting = "Привет") { ... }
>
> Вызов greet("Игорь"):
>   Шаг 1: name = "Игорь"                         (аргумент передан)
>   Шаг 2: greeting = undefined (не передан)
>   Шаг 3: greeting === undefined? ДА → вычислить выражение "Привет" → greeting = "Привет"
>
> Вызов greet("Игорь", "Хай"):
>   Шаг 2: greeting = "Хай"                        (аргумент передан)
>   Шаг 3: greeting === undefined? НЕТ → оставить "Хай"
>
> Вызов greet("Игорь", null):
>   Шаг 2: greeting = null                          (null — это конкретное значение)
>   Шаг 3: greeting === undefined? НЕТ → оставить null
> ```
> Выражение по умолчанию **вычисляется заново** при каждом вызове.
> Поэтому `function f(date = new Date())` даст разное время при разных вызовах.

> ⚠️ **Важно:** `null` **не** запускает значение по умолчанию. Только `undefined` и отсутствие аргумента.
> ```js
> greet("Игорь", null);  // "null, Игорь!" — null это конкретное значение, не "отсутствие"
> ```

> ❌ **Частые заблуждения:**
> - "Значение по умолчанию вычисляется при объявлении функции" — нет, оно вычисляется **при каждом вызове** если аргумент не передан. Поэтому `new Date()` в значении по умолчанию будет создавать новую дату при каждом вызове.
> - "Значение по умолчанию может быть только число или строка" — нет, может быть любым выражением, включая вызов функции.

## return — возврат значения

### Что это такое

`return` делает две вещи:
1. Возвращает значение из функции
2. **Немедленно прекращает** выполнение функции

```js
function add(a, b) {
  return a + b; // возвращает результат и выходит
  console.log("это не выполнится"); // недостижимый код
}

const result = add(2, 3); // result = 5
```

### Функция без return

Если `return` нет или написан без значения — функция возвращает `undefined`:

```js
function greet(name) {
  console.log(`Привет, ${name}!`); // побочный эффект
  // нет return
}

const result = greet("Игорь"); // result = undefined
```

### Ранний return — guard clauses

Мощный паттерн: выходить из функции рано при неправильных условиях. Убирает вложенность.

```js
// ❌ Плохо: вложенный код — трудно читать
function processAge(age) {
  if (typeof age === "number") {
    if (age >= 0) {
      if (age <= 150) {
        return `Возраст: ${age}`;
      }
    }
  }
  return "Ошибка";
}

// ✅ Хорошо: ранний return — чистый, линейный код
function processAge(age) {
  if (typeof age !== "number") return "Ошибка: не число";
  if (age < 0) return "Ошибка: отрицательный возраст";
  if (age > 150) return "Ошибка: нереальный возраст";

  return `Возраст: ${age}`; // основная логика — когда всё ок
}
```

> 🔍 **Мини-проверка:** что выведет этот код? Подумай до запуска.
> ```js
> function test(x) {

>   if (x > 5) return "среднее";
>   return "маленькое";
> }
>
> console.log(test(7));
> console.log(test(3));
> console.log(test(15));
> ```
> <details><summary>Ответ</summary>

> `test(7)`: 7 > 10 — false, 7 > 5 — true → `"среднее"`.
> `test(3)`: 3 > 10 — false, 3 > 5 — false → `"маленькое"`.
> `test(15)`: 15 > 10 — true → `"большое"` (второй if не проверяется).

## return — полный справочник синтаксиса

### Когда нужны скобки, когда if, когда нет

Многие новички путают: "нужны ли скобки после return?", "нужен ли if?". Вот полная система.

### return + значение (основной случай)

После `return` пишешь выражение — и всё. Никаких скобок не нужно:

```js
return 1;
return "hello";
return a + b;
return base * power(base, exp - 1);
```

### Круглые скобки `()` — группировка (необязательны)

```js
// Без скобок — работает:
return n % 10 + sumDigits(Math.floor(n / 10));

// Со скобками — тоже работает, читать легче:
return (n % 10) + sumDigits(Math.floor(n / 10));
```

Круглые скобки после return — **не обязательны**. Они просто группируют, как в математике: `(2 + 3) * 4`.

**Единственное исключение** — возврат объекта из стрелочной функции:
```js
// ❌ JS думает что {} — тело функции
const getUser = () => { name: "Игорь" };  // undefined!

// ✅ Скобки говорят: это объект, не тело
const getUser = () => ({ name: "Игорь" });
```

### Фигурные скобки `{}` — тело функции (не часть return)

Фигурные скобки — это **не про return**. Это тело функции:

```js
// Обычная функция — фигурные = тело, return нужен
function add(a, b) {
  return a + b;
}

// Стрелочная БЕЗ фигурных — return неявный (автоматический)
const add = (a, b) => a + b;

// Стрелочная С фигурными — return нужен явно
const add = (a, b) => { return a + b; };
```

| Запись | return нужен? |
|--------|---------------|
| `function f() { ... }` | Да, всегда |
| `() => выражение` | Нет (неявный) |
| `() => { ... }` | Да (фигурные = тело) |

### if + return — когда нужен if, когда нет

`if` нужен когда есть **развилка**. В рекурсии — только для базового случая:

```js
function power(base, exp) {
  if (exp === 0) return 1;              // if нужен: проверяем "стоп?"
  return base * power(base, exp - 1);   // if НЕ нужен: единственный путь дальше
}
```

Почему второй `if` не нужен? Потому что первый `return` уже **вышел из функции**. Если мы дошли до второй строки — значит `exp !== 0` точно:

```
power(2, 0):  exp === 0? ДА → return 1 → ВЫШЛИ. Строка 3 не выполнится.
power(2, 3):  exp === 0? НЕТ → пропускаем → строка 3 выполняется ВСЕГДА
```

### Сводная таблица

| Что пишешь | Зачем | Пример |
|-----------|-------|--------|
| `return выражение;` | Вернуть результат | `return a + b;` |
| `return (выражение);` | То же + группировка | `return (n % 10) + f(n);` |
| `if (условие) return X;` | Ранний выход / базовый случай | `if (n < 10) return 1;` |
| `=> выражение` | Неявный return (стрелочная) | `x => x * 2` |
| `=> { return выражение; }` | Явный return (стрелочная с телом) | `x => { return x * 2; }` |
| `=> ({ ключ: значение })` | Вернуть объект (стрелочная) | `() => ({ name: "Игорь" })` |

## Область видимости (Scope)

### Что это такое

**Область видимости** — пространство в коде где переменная доступна. Функция создаёт свою область видимости: переменные объявленные внутри функции — невидимы снаружи.

```
┌─────────────── Глобальная область ─────────────────┐
│ const name = "Игорь";                               │
│                                                      │
│ ┌──────── Область функции greet ─────────┐          │
│ │ function greet() {                      │          │
│ │   const message = "Привет";  // видна   │          │
│ │   console.log(name);  // "Игорь" ✅     │          │
│ │   // name видна — она во внешней области │          │
│ │ }                                       │          │
│ └─────────────────────────────────────────┘          │
│                                                      │
│ console.log(message); // ❌ ReferenceError            │
│ // message не видна — она внутри функции             │
└──────────────────────────────────────────────────────┘
```

### Правила

1. **Внутренняя область видит внешнюю** — функция может читать переменные из окружения где она создана
2. **Внешняя не видит внутреннюю** — переменные из функции недоступны снаружи
3. **Каждый вызов функции** создаёт новую область видимости

```js
const globalVar = "глобальная";

function outer() {
  const outerVar = "внешняя";

  function inner() {
    const innerVar = "внутренняя";
    console.log(globalVar); // ✅ видит
    console.log(outerVar);  // ✅ видит
    console.log(innerVar);  // ✅ видит
  }

  inner();
  // console.log(innerVar); // ❌ ReferenceError
}

outer();
// console.log(outerVar); // ❌ ReferenceError
```

> 💡 **На заметку:** это правило вложенных областей видимости называется **лексическая область видимости** (lexical scope). Функция запоминает область где была **создана**, а не где была **вызвана**. Подробнее — в теме замыканий (продвинутый блок).

## Rest параметры (...)

### Откуда взялось и зачем

Иногда не знаешь заранее сколько аргументов придёт. До ES6 для этого использовали специальный объект `arguments` (неудобный, не настоящий массив). ES6 добавил rest-параметры — чистый и удобный синтаксис.

### Что это такое

<!-- добавлено: 2026-03-14 -->
> 💡 **Что такое массив?** (кратко, подробно в гл. 6)
>
> Массив — это упорядоченный список значений, записывается в квадратных скобках: `[1, 2, 3]`. Обращение к элементу по номеру (с нуля): `args[0]` — первый элемент, `args[1]` — второй. Цикл `for...of` перебирает все элементы массива по очереди.

**Rest параметры** — синтаксис `...имя` в параметрах функции. **Собирает** все оставшиеся аргументы в **настоящий массив**.

<!-- добавлено: 2026-03-14 -->
> **Аналогия:** ты стоишь с коробкой и говоришь "кидайте сколько хотите". Сколько бы мячей ни кинули (1, 5, 100) — все попадут в коробку. `...numbers` — это та самая коробка.

```js
function sum(...numbers) {
  // numbers — это массив, например [1, 2, 3, 4, 5]
  let total = 0;
  for (const n of numbers) {
    total += n;
  }
  return total;
}

sum(1, 2);          // 3
sum(1, 2, 3);       // 6
sum(1, 2, 3, 4, 5); // 15
sum();              // 0
```

> **Механика: как rest-параметры собирают аргументы**
>
> ```
> function log(level, ...messages) { }
>
> Вызов: log("INFO", "Сервер запущен", "Порт: 3000")
>
> Шаг 1: level = "INFO"              ← первый аргумент → первый параметр
> Шаг 2: оставшиеся аргументы: "Сервер запущен", "Порт: 3000"
> Шаг 3: создать НОВЫЙ массив ["Сервер запущен", "Порт: 3000"]
> Шаг 4: messages = этот массив
>
> Вызов: log("WARN")
> Шаг 1: level = "WARN"
> Шаг 2: оставшихся аргументов нет
> Шаг 3: messages = []               ← пустой массив, НЕ undefined
> ```
> Rest всегда создаёт **настоящий массив** (можно вызывать методы массивов — о них в главе 6),
> в отличие от устаревшего `arguments`, который массивом только притворяется.

### Правило: rest — только последний параметр

```js
// ✅ Правильно — обычные параметры перед rest
function log(level, ...messages) {
  for (const msg of messages) {
    console.log(`[${level}] ${msg}`);
  }
}

log("INFO", "Сервер запущен", "Порт: 3000");
// [INFO] Сервер запущен
// [INFO] Порт: 3000

// ❌ Неправильно — rest не может быть в середине
// function wrong(a, ...rest, b) {} // SyntaxError!
```

> ❌ **Частые заблуждения:**
> - "`...` — это всегда spread" — нет. В **параметрах** функции `...` — это **rest** (собирает). При **вызове** или в литералах — это **spread** (разворачивает). Одинаковый синтаксис, противоположные действия.

## Spread оператор (...)

### Откуда взялось и зачем

Тот же синтаксис `...`, но используется не в параметрах, а при **вызове** функции или **создании** массива/объекта. Spread **разворачивает** (распаковывает) коллекцию в отдельные элементы.

### Ключевая разница Rest vs Spread

```
Rest   — в ПАРАМЕТРАХ функции     → СОБИРАЕТ много значений В массив
Spread — при ВЫЗОВЕ / в ЛИТЕРАЛЕ  → РАЗВОРАЧИВАЕТ массив В отдельные значения
```

### Spread при вызове функции

```js
const numbers = [3, 1, 4, 1, 5, 9];

// Math — встроенный объект JS с математическими функциями
// Math.max() возвращает наибольшее из переданных чисел
Math.max(3, 1, 4, 1, 5, 9); // 9 ✅
Math.max(numbers);           // NaN ❌ — передали массив, а не числа

// Spread разворачивает массив в отдельные аргументы
Math.max(...numbers);        // 9 ✅ — то же что Math.max(3, 1, 4, 1, 5, 9)
```

### Spread с массивами

> **Забегая вперёд:** массивы подробно в главе 6. `.push(значение)` — метод массива, добавляет элемент в конец.

```js
const a = [1, 2, 3];
const b = [4, 5, 6];

// Объединить два массива
const combined = [...a, ...b];       // [1, 2, 3, 4, 5, 6]

// Добавить элементы в начало/конец
const withZero = [0, ...a];          // [0, 1, 2, 3]
const withSeven = [...a, 7];         // [1, 2, 3, 7]

// Скопировать массив (независимая копия)
const copy = [...a];
copy.push(4);
console.log(a);    // [1, 2, 3] — оригинал не изменился
console.log(copy); // [1, 2, 3, 4]
```

### Spread с объектами

> **Забегая вперёд:** объекты `{ ключ: значение }` — способ хранить несколько связанных данных в одной переменной. Spread (`...`) копирует все пары ключ-значение из одного объекта в другой. Подробно в главе 7 — вернись к этому разделу после неё.

```js
const user = { name: "Игорь", age: 25 };

// Скопировать с изменением одного поля
const updatedUser = { ...user, age: 26 };
// { name: "Игорь", age: 26 }

console.log(user); // { name: "Игорь", age: 25 } — не изменился

// При конфликте ключей — побеждает ПОСЛЕДНИЙ
const result = { a: 1, b: 2, ...{ b: 99, c: 3 } };
// { a: 1, b: 99, c: 3 } — b перезаписан
```

> 🔍 **Мини-проверка:** что выведет этот код? Подумай до запуска.
> ```js
> function first(...args) {
>   return args[0];
> }
>
> const nums = [10, 20, 30];
> console.log(first(...nums));
> console.log(first(nums));
> ```
> <details><summary>Ответ</summary>
>
> Первый вызов: `10`. `...nums` разворачивает массив → `first(10, 20, 30)` → `args = [10, 20, 30]` → `args[0] = 10`.
> Второй вызов: `[10, 20, 30]`. Без spread весь массив передаётся как один аргумент → `args = [[10, 20, 30]]` → `args[0] = [10, 20, 30]`.
> </details>

## Функции высшего порядка и коллбэки

<!-- добавлено: 2026-03-18 -->

### Что такое функция высшего порядка?

**Функция высшего порядка (higher-order function)** — это функция, которая делает хотя бы одно из двух:
1. **Принимает** другую функцию как аргумент
2. **Возвращает** функцию как результат

```js
// repeat ПРИНИМАЕТ функцию action → это функция высшего порядка
function repeat(n, action) {
  for (let i = 0; i < n; i++) {
    action(i);
  }
}

repeat(3, console.log);  // 0, 1, 2
```

**Обычная функция** работает с данными: числами, строками, boolean.
**Функция высшего порядка** работает с **поведением**: принимает "что делать" как аргумент.

**Аналогия:** обычная функция — это рабочий, который знает одно дело. Функция высшего порядка — это менеджер: сам не делает работу, а получает инструкцию (другую функцию) и выполняет её.

### Зачем нужны?

Без функций высшего порядка пришлось бы писать отдельную функцию на каждое поведение:

```js
// ❌ Без ФВП — дублирование
function repeat3TimesLog(n) {
  for (let i = 0; i < n; i++) console.log(i);
}
function repeat3TimesAlert(n) {
  for (let i = 0; i < n; i++) alert(i);
}

// ✅ С ФВП — одна функция, любое поведение
function repeat(n, action) {
  for (let i = 0; i < n; i++) action(i);
}
repeat(3, console.log);   // выводит в консоль
repeat(3, alert);          // показывает alert
repeat(3, i => {});        // ничего не делает (но можно!)
```

### Два термина — одна связка

| Термин | Кто это | Пример |
|--------|---------|--------|
| **Функция высшего порядка** | Та, которая **принимает** функцию | `repeat`, `operate`, `applyTwice` |
| **Коллбэк (callback)** | Та, которую **передают** как аргумент | `console.log`, `i => i * 2`, `action` |

Они всегда идут парой: функция высшего порядка вызывает коллбэк.

```js
function operate(a, b, operation) {  // operate — ФВП
  return operation(a, b);            // operation — коллбэк
}

operate(10, 5, (a, b) => a + b);    // (a, b) => a + b — коллбэк
```

> В будущих главах ты встретишь встроенные ФВП: `.map()`, `.filter()`, `.forEach()`, `.reduce()` — все они принимают коллбэк. Это основа работы с массивами в JS.

### Откуда взялись коллбэки и зачем

Раз функции — это значения, их можно передавать как аргументы другим функциям. Функция, переданная как аргумент, называется **коллбэком** (callback — "обратный вызов"). Эта концепция — основа программирования на JavaScript.

### Что такое коллбэк

**Коллбэк** — функция которую передают другой функции как аргумент, чтобы та вызвала её в нужный момент.

**Аналогия:** ты заказываешь пиццу и оставляешь номер телефона. Курьер **перезвонит** (callback) когда приедет. Ты не ждёшь у двери — занимаешься другими делами.

<!-- добавлено: 2026-03-17 -->
### Зачем вообще передавать функцию в функцию?

Представь обычную функцию — она **заранее знает** что делать:

```js
function addTen(n) {
    return n + 10;  // всегда прибавляет 10, и больше ничего
}
```

А что если ты хочешь функцию которая умеет прибавлять, умножать, делить — **что угодно**? Без коллбэка пришлось бы писать отдельную функцию на каждый случай:

```js
function add(a, b) { return a + b; }
function multiply(a, b) { return a * b; }
function subtract(a, b) { return a - b; }
function divide(a, b) { return a / b; }
// ... и так бесконечно
```

С коллбэком — **одна функция, бесконечное количество поведений**:

```js
function operate(a, b, operation) {
    return operation(a, b);  // "возьми a и b, сделай с ними ЧТО СКАЖУТ"
}

operate(10, 5, function(a, b) { return a + b; });   // 15
operate(10, 5, function(a, b) { return a * b; });   // 50
operate(10, 5, function(a, b) { return a - b; });   // 5
```

**Суть коллбэка:** функция с коллбэком — это **универсальный инструмент**. Она знает **когда** и **с чем** выполнить действие, но **что именно делать** — решаешь ты снаружи, при вызове.

**Аналогия с курьером:**

```js
function courier(box, whatToDo) {
    whatToDo(box);  // курьер делает с коробкой то, что ему сказали
}

// Ты решаешь ЧТО делать — передаёшь инструкцию (функцию):
courier("пицца", function(item) { console.log("Доставил: " + item); });
// → "Доставил: пицца"

courier("документы", function(item) { console.log("Сжёг: " + item); });
// → "Сжёг: документы"
```

Курьер (`courier`) — один и тот же. Коробка (`box`) — разная. А что делать (`whatToDo`) — ты каждый раз передаёшь как функцию. Вот это и есть коллбэк.

### Самый простой коллбэк: функция-операция

<!-- добавлено: 2026-03-17 -->

Начнём с самого простого. У тебя есть два числа и ты хочешь что-то с ними сделать — но **что именно**, решает не функция, а тот кто её вызывает:

```js
function operate(a, b, operation) {
  // operation — это функция, которую нам передали
  // мы просто вызываем её с нашими числами
  return operation(a, b);
}
```

Что здесь происходит **пошагово**:

```
// Вызов:
operate(10, 5, function(x, y) { return x + y; });

// Шаг 1: JS подставляет аргументы
//   a = 10
//   b = 5
//   operation = function(x, y) { return x + y }  ← вся функция!

// Шаг 2: выполняется тело operate
//   return operation(a, b)
//   = operation(10, 5)
//   = function(10, 5) { return 10 + 5 }
//   = 15
```

**Ключевое:** `operation` — это обычный параметр, но в нём лежит **функция**. Вызвать её — это как вызвать любую другую функцию: `operation(a, b)`.

Теперь можно передать **разные операции** — функция `operate` одна и та же:

```js
// Сложение
operate(10, 5, (a, b) => a + b);   // 15

// Умножение
operate(10, 5, (a, b) => a * b);   // 50

// Максимум
operate(10, 5, (a, b) => a > b ? a : b);  // 10
```

> **Аналогия:** `operate` — это калькулятор с пустой кнопкой. Ты сам решаешь что эта кнопка делает: складывает, умножает, или что угодно. Каждый раз ты вставляешь новую кнопку (коллбэк).

**Три формы записи коллбэка — одно и то же:**

```js
// 1. Обычная функция (многословно, но наглядно)
operate(10, 5, function(a, b) {
  return a + b;
});

// 2. Стрелочная функция (коротко)
operate(10, 5, (a, b) => a + b);

// 3. Заранее объявленная функция (передаём по имени)
function add(a, b) { return a + b; }
operate(10, 5, add);  // передаём саму функцию, без ()!
```

> **Ловушка:** `operate(10, 5, add)` — передаём функцию. `operate(10, 5, add())` — вызываем add() и передаём **результат** (число), не функцию! Без скобок = передать. Со скобками = вызвать.

<!-- добавлено: 2026-03-17 -->
### Почему в вызове функции можно писать `=>`?

Может показаться странным: почему при вызове `transform(5, n => n + 1, n => n * 2)` мы пишем стрелочные функции прямо внутри скобок? Это выглядит непривычно, но на самом деле просто.

**Ключевое правило:** функция — это значение, как число или строка. Везде где можно написать число, можно написать функцию.

```js
// Число можно написать прямо в вызове:
console.log(42);

// Строку тоже:
console.log("hello");

// Функцию — точно так же:
console.log(n => n * 2);  // выведет: [Function (anonymous)]
```

Поэтому когда функция принимает коллбэк, мы можем передать функцию **прямо при вызове**:

```js
transform(5, n => n + 1, n => n * 2)
//        ↑  ↑            ↑
//        |  |            └─ третий аргумент: функция
//        |  └─ второй аргумент: функция
//        └─ первый аргумент: число
```

**Это то же самое что объявить функции заранее:**

```js
// Длинный способ — объявляем отдельно
function addOne(n) {
  return n + 1;
}
function double(n) {
  return n * 2;
}
transform(5, addOne, double);  // → 12

// Короткий способ — пишем прямо в вызове
transform(5, n => n + 1, n => n * 2);  // → 12

// Результат одинаковый!
```

**Пошаговый разбор `transform(5, n => n + 1, n => n * 2)`:**

```
// Внутри transform:
function transform(value, fn1, fn2) {
    return fn2(fn1(value));
}

// JS подставляет аргументы:
//   value = 5
//   fn1   = n => n + 1      ← целая функция лежит в переменной fn1
//   fn2   = n => n * 2      ← целая функция лежит в переменной fn2

// Выполнение:
// fn1(value) = fn1(5) = 5 + 1 = 6
// fn2(6)     = 6 * 2 = 12
// return 12
```

> **Правило:** `n => n + 1` в аргументах вызова — это не магия и не особый синтаксис. Это обычная стрелочная функция, записанная там где ожидается значение. Точно так же как `5` — обычное число, записанное там где ожидается значение.

### Зачем коллбэки?

Коллбэк позволяет **отделить ЧТО делать от КОГДА делать**:
- `operate` знает КОГДА (получил два числа — применяй)
- Коллбэк знает ЧТО (складывать? умножать? делить?)

Это делает код гибким — одна функция, много поведений. Без коллбэков пришлось бы писать `operateAdd`, `operateMultiply`, `operateSubtract` — отдельную функцию на каждый случай.

### Пример посложнее: собственная функция с коллбэком

```js
// repeat вызывает коллбэк n раз
function repeat(n, action) {
  for (let i = 0; i < n; i++) {
    action(i);  // вызываем коллбэк, передавая текущий номер
  }
}

// Передаём функцию как аргумент
repeat(3, function(i) {
  console.log(`Итерация ${i}`);
});
// "Итерация 0"
// "Итерация 1"
// "Итерация 2"

// То же самое стрелочной функцией — короче
repeat(3, i => console.log(`Итерация ${i}`));
```

> **Механика: что происходит при передаче коллбэка пошагово**
>
> ```
> function repeat(n, action) {
>   for (let i = 0; i < n; i++) { action(i); }
> }
> repeat(3, i => console.log(i));
>
> Шаг 1: вызов repeat(3, i => console.log(i))
>         n = 3
>         action = ссылка на стрелочную функцию (функция — это значение!)
>
> Шаг 2: цикл, i = 0
>         action(0) → вызывается стрелочная функция с аргументом 0
>         → console.log(0) → вывод: 0
>
> Шаг 3: цикл, i = 1
>         action(1) → та же функция, новый аргумент
>         → console.log(1) → вывод: 1
>
> Шаг 4: цикл, i = 2
>         action(2) → console.log(2) → вывод: 2
>
> Шаг 5: i = 3, условие 3 < 3 ложно → цикл завершён
> ```
> Ключевое: `action` — это обычная переменная, в которой лежит функция.
> Вызов `action(i)` — это вызов той функции, которую передали при вызове `repeat`.

<!-- добавлено: 2026-03-14 -->
> **Аналогия:** `repeat` — это робот. Ты говоришь ему: "сделай **это** 3 раза". **Это** — та функция, которую ты передал. Робот не знает заранее что делать — ты ему говоришь при вызове.
> ```js
> // "Три раза скажи номер"
> repeat(3, function(i) { console.log(i); });
>
> // "Три раза скажи привет"
> repeat(3, function(i) { console.log("Привет!"); });
> ```
> Один и тот же робот (`repeat`), но разные задания (разные коллбэки).

### Коллбэки в методах массивов (предпросмотр)

> **Забегая вперёд:** методы массивов `.map()`, `.filter()`, `.forEach()` подробно в главе 6. Здесь — краткое знакомство, потому что коллбэки и методы массивов неразрывно связаны.
>
> `forEach` вызывает переданную функцию (коллбэк) **один раз для каждого элемента** массива.

```js
const numbers = [1, 2, 3, 4, 5];

// forEach вызывает коллбэк для каждого элемента
numbers.forEach(n => console.log(n * 2));
// 2, 4, 6, 8, 10
```

### Асинхронные коллбэки (предпросмотр)

```js
console.log("1. Начало");

// setTimeout — вызвать коллбэк через N миллисекунд
setTimeout(() => {
  console.log("3. Через 2 секунды");
}, 2000);

console.log("2. Конец синхронного кода");

// Порядок вывода:
// "1. Начало"
// "2. Конец синхронного кода"
// (через 2 сек) "3. Через 2 секунды"
```

JavaScript не останавливается и не ждёт — он продолжает выполнять остальной код, а коллбэк вызовется позже. Это называется **асинхронность** — подробно в отдельном блоке.

## Чистые функции vs побочные эффекты

### Откуда взялось и зачем

Когда программа растёт, становится трудно понять почему результат неправильный. Причина часто в том, что функции незаметно меняют данные в другом месте программы. Концепция чистых функций пришла из функционального программирования — она помогает писать предсказуемый код, который легко тестировать и отлаживать.

### Что такое чистая функция?

**Чистая функция (pure function)** — функция которая соответствует **двум критериям**:

**Критерий 1: Детерминизм** — одинаковый вход всегда даёт одинаковый выход.
```js
// ✅ Чистая — add(2, 3) ВСЕГДА вернёт 5, хоть вызови 1000 раз
function add(a, b) {
  return a + b;
}

// ❌ Нечистая — каждый вызов даёт разный результат
function getRandom() {
  return Math.random();  // 0.123... потом 0.789... потом 0.456...
}
// Тест: вызови getRandom() дважды — результаты разные. Значит, нечистая.
```

**Критерий 2: Нет побочных эффектов** — функция не меняет ничего за своими пределами.
```js
// ✅ Чистая — не трогает ничего снаружи
function double(n) {
  return n * 2;
}

// ❌ Нечистая — меняет внешнюю переменную total
let total = 0;
function addToTotal(n) {
  total += n;    // ПОБОЧНЫЙ ЭФФЕКТ: изменяет переменную снаружи функции
  return total;
}
// addToTotal(5) → 5
// addToTotal(5) → 10  ← тот же вход (5), но другой результат!
```

**Аналогия:** чистая функция — это калькулятор. Набираешь 2 + 3, получаешь 5. Всегда. Калькулятор не меняет температуру в комнате и не переставляет мебель — он просто считает и выдаёт ответ.

Нечистая функция — это автомат с кофе. Нажимаешь кнопку "эспрессо" — получаешь кофе, **но** уровень воды уменьшается, зёрна расходуются, счётчик порций увеличивается. Куча "побочных эффектов" за пределами самой кнопки.

### Что такое побочный эффект?

**Побочный эффект (side effect)** — любое воздействие функции на мир за её пределами. Вот полный список:

```js
let count = 0;

// 1. Изменение внешней переменной
function increment() {
  count++;  // побочный эффект: count изменился
}

// 2. Мутация входных данных (забегая вперёд: это изменение массива/объекта, переданного в функцию)
function addItem(arr, item) {
  arr.push(item);  // побочный эффект: оригинальный массив изменился
  return arr;
}

// 3. Вывод в консоль
function logUser(name) {
  console.log(name);  // побочный эффект: что-то выводится наружу
}

// 4. Запрос к серверу (будет в теме Async)
// function fetchData() { fetch("..."); }

// 5. Запись в файл, localStorage и т.д. (будет в теме DOM)
```

### Анализ 5 функций: чистая или нечистая?

Это ключевое упражнение — научись определять чистоту функции "на глаз":

```js
// 1. add — ЧИСТАЯ ✅
function add(a, b) { return a + b; }
// Почему: одинаковый вход → одинаковый выход. Ничего снаружи не трогает.
// add(2, 3) → 5, всегда 5.

// 2. addToArray — НЕЧИСТАЯ ❌
function addToArray(arr, item) { arr.push(item); return arr; }
// Почему: arr.push(item) МУТИРУЕТ (изменяет) входной массив.
// Если кто-то передал [1, 2], после вызова его массив станет [1, 2, 3].
// Оригинал изменён — это побочный эффект.
// Чистая версия: return [...arr, item] — создаёт НОВЫЙ массив, не трогая оригинал.

// 3. getRandom — НЕЧИСТАЯ ❌
function getRandom() { return Math.random(); }
// Почему: нет детерминизма. Вызови 100 раз — 100 разных результатов.
// Одинаковый вход (нет аргументов) → разный выход.

// 4. formatName — ЧИСТАЯ ✅
function formatName(name) { return name.trim().toUpperCase(); }
// Почему: "  Игорь  " → "ИГОРЬ" — всегда. trim() и toUpperCase()
// создают НОВУЮ строку, не изменяя оригинал. Строки в JS неизменяемы (immutable).

// 5. addToTotal — НЕЧИСТАЯ ❌
let total = 0;
function addToTotal(n) { total += n; return total; }
// Почему: читает И меняет внешнюю переменную total.
// addToTotal(10) → 10, addToTotal(10) → 20 — разный результат при одном входе!
```

> **Как определить чистоту? Два вопроса:**
> 1. "Если вызвать 100 раз с теми же аргументами — всегда одинаковый результат?" Нет → нечистая.
> 2. "Меняется ли что-то, что существовало ДО вызова функции?" Да → нечистая.

### Как сделать нечистую функцию чистой?

**Правило:** всё что нужно — ПЕРЕДАВАЙ через параметры. Ничего снаружи НЕ ТРОГАЙ. Результат ВОЗВРАЩАЙ через return.

```js
// ❌ НЕЧИСТАЯ — зависит от внешней переменной tax
let tax = 0.2;
function getPrice(price) {
  return price + price * tax;  // читает tax СНАРУЖИ
}
// Если кто-то изменит tax = 0.5, результат getPrice(100) изменится!

// ✅ ЧИСТАЯ — tax передаётся как параметр
function getPrice(price, tax) {
  return price + price * tax;
}
// getPrice(100, 0.2) → 120, всегда 120. Не зависит ни от чего снаружи.
```

Ещё пример:

```js
// ❌ НЕЧИСТАЯ — меняет внешнюю переменную
let total = 0;
function addToTotal(n) { total += n; return total; }

// ✅ ЧИСТАЯ — total передаётся как параметр, возвращается новое значение
function addToTotal(currentTotal, n) {
  return currentTotal + n;  // не трогаем ничего снаружи
}
// addToTotal(100, 50) → 150
// addToTotal(100, 50) → 150 — тот же вход, тот же выход!
```

> **Механика: почему параметр "безопаснее" внешней переменной**
>
> ```
> // Нечистая версия:
> let total = 0;                    // ← одна переменная на ВСЮ программу
> function addToTotal(n) {
>   total += n;                     // ← кто угодно мог уже изменить total
>   return total;                   // ← результат зависит от "истории вызовов"
> }
>
> // Чистая версия:
> function addToTotal(total, n) {   // ← total — ЛОКАЛЬНАЯ копия, только для этого вызова
>   return total + n;               // ← total не может измениться извне
> }
> ```
> Параметры — это **локальные переменные**. Они живут только внутри вызова.
> Внешние переменные — **общие** для всей программы. Кто угодно может их изменить.

### Почему это важно?

**Чистые функции легко тестировать:**
```js
// Чистую функцию тестировать просто: вход → выход, точка
console.log(add(2, 3) === 5);  // true — готово, работает

// Нечистую — нужно учитывать состояние:
total = 0;           // сначала сбросить total
addToTotal(5);       // вызвать
console.log(total);  // проверить... но а если другая функция тоже меняет total?
```

**Чистые функции предсказуемы:**
```js
// Чистая: нет скрытых зависимостей
calculateTotal(100, 3, 0.2);  // 360 — можно понять БЕЗ контекста

// Нечистая: нужно знать текущее состояние ВСЕЙ программы
addToTotal(5);  // что вернёт? Зависит от того, что было раньше
```

**Побочные эффекты НЕ плохие** — без них программа не делала бы ничего полезного (не выводила бы данные, не сохраняла). Но их нужно **контролировать и изолировать**: держи чистую логику отдельно от побочных эффектов.

> **В реальной разработке:**
> - React-компоненты должны быть "чистыми" — одни и те же props = одинаковый рендер
> - Redux reducers обязаны быть чистыми функциями
> - Тесты: чистые функции легко покрыть unit-тестами (вход → выход)
> - Отладка: если результат неправильный, проблема только в аргументах — не нужно искать "кто изменил переменную"

## Рекурсия

### Откуда взялось и зачем

Некоторые задачи естественно разбиваются на подзадачи того же типа: обход дерева файлов, математические вычисления, поиск в структурах с вложенностью. Рекурсия — когда функция вызывает саму себя для решения уменьшенной версии той же задачи. Это один из фундаментальных приёмов программирования, и он повсюду: от разбора HTML-документа до отображения комментариев с ответами.

### Что это такое

**Рекурсия** — функция вызывает саму себя.

**Аналогия:** матрёшка. Открываешь — внутри такая же, только меньше. И так пока не дойдёшь до самой маленькой (базовый случай).

Другая аналогия: ты стоишь в очереди и спрашиваешь "какой я по счёту?". Человек перед тобой не знает — он спрашивает у того, кто перед ним. И так по цепочке до первого в очереди, который говорит "я первый!". Ответ идёт обратно: "я второй", "я третий"... Это и есть рекурсия.

### Два обязательных условия

Любая рекурсия состоит из двух частей:

1. **Базовый случай** (условие выхода) — когда функция **НЕ** вызывает себя, а возвращает готовый ответ
2. **Рекурсивный случай** — вызов **самой себя** с **изменёнными** данными (меньше, короче, проще)

```js
function рекурсия(данные) {
  if (базовый_случай) return готовый_ответ;  // СТОП — дальше не идём
  return моя_часть + рекурсия(меньшие_данные); // разбиваем задачу
}
```

**Без базового случая** — бесконечная рекурсия → ошибка `RangeError: Maximum call stack size exceeded`.

> **Механика: что такое стек вызовов (call stack)**
>
> Каждый вызов функции попадает в **стек** — как стопка тарелок. Новый вызов кладётся сверху. Когда функция возвращает результат — тарелка снимается.
> ```
> factorial(4)
>   → factorial(3)         Стек растёт ↓
>     → factorial(2)
>       → factorial(1)     ← базовый случай, return 1
>       ← 1                Стек сворачивается ↑
>     ← 2 * 1 = 2
>   ← 3 * 2 = 6
> ← 4 * 6 = 24
> ```
> Если базового случая нет, стек растёт бесконечно и переполняется (обычно ~10 000 вызовов).

### Универсальный паттерн рекурсии

Запомни эту формулу — она работает для 90% рекурсивных задач:

```
return МОЯ_ЧАСТЬ_РАБОТЫ + рекурсия(ОСТАЛЬНОЕ)
```

- **"Моя часть"** — то, что я делаю на текущем шаге
- **"Остальное"** — та же задача, но меньше

```js
// Сумма от 1 до n:  "я (n)" + "сумма остальных (от 1 до n-1)"
return n + sumRange(n - 1);

// Факториал n:      "я (n)" * "факториал остальных (n-1)"
return n * factorial(n - 1);
```

### Пример 1: факториал

Факториал числа — произведение всех целых чисел от 1 до n.
`5! = 5 × 4 × 3 × 2 × 1 = 120`

```js
function factorial(n) {
  if (n <= 1) return 1;        // базовый случай: 0! = 1, 1! = 1
  return n * factorial(n - 1); // рекурсивный: n * факториал остального
}

factorial(5); // 120
```

**Пошагово — фаза "погружения" (вызовы вглубь):**
```
factorial(5)  → 5 <= 1? НЕТ → return 5 * factorial(4)   // ждём ответа от factorial(4)
factorial(4)  → 4 <= 1? НЕТ → return 4 * factorial(3)   // ждём ответа от factorial(3)
factorial(3)  → 3 <= 1? НЕТ → return 3 * factorial(2)   // ждём ответа от factorial(2)
factorial(2)  → 2 <= 1? НЕТ → return 2 * factorial(1)   // ждём ответа от factorial(1)
factorial(1)  → 1 <= 1? ДА  → return 1                  // БАЗОВЫЙ СЛУЧАЙ! Ответ готов.
```

**Фаза "раскрутки" (ответы идут обратно):**
```
factorial(1) = 1
factorial(2) = 2 * 1 = 2
factorial(3) = 3 * 2 = 6
factorial(4) = 4 * 6 = 24
factorial(5) = 5 * 24 = 120  ← финальный ответ
```

### Пример 2: сумма чисел от 1 до n

```js
function sumRange(n) {
  if (n <= 1) return n;          // базовый случай: sumRange(1) = 1, sumRange(0) = 0
  return n + sumRange(n - 1);    // я (n) + сумма остальных
}

sumRange(5); // 15 (5 + 4 + 3 + 2 + 1)
```

**Пошагово:**
```
sumRange(5) = 5 + sumRange(4)
sumRange(4) = 4 + sumRange(3)
sumRange(3) = 3 + sumRange(2)
sumRange(2) = 2 + sumRange(1)
sumRange(1) = 1  ← базовый случай

Раскрутка: 2+1=3 → 3+3=6 → 4+6=10 → 5+10=15
```

> **Частая ошибка:** написать `return sumRange(n - 1)` без `n +`. Это просто спустится до базового случая и вернёт 1. Забыл "свою часть работы"! Правильно: `return n + sumRange(n - 1)` — "я + остальное".

### Пример 3: числа Фибоначчи

Последовательность Фибоначчи: 0, 1, 1, 2, 3, 5, 8, 13, 21...
Каждое число = сумма двух предыдущих.

```js
function fibonacci(n) {
  if (n === 0) return 0;  // базовый случай 1
  if (n === 1) return 1;  // базовый случай 2
  return fibonacci(n - 1) + fibonacci(n - 2);  // сумма двух предыдущих
}

fibonacci(7); // 13
```

**Пошагово для fibonacci(5):**
```
fib(5) = fib(4) + fib(3)
fib(4) = fib(3) + fib(2)
fib(3) = fib(2) + fib(1)
fib(2) = fib(1) + fib(0) = 1 + 0 = 1
fib(3) = 1 + 1 = 2
fib(4) = 2 + 1 = 3
fib(5) = 3 + 2 = 5
```

Здесь **два базовых случая** (0 и 1) и **два рекурсивных вызова** на каждом шаге. Это "двойная рекурсия" — дерево вызовов разрастается.

> **Частая ошибка:** написать `return (n - 1) + (n - 2)` вместо `fibonacci(n - 1) + fibonacci(n - 2)`. Первый вариант — просто арифметика: `(5-1) + (5-2) = 7`. Это НЕ рекурсия. Рекурсия = функция **вызывает САМУ СЕБЯ**. Без `fibonacci(...)` нет рекурсивного вызова.

### Пример 4: разворачивание вложенного массива (flatten)

> **Забегая вперёд:** `Array.isArray(x)` — проверяет, является ли `x` массивом (возвращает `true`/`false`). `concat()` — соединяет два массива в новый, не меняя оригиналы. `push()` — добавляет элемент в конец массива. Подробно о массивах — в главе 6.

Задача: превратить `[1, [2, 3], [4, [5, [6]]]]` в `[1, 2, 3, 4, 5, 6]`.

```js
function flatten(arr) {
  let result = [];
  for (const element of arr) {
    if (Array.isArray(element)) {
      // элемент — массив → рекурсивно разворачиваем и добавляем
      result = result.concat(flatten(element));
    } else {
      // элемент — НЕ массив → просто добавляем
      result.push(element);
    }
  }
  return result;  // return ПОСЛЕ цикла — когда ВСЕ элементы обработаны
}

flatten([1, [2, 3], [4, [5, [6]]]]); // [1, 2, 3, 4, 5, 6]
```

**Пошагово для `flatten([1, [2, 3]])`:**
```
result = []

Элемент 1: Array.isArray(1)? НЕТ → result.push(1) → result = [1]

Элемент [2, 3]: Array.isArray([2, 3])? ДА → вызываем flatten([2, 3])
  └─ Внутри flatten([2, 3]):
     result = []
     Элемент 2: не массив → push → result = [2]
     Элемент 3: не массив → push → result = [2, 3]
     return [2, 3]

result = [1].concat([2, 3]) = [1, 2, 3]

return [1, 2, 3]
```

Это рекурсия другого типа — здесь нет формулы `n + f(n-1)`. Вместо этого рекурсия срабатывает **по условию**: "если встретил массив внутри массива — разверни его рекурсивно". Количество вложенности может быть любым.

> **Частая ошибка:** написать `return result.concat(flatten(element))` ВНУТРИ цикла. `return` прерывает **ВСЮ** функцию — обработается только первый элемент-массив, остальные пропустятся. `return` должен быть **после** цикла.

### Рекурсия vs цикл

Любую рекурсию можно переписать циклом и наоборот. Сравнение:

```js
// Факториал ЦИКЛОМ — итеративный подход
function factorialLoop(n) {
  let result = 1;
  for (let i = 2; i <= n; i++) {
    result *= i;  // result = result * i
  }
  return result;
}
// Шаг за шагом: result=1 → 1*2=2 → 2*3=6 → 6*4=24 → 24*5=120

// Факториал РЕКУРСИЕЙ
function factorialRec(n) {
  if (n <= 1) return 1;
  return n * factorialRec(n - 1);
}
// Шаг за шагом: 5*f(4) → 4*f(3) → 3*f(2) → 2*f(1) → 1 → раскрутка → 120
```

| | Цикл | Рекурсия |
|---|------|---------|
| Скорость | Быстрее | Медленнее (стек вызовов) |
| Память | Меньше (одна переменная result) | Больше (каждый вызов хранится в стеке) |
| Когда использовать | Простые итерации (сумма, поиск) | Деревья, вложенные структуры |
| Читаемость | Лучше для линейных задач | Лучше для задач с вложенностью |

**Когда рекурсия лучше цикла?** Когда данные **вложенные** — дерево файлов, вложенные комментарии, HTML-документ. Цикл не умеет "нырять" в глубину и "возвращаться" — а рекурсия делает это естественно.

```js
// flatten НЕВОЗМОЖНО написать простым циклом for, потому что
// вложенность может быть любой глубины:
// [1, [2, [3, [4, [5]]]]]
// Рекурсия обрабатывает любую глубину автоматически
```

### Как мыслить при написании рекурсии — 3 вопроса

<!-- добавлено: 2026-03-18 -->

Не нужно "догадываться" что писать. Нужно задать себе **три вопроса** и собрать формулу:

**Вопрос 1: "Когда ответ очевиден?"** (это базовый случай)
```
power:       степень = 0         → ответ 1
countDigits: число < 10          → ответ 1 (одна цифра)
sumDigits:   число < 10          → ответ само число
factorial:   n = 0 или n = 1     → ответ 1
sumRange:    n = 1               → ответ 1
```

**Вопрос 2: "Как сделать задачу ЧУТЬ меньше?"** (это рекурсивный вызов)
```
power:       уменьшить степень на 1    → power(base, exp - 1)
countDigits: убрать одну цифру         → countDigits(Math.floor(n / 10))
sumDigits:   убрать одну цифру         → sumDigits(Math.floor(n / 10))
factorial:   уменьшить n на 1          → factorial(n - 1)
sumRange:    уменьшить n на 1          → sumRange(n - 1)
```

**Вопрос 3: "Что Я делаю на этом шаге?"** (это "моя часть работы")
```
power:       умножаю на base один раз  → base * ...
countDigits: считаю одну цифру         → 1 + ...
sumDigits:   беру последнюю цифру      → (n % 10) + ...
factorial:   умножаю на n              → n * ...
sumRange:    прибавляю n               → n + ...
```

**Собираем по шаблону:**
```
return [мой шаг] + функция([задача поменьше])
```

| Задача | Мой шаг | Задача поменьше | Формула |
|--------|---------|-----------------|---------|
| power | `base *` | `exp - 1` | `base * power(base, exp-1)` |
| countDigits | `1 +` | убрать цифру | `1 + countDigits(Math.floor(n/10))` |
| sumDigits | `(n%10) +` | убрать цифру | `(n%10) + sumDigits(Math.floor(n/10))` |
| factorial | `n *` | `n - 1` | `n * factorial(n - 1)` |
| sumRange | `n +` | `n - 1` | `n + sumRange(n - 1)` |

### Пример: новая задача с нуля

Задача: "перевернуть строку рекурсивно" (reverseString).

> **Забегая вперёд:** `str[0]` — первый символ строки. `str.slice(1)` — строка без первого символа: `"abc".slice(1)` → `"bc"`. Подробно о строках — в главе 5.

Применяем 3 вопроса:
1. **Когда очевидно?** Строка из 0-1 символов → вернуть как есть
2. **Как сделать меньше?** Убрать первый символ → `str.slice(1)`
3. **Что я делаю?** Беру первый символ и ставлю в **конец**

```js
function reverseString(str) {
  if (str.length <= 1) return str;                // базовый случай
  return reverseString(str.slice(1)) + str[0];    // остаток перевёрнутый + первый символ
}

// reverseString("abc")
// = reverseString("bc") + "a"
// = reverseString("c") + "b" + "a"
// = "c" + "b" + "a"
// = "cba"
```

Не магия — три вопроса и шаблон `return мойШаг + функция(поменьше)`.

> **Ключевая мысль:** ты не должен "видеть" всю цепочку вызовов в голове. Ты решаешь **только один шаг** и доверяешь остальное самой функции. "Одна цифра — моя, остальное — не моя проблема."

> Рекурсия выглядит сложно — это нормально. Большинство новичков осваивают её не с первого раза. Интуиция приходит через практику — после 5-7 решённых задач паттерн начинает ложиться.

> **В реальной разработке:**
> - Обход DOM-дерева (HTML-документ — это дерево вложенных элементов)
> - Файловая система: папка содержит файлы и другие папки (рекурсивно)
> - Вложенные комментарии (Reddit, Habr): комментарий → ответы → ответы на ответы
> - В React: рекурсивный рендеринг дерева компонентов (меню, файловый менеджер)
> - JSON: вложенные объекты и массивы — flatten, поиск по ключу
