# this — контекст выполнения

> [[js-fundamentals|← Назад к оглавлению JS]]

---

## Зачем эта тема существует?

В JavaScript функции не привязаны к объектам навсегда — одна и та же функция может быть вызвана от имени разных объектов. Чтобы функция знала, с каким объектом она работает, существует `this`. Это один из самых запутанных механизмов в JS: `this` определяется не при написании кода, а в момент вызова функции. Понимание `this` критично для работы с классами, React-компонентами, обработчиками событий.

---

## Что такое this?

**`this`** — это специальная переменная доступная внутри любой функции, которая содержит **ссылку на объект** — так называемый "контекст выполнения".

Проще говоря: `this` — это **объект для которого была вызвана функция**.

### Зачем нужен this?

Представь что у тебя есть метод `greet` и несколько объектов. Вместо того чтобы писать отдельный метод для каждого объекта — один метод через `this` работает для любого:

```js
function greet() {
  return `Привет, я ${this.name}`; // this зависит от того для кого вызван
}

const igor = { name: "Игорь", greet };
const anna = { name: "Анна",  greet };

igor.greet(); // "Привет, я Игорь"
anna.greet(); // "Привет, я Анна"
// Одна функция — два разных результата, в зависимости от объекта
```

---

## Главное правило

> **`this` — это не то где функция написана, а то как и откуда она вызвана.**

Это главное отличие от большинства других языков. В Java, C# `this` всегда = объект класса. В JavaScript это определяется в момент вызова.

---

> **Механика: как JS определяет this в момент вызова**
>
> JS смотрит на **синтаксис вызова** — что стоит перед точкой:
> ```
> user.greet()     JS видит: obj.method() → this = obj (то что перед точкой)
> greet()          JS видит: func()       → this = undefined (ничего перед точкой)
> const fn = user.greet;
> fn()             JS видит: func()       → this = undefined (перед точкой ничего!)
> ```
> Запомни: JS **не помнит** откуда функция взялась. Он смотрит только на
> **момент вызова**. `fn` — это просто функция, JS не знает что она была методом `user`.
> Поэтому `user.greet()` и `fn()` дают разный `this`, хотя это одна и та же функция.

## Случай 1: Метод объекта → this = объект перед точкой

Когда вызываешь функцию через точку (`obj.method()`), `this` внутри метода — это объект перед точкой.

```js
const user = {
  name: "Игорь",
  age: 25,
  greet() {
    // this = user (объект перед точкой)
    return `${this.name}, ${this.age} лет`;
  }
};

user.greet(); // "Игорь, 25 лет" — this = user ✅
```

**Другой объект — другой this:**

```js
const greetFn = function() {
  return `Привет, я ${this.name}`;
};

const igor = { name: "Игорь", greet: greetFn };
const anna = { name: "Анна",  greet: greetFn };

igor.greet(); // "Привет, я Игорь" — this = igor
anna.greet(); // "Привет, я Анна"  — this = anna
// Одна функция — разный this в зависимости от объекта вызова
```

---

## Случай 2: Обычный вызов функции → this = undefined

Когда функция вызывается без объекта перед точкой, `this` = `undefined` (в строгом режиме) или глобальный объект `window` (в браузере без строгого режима).

```js
function show() {
  console.log(this);
}

show(); // undefined (в Node.js с 'use strict' или ES-модулях)
       // window (в браузере без строгого режима)
```

### Что такое строгий режим?

**Строгий режим (`'use strict'`)** — режим выполнения JavaScript который убирает опасные возможности и делает поведение более предсказуемым. В модулях (`import`/`export`) строгий режим включён автоматически.

---

## Случай 3: Стрелочная функция → this из внешнего контекста

### Что происходит со стрелочной функцией?

**Стрелочная функция не имеет своего `this`.** Когда внутри стрелочной функции ты пишешь `this` — это `this` из того места где стрелочная функция была **создана** (не вызвана).

### Зачем это сделали?

Чтобы решить классическую проблему потери `this` в коллбэках:

```js
// ❌ Проблема с обычной функцией в коллбэке
const user = {
  name: "Игорь",
  hobbies: ["программирование", "чтение"],

  showHobbies() {
    // this = user ✅
    this.hobbies.forEach(function(hobby) {
      // ❌ Обычная функция создаёт свой this = undefined
      console.log(`${this.name} любит ${hobby}`); // TypeError!
    });
  }
};

// ✅ Решение: стрелочная функция берёт this из showHobbies
const user = {
  name: "Игорь",
  hobbies: ["программирование", "чтение"],

  showHobbies() {
    // this = user ✅
    this.hobbies.forEach(hobby => {
      // ✅ Стрелка взяла this из showHobbies → это user
      console.log(`${this.name} любит ${hobby}`);
    });
  }
};

user.showHobbies();
// "Игорь любит программирование"
// "Игорь любит чтение"
```

### Как запомнить?

Стрелочная функция — это как фотография. Она "фотографирует" `this` в момент создания и всегда использует это значение, где бы её потом ни вызвали.

## Случай 4: Класс → this = экземпляр

В конструкторе и методах класса `this` — это конкретный создаваемый объект (экземпляр).

```js
class User {
  constructor(name, age) {
    // this = новый объект User который сейчас создаётся
    this.name = name;
    this.age = age;
  }

  greet() {
    // this = конкретный экземпляр (тот на котором вызван метод)
    return `Привет, я ${this.name}, мне ${this.age} лет`;
  }
}

const igor = new User("Игорь", 25); // this в конструкторе = igor
const anna = new User("Анна", 30);  // this в конструкторе = anna

igor.greet(); // this в greet = igor → "Привет, я Игорь, мне 25 лет"
anna.greet(); // this в greet = anna → "Привет, я Анна, мне 30 лет"
```

## Самая частая ловушка: потеря контекста

### Что это?

Когда берёшь метод объекта и присваиваешь его переменной — связь с объектом теряется. При вызове через переменную `this` больше не равен объекту.

```js
const user = {
  name: "Игорь",
  greet() {
    console.log(`Привет, я ${this.name}`);
  }
};

user.greet(); // "Привет, я Игорь" ✅ — this = user

// Сохраняем метод в переменную
const fn = user.greet;
// fn — это просто функция, без связи с user

fn(); // "Привет, я undefined" ❌ — this = undefined!
```

**Почему?** При `fn()` нет объекта перед точкой → `this` = undefined.

### Где это встречается в реальном коде

**1. Передача метода как коллбэк:**

```js
class Button {
  constructor(label) {
    this.label = label;
  }

  handleClick() {
    console.log(`Нажата кнопка: ${this.label}`);
  }

  init() {
    // ❌ Передаём метод — this теряется
    document.addEventListener("click", this.handleClick);
    // При клике: this = undefined (или window)

    // ✅ Вариант 1: стрелочная обёртка
    document.addEventListener("click", () => this.handleClick());

    // ✅ Вариант 2: bind
    document.addEventListener("click", this.handleClick.bind(this));

    // ✅ Вариант 3: сохранить this в переменную (устаревший способ, до ES6)
    const self = this;
    document.addEventListener("click", function() {
      self.handleClick(); // self ссылается на экземпляр Button
    });
  }
}
```

**2. setTimeout с методом:**

```js
class Notification {
  constructor(message) {
    this.message = message;
  }

  show() {
    // ❌ Теряет this
    setTimeout(function() {
      console.log(this.message); // undefined
    }, 1000);

    // ✅ Стрелочная — сохраняет this
    setTimeout(() => {
      console.log(this.message); // "Сообщение" ✅
    }, 1000);
  }
}
```

## Как явно управлять this: call, apply, bind

### call — вызвать функцию с нужным this

**Что делает:** вызывает функцию немедленно, устанавливая `this` равным первому аргументу. Остальные аргументы передаются через запятую.

```js
function introduce(greeting, punctuation) {
  return `${greeting}, я ${this.name}${punctuation}`;
}

const user = { name: "Игорь" };

// call(thisValue, arg1, arg2, ...)
introduce.call(user, "Привет", "!"); // "Привет, я Игорь!"
```

### apply — то же что call, но аргументы массивом

**Что делает:** то же что `call`, но принимает аргументы как массив.

```js
// apply(thisValue, [arg1, arg2, ...])
introduce.apply(user, ["Здравствуйте", "."]);
// "Здравствуйте, я Игорь."
```

**Запомнить разницу:** **C**all = **C**omma (через запятую), **A**pply = **A**rray (массив).

### bind — создать функцию с привязанным this

**Что делает:** возвращает **новую функцию** с намертво привязанным `this`. Не вызывает — только создаёт.

**Зачем нужен:** когда нужно сохранить функцию для последующего вызова и гарантировать нужный `this`.

```js
function greet(greeting) {
  return `${greeting}, я ${this.name}`;
}

const user = { name: "Игорь" };

// Создаём новую функцию с привязанным this
const boundGreet = greet.bind(user);

boundGreet("Привет");   // "Привет, я Игорь" ✅
boundGreet("Хай");      // "Хай, я Игорь" ✅

// Привязанный this нельзя изменить
const another = { name: "Анна" };
boundGreet.call(another, "Привет"); // "Привет, я Игорь" — this = user, не another
```

> **Механика: как bind, call и apply меняют this**
>
> ```
> function greet() { return this.name; }
> const user = { name: "Игорь" };
>
> call:  greet.call(user)
>   → JS вызывает greet() ПРЯМО СЕЙЧАС, подставляя this = user → "Игорь"
>
> apply: greet.apply(user, [])
>   → то же что call, но аргументы передаются массивом → "Игорь"
>
> bind:  const fn = greet.bind(user)
>   → JS создаёт НОВУЮ функцию fn, где this НАВСЕГДА = user
>   → greet не вызвана! fn — это обёртка, которую можно вызвать позже
>   → fn() → "Игорь"
>   → fn.call({ name: "Анна" }) → "Игорь" (bind нельзя перебить!)
> ```
> call/apply — вызывают сразу. bind — создаёт новую функцию для вызова потом.

**Частичное применение (partial application):**

`bind` можно использовать чтобы зафиксировать не только `this`, но и первые аргументы:

```js
function multiply(a, b) {
  return a * b;
}

const double = multiply.bind(null, 2);  // фиксируем a = 2
const triple = multiply.bind(null, 3);  // фиксируем a = 3

double(5);  // 10 — multiply(2, 5)
triple(5);  // 15 — multiply(3, 5)
```

## Стрелочная функция как метод объекта — опасная ловушка

Стрелочные функции решают проблему `this` в коллбэках, но **нельзя** использовать их как методы объекта:

```js
const user = {
  name: "Игорь",

  // ❌ Стрелочная функция как метод
  greetArrow: () => {
    console.log(this.name);
    // this НЕ равен user!
    // Стрелка создана в момент создания объекта (в глобальном контексте)
    // В глобальном контексте this = undefined (или window)
  },

  // ✅ Обычный метод
  greetNormal() {
    console.log(this.name); // "Игорь" ✅
  }
};

user.greetArrow();  // undefined ❌
user.greetNormal(); // "Игорь" ✅
```

**Правило:** в объектных методах — всегда обычные функции. Стрелочные — только внутри методов для коллбэков.

## Свойства класса (class fields) и стрелочные методы

### Что такое свойства класса?

**Свойства класса (class fields)** — это синтаксис объявления свойств прямо в теле класса, без `constructor`. Каждый экземпляр получает свою копию этих свойств.

```js
class User {
  // Свойства класса — создаются для каждого экземпляра
  role = "user";
  loginCount = 0;

  constructor(name) {
    this.name = name; // это тоже свойство, но заданное через конструктор
  }
}

const u = new User("Игорь");
console.log(u.role);       // "user"
console.log(u.loginCount); // 0
```

### Стрелочный метод класса — решение проблемы потери this

Если объявить свойство класса как стрелочную функцию, `this` в ней **навсегда привязан** к экземпляру. Это решает проблему потери контекста при передаче метода как коллбэка.

```js
class Timer {
  count = 0;

  // Обычный метод — this зависит от того КАК вызван
  tickNormal() {
    this.count++;
    console.log(this.count);
  }

  // Стрелочный метод (class field) — this ВСЕГДА = экземпляр
  tick = () => {
    this.count++;
    console.log(this.count);
  }

  start() {
    // ❌ Обычный метод теряет this в setInterval
    // setInterval(this.tickNormal, 1000); // TypeError: this.count is undefined

    // ✅ Стрелочный метод — this не потеряется
    setInterval(this.tick, 1000);
  }
}

const timer = new Timer();
timer.start(); // 1, 2, 3, ... каждую секунду
```

### Обычный метод vs стрелочное свойство — когда что?

| | Обычный метод `greet() {}` | Стрелочное свойство `greet = () => {}` |
|---|---|---|
| `this` | Зависит от вызова | Всегда = экземпляр |
| Хранится | В прототипе (один на всех) | В каждом экземпляре (копия) |
| Передача как коллбэк | Теряет `this` | Не теряет `this` |
| Использование памяти | Экономнее | Больше (копия на экземпляр) |
| Рекомендация | Обычные вызовы | Коллбэки, обработчики событий |

---

> **Частые заблуждения:**
> - Многие думают что `this` определяется при **написании** функции — на самом деле `this` определяется при **вызове**. Одна и та же функция может иметь разный `this` в зависимости от того, как её вызвали.
> - Кажется что стрелочная функция "не имеет `this`" значит `this` в ней будет `undefined` — на самом деле стрелочная функция **берёт `this`** из внешнего лексического окружения (там где она создана).
> - Часто думают что `bind` изменяет существующую функцию — на самом деле `bind` **создаёт новую** функцию. Оригинал остаётся без изменений.
> - Распространённая ошибка: использовать стрелочную функцию как метод объекта и ожидать что `this` будет указывать на объект. Стрелка в литерале объекта берёт `this` из **внешнего** контекста (обычно `undefined` или `window`), а не из объекта.

> **Мини-проверка:** что выведет этот код? Подумай до запуска.
> ```js
> const user = {
>   name: "Игорь",
>   greet() {
>     return `Привет, я ${this.name}`;
>   }
> };
>
> const fn = user.greet;
> console.log(fn());
> ```
> <details><summary>Ответ</summary>
>
> `"Привет, я undefined"` — метод сохранён в переменную и вызван без объекта перед точкой. `this` = `undefined` (strict mode), поэтому `this.name` = `undefined`. Чтобы сохранить контекст: `const fn = user.greet.bind(user)`.
> </details>

---

> **Мини-проверка:** что выведет этот код?
> ```js
> const obj = {
>   name: "Тест",
>   getName: () => this.name,
>   getNameNormal() { return this.name; }
> };
>
> console.log(obj.getName());
> console.log(obj.getNameNormal());
> ```
> <details><summary>Ответ</summary>
>
> `obj.getName()` → `undefined` — стрелочная функция берёт `this` из внешнего контекста (глобальный/модуль), а не из `obj`.
> `obj.getNameNormal()` → `"Тест"` — обычный метод, `this` = объект перед точкой = `obj`.
> </details>

## Итоговая таблица всех случаев

| Как вызвана | Чему равен this | Пример |
|-------------|-----------------|--------|
| `obj.method()` | `obj` | `user.greet()` → `this = user` |
| `function()` | `undefined` / `window` | `greet()` → `this = undefined` |
| `() => {}` | Из внешнего контекста | `arr.map(() => this)` |
| `new Class()` | Новый экземпляр | `new User()` → `this = {}` |
| `fn.call(obj)` | `obj` | `greet.call(user)` |
| `fn.apply(obj, [])` | `obj` | `greet.apply(user, ["Привет"])` |
| `fn.bind(obj)()` | `obj` (навсегда) | `greet.bind(user)()` |

## Итог

```
ЧТО ТАКОЕ this:
  Ссылка на объект для которого вызвана функция.
  Определяется в момент ВЫЗОВА, не в момент написания.

ПРАВИЛА:
  obj.method()   → this = obj
  func()         → this = undefined
  () => {}       → this из внешнего контекста (нет своего)
  new Class()    → this = новый объект

ГЛАВНАЯ ЛОВУШКА:
  const fn = obj.method; fn() → this потерян (undefined)

ГДЕ ТЕРЯЕТСЯ:
  - Метод сохранён в переменную
  - Метод передан как коллбэк (setTimeout, addEventListener, map)

КАК СОХРАНИТЬ:
  () => obj.method()     → обёртка стрелкой
  method.bind(this)      → привязать навсегда
  tick = () => {}        → стрелочный метод класса (class field)

CLASS FIELDS:
  class X { prop = value; method = () => {} }
  Обычный метод → в прототипе, экономит память
  Стрелочное свойство → в каждом экземпляре, зато this привязан

call  → вызвать с нужным this, аргументы через запятую
apply → вызвать с нужным this, аргументы массивом
bind  → создать новую функцию с привязанным this (не вызывает)

ПРАВИЛО ДЛЯ МЕТОДОВ ОБЪЕКТА:
  Методы объекта → обычные функции method() {}
  Коллбэки внутри методов → стрелочные () =>
```
