## Зачем эта тема существует?

Первые программы были линейными: список команд выполнялся всегда одинаково. Потом понадобилось делать разное в зависимости от данных — проверять пароль, давать скидку постоянным клиентам, показывать разные страницы разным пользователям. Так появились условные конструкции.

Без условий у тебя не программа, а скрипт. Условия — это то что превращает набор команд в логику.

<!-- добавлено: 2026-03-07 -->
### Точечная нотация: свойства vs методы

В этой главе ты встретишь записи вроде `cart.length`, `console.log()`, `str.includes("@")`. Все они используют **точечную нотацию** — обращение к чему-то через точку. Есть два вида:

- **Свойства** (данные) — пишутся **без скобок**. Это информация, которая уже хранится:
  ```js
  cart.length    // свойство — количество элементов в массиве
  user.name      // свойство — имя пользователя
  ```

- **Методы** (действия) — пишутся **со скобками `()`**. Это функции, которые что-то делают:
  ```js
  console.log()       // метод — действие: вывести в консоль
  str.includes("@")   // метод — действие: проверить, содержит ли строка символ
  str.toUpperCase()   // метод — действие: преобразовать в верхний регистр
  ```

**Как запомнить:** скобки `()` = действие (глагол). Без скобок = данные (существительное).

> 💡 **На заметку:** подробнее свойства и методы разберём в главах 5–7. Пока достаточно понимать разницу: свойство — это данные, метод — это действие.

## if/else — принятие решений

### Откуда взялось и зачем

`if` — прямой перевод английского "если". Конструкция появилась в одном из первых языков программирования (Fortran, 1957) и с тех пор почти не изменилась: проверь условие — выбери ветку.

### Что это такое

`if` проверяет выражение в скобках. Если оно истинно (`true`) — выполняет первый блок. Если ложно (`false`) — блок `else` (если он есть).

**Аналогия:** турникет в метро. Если баланс > 0 — пропускает. Иначе — сигнал и блокировка. Две ветки, одно условие.

### Синтаксис

```js
if (условие) {
  // выполнится если условие true
} else {
  // выполнится если условие false
}
```

`else` — необязателен. Пиши его только когда нужен альтернативный сценарий.

### Плохой пример → Хороший пример

```js
// ❌ Плохо: два отдельных if — дублирует логику
let message = "";
if (age >= 18) {
  message = "Доступ открыт";
}
if (age < 18) {       // проверяется всегда, даже если первый сработал
  message = "Доступ закрыт";
}

// ✅ Хорошо: if/else — одна конструкция, два сценария
if (age >= 18) {
  message = "Доступ открыт";
} else {
  message = "Доступ закрыт";  // выполнится только если выше false
}
```

> ⚠️ **Важно:** два отдельных `if` — это не то же самое что `if/else`. Два `if` проверяют оба условия всегда. `if/else` — только одно из двух.

### Когда `else` не нужен

```js
// Только один сценарий — просто if без else:
if (!isLoggedIn) {
  redirectToLogin();
}
// если авторизован — просто идём дальше, ничего делать не нужно
```

> ❌ **Частые заблуждения:**
> - "Фигурные скобки `{}` необязательны для одной строки" — технически да, но без них легко сломать код при добавлении второй строки. Всегда пиши `{}`.
> - "Два `if` — то же самое что `if/else`" — нет. Два `if` выполняют оба условия по очереди. `if/else` — только одно из двух.

> 🔍 **Мини-проверка:** что выведет код?
> ```js
> const score = 85;
>
> if (score >= 90) {
>   console.log("Отлично");
> }
> if (score >= 80) {
>   console.log("Хорошо");
> }
> ```
> <details><summary>Ответ</summary>
>
> Выведет `"Хорошо"`. Оба `if` проверяются независимо. `score >= 90` — false, пропускаем. `score >= 80` — true, выводим. Если бы был `else if` — второй не проверялся бы после первого false.
> </details>

## Truthy и Falsy — что считается "истинным"

### Откуда взялось и зачем

В JS условие в `if` не обязано быть строго `true` или `false`. JavaScript автоматически приводит любое значение к булевому типу. Это удобно: `if (user)` вместо `if (user !== null && user !== undefined)`. Но у этого есть ловушки.

### Что это такое

**Falsy** — значения которые ведут себя как `false` в условии. Их ровно 8:

```
false   0   -0   0n   ""   null   undefined   NaN
```

**Truthy** — всё остальное, включая:
```
"0"   " "   []   {}   42   -1   "false"
```

> 💡 **На заметку:** Запоминать все truthy значения не нужно — запомни 8 falsy. Всё остальное — truthy.

> **Механика: как движок JS преобразует значение в boolean**
>
> Когда JS встречает значение в `if (значение)`, он делает ровно одну вещь: сверяет значение с фиксированным списком из 8 falsy-значений.
> - Значение **есть** в списке? Результат: `false`. Значения **нет** в списке? Результат: `true`. Всё.
> - Почему `0` falsy: ноль означает "пустое число", отсутствие количества. Почему `""` falsy: пустая строка = "нет текста".
> - Почему `null` и `undefined` falsy: оба означают "значения нет" (null — явно, undefined — не задано).
> - Почему `[]` (пустой массив) **truthy**: массив — это объект, он существует в памяти как структура. Он не "ничто", он "коробка без содержимого". Коробка есть — значит truthy. А вот `коробка.length === 0` — это уже проверка содержимого.

### Примеры

```js
if ("")       { ... }  // не выполнится — "" это falsy
if (0)        { ... }  // не выполнится — 0 это falsy
if (null)     { ... }  // не выполнится — null это falsy
if (undefined){ ... }  // не выполнится — undefined это falsy

if ("0")      { ... }  // ✅ выполнится — строка "0" это truthy!
if ([])       { ... }  // ✅ выполнится — пустой массив это truthy!
if ({})       { ... }  // ✅ выполнится — пустой объект это truthy!
if (-1)       { ... }  // ✅ выполнится — любое ненулевое число truthy
```

### Ловушка с пустым массивом

```js
const items = [];

// ❌ Неправильная проверка на "пустой":
if (items) {
  console.log("Массив есть"); // ✅ выполнится — [] это truthy!
}

// ✅ Правильная проверка на пустой массив:
// .length — свойство массива, возвращает количество элементов (массивы изучим в главе 6)
if (items.length === 0) {
  console.log("Массив пустой");
}
```

> ❌ **Частые заблуждения:**
> - "Пустой массив `[]` — это falsy". Нет. `[]` — truthy. Falsy только пустая **строка** `""`.
> - "`null` и `undefined` — одно и то же в if". Оба falsy — да. Но `null !== undefined` при строгом сравнении `===`.
> - "`"false"` — это falsy". Нет. Строка `"false"` — truthy, потому что это непустая строка.

> 🔍 **Мини-проверка:** что выведет код? Подумай до запуска.
> ```js
> if ([])  console.log("A");
> if ("")  console.log("B");
> if (0)   console.log("C");
> if (-1)  console.log("D");
> ```
> <details><summary>Ответ</summary>
>
> Выведет `A` и `D`.
> `[]` — truthy ✅, `""` — falsy ❌, `0` — falsy ❌, `-1` — truthy ✅
> </details>

<!-- добавлено: 2026-03-07 -->
### Зачем нужны truthy и falsy на практике

Truthy/falsy существуют не просто так — они позволяют писать **короткие и читаемые проверки** вместо длинных условий.

**Проверка существования значения:**

```js
// ❌ Длинно — проверяем каждый "пустой" вариант вручную:
if (user !== null && user !== undefined) {
  console.log("Пользователь найден");
}

// ✅ Коротко — truthy/falsy делают то же самое:
if (user) {
  console.log("Пользователь найден");
}
// Сработает для любого "непустого" значения:
// строка с именем, объект, число и т.д.
```

**Ловушка с массивами — `[]` truthy, но `length` может быть falsy:**

```js
const cart = []; // пустая корзина

// ❌ Неправильно — пустой массив [] это truthy!
if (cart) {
  console.log("В корзине есть товары"); // выполнится, хотя корзина пустая!
}

// ✅ Правильно — проверяем длину:
if (cart.length) {
  console.log("В корзине есть товары"); // НЕ выполнится: cart.length === 0, а 0 — falsy
}

// Что здесь происходит:
// cart         → []  → truthy (массив существует)
// cart.length  → 0   → falsy  (элементов нет)
```

**Вывод:** truthy/falsy — это инструмент для сокращения кода. Но с массивами и объектами будь внимателен: они всегда truthy, даже если пустые. Проверяй `.length` для массивов.

## else if — несколько вариантов

Когда вариантов больше двух — добавляй `else if`:

```js
const score = 75;

if (score >= 90) {
  console.log("Отлично");
} else if (score >= 70) {
  console.log("Хорошо");         // ✅ выполнится (75 >= 70)
} else if (score >= 50) {
  console.log("Удовлетворительно");
} else {
  console.log("Неудовлетворительно");
}
```

> ⚠️ **Важно:** JS проверяет ветки сверху вниз и **останавливается** при первом совпадении. Порядок условий имеет значение.

<!-- добавлено: 2026-03-07 -->
### Отдельные if vs else if — принципиальная разница

`else if` — **взаимоисключающие** условия. Сработает только ОДНА ветка, остальные даже не проверяются.

Отдельные `if` — **независимые** проверки. Каждое условие проверяется отдельно, могут сработать несколько.

```js
// else if — только ОДНА ветка:
const score = 95;

if (score >= 90) {
  console.log("Отлично");        // ✅ сработает
} else if (score >= 80) {
  console.log("Хорошо");         // ❌ не проверится — уже сработало выше
}

// Отдельные if — обе проверки независимы:
if (score >= 90) {
  console.log("Отлично");        // ✅ сработает
}
if (score >= 80) {
  console.log("Хорошо");         // ✅ тоже сработает! (95 >= 80)
}
```

**Паттерн накопления — когда нужны отдельные `if`:**

Иногда условия не взаимоисключающие, а **накапливающиеся**. Например, скидки:

```js
let discount = 0;

// Каждое условие ДОБАВЛЯЕТ скидку — они не исключают друг друга:
if (isStudent)       discount += 10;  // студент: +10%
if (hasCoupon)       discount += 15;  // купон: +15%
if (isBirthday)      discount += 5;   // день рождения: +5%

// Студент с купоном в день рождения получит 30%!
console.log(`Итоговая скидка: ${discount}%`);
```

Если бы здесь стоял `else if` — сработала бы только первая подходящая скидка, остальные бы игнорировались.

**Правило:** если условия взаимоисключающие (одно ИЛИ другое) — `else if`. Если условия могут комбинироваться — отдельные `if`.
<!-- /добавлено: 2026-03-07 -->

### Порядок меняет результат

```js
// ❌ Неверный порядок — score = 95 попадёт в "Удовлетворительно":
if (score >= 50) {
  console.log("Удовлетворительно");  // сработает для 95!
} else if (score >= 70) {
  console.log("Хорошо");             // сюда уже не дойдём
} else if (score >= 90) {
  console.log("Отлично");            // сюда тоже не дойдём
}

// ✅ Верный порядок — от большего к меньшему:
if (score >= 90) {
  console.log("Отлично");
} else if (score >= 70) {
  console.log("Хорошо");
} else if (score >= 50) {
  console.log("Удовлетворительно");
}
```

## Логические операторы в условиях

Три оператора для составных условий.

### && (И) — оба условия должны быть true

```js
const age = 25;
const hasTicket = true;

if (age >= 18 && hasTicket) {
  console.log("Вход разрешён"); // ✅ оба true
}
```

### || (ИЛИ) — хотя бы одно условие true

```js
const isAdmin = false;
const isModerator = true;

if (isAdmin || isModerator) {
  console.log("Есть права"); // ✅ isModerator = true
}
```

### ! (НЕ) — инверсия

```js
const isLoading = false;

if (!isLoading) {
  console.log("Данные загружены"); // ✅ !false = true
}
```

### Короткое замыкание (Short-circuit evaluation)

`&&` и `||` — ленивые операторы. Они не вычисляют правую часть если левой уже достаточно:

```
&&  — если левая часть falsy → правая не проверяется, возвращается левая часть
||  — если левая часть truthy → правая не проверяется, возвращается левая часть
```

```js
// && возвращает первое falsy или последнее значение:
0 && "привет"       // 0       (левая falsy — стоп, вернули её)
"user" && "admin"   // "admin" (оба truthy — вернули последнее)

// || возвращает первое truthy:
0 || "гость"        // "гость" (левая falsy — идём дальше)
"user" || "гость"   // "user"  (левая truthy — стоп, вернули её)
```

<!-- добавлено: 2026-03-07 -->
**Подробнее о коротком замыкании с цепочками:**

Правило для `&&`: идёт слева направо, возвращает **первое falsy** значение. Если все truthy — возвращает **последнее**.

```js
1 && 2 && 3         // 3   — все truthy, вернул последний
1 && 0 && 3         // 0   — первый falsy, дальше не проверяет
"a" && "" && "c"    // ""  — пустая строка falsy, стоп
true && true && 42  // 42  — все truthy, вернул последний
```

Правило для `||`: идёт слева направо, возвращает **первое truthy** значение. Если все falsy — возвращает **последнее**.

```js
0 || null || "найден" || undefined   // "найден" — первый truthy
0 || "" || null                      // null     — все falsy, вернул последний
false || 0 || 42                     // 42       — первый truthy
```

Правило для `??`: похож на `||`, но проверяет только `null` и `undefined` (не все falsy):

```js
null ?? undefined ?? "дефолт"   // "дефолт" — первый не-null/undefined
0 ?? 42                         // 0        — ноль НЕ null/undefined, вернул его
"" ?? "запасной"                // ""       — пустая строка НЕ null/undefined
```

> 💡 **Как запомнить:** `&&` ищет причину сказать "нет" (первый falsy). `||` ищет причину сказать "да" (первый truthy). `??` ищет причину сказать "нет данных" (null/undefined).

> **Механика: пошаговая трассировка `&&` и `||`**
>
> `console.log(1 && 2 && 3)` — что вернёт?
> Шаг 1: берём `1` -- truthy? Да -- значит `&&` ещё не может сказать "нет", идём дальше.
> Шаг 2: берём `2` -- truthy? Да -- всё ещё не может сказать "нет", идём дальше.
> Шаг 3: берём `3` -- это последнее значение -- возвращаем его как есть: **3**.
> Ключевой факт: `&&` возвращает не `true`/`false`, а **само значение**, на котором остановился.
>
> `console.log(0 || "" || "hello")` — что вернёт?
> Шаг 1: берём `0` -- truthy? Нет -- `||` ещё не нашёл "да", идём дальше.
> Шаг 2: берём `""` -- truthy? Нет -- всё ещё не нашёл, идём дальше.
> Шаг 3: берём `"hello"` -- это последнее значение -- возвращаем его: **"hello"**.
> Так же как `&&`, оператор `||` возвращает **само значение**, а не boolean.
<!-- /добавлено: 2026-03-07 -->

**Практическое применение:**

```js
// Значение по умолчанию через ||:
// user.name — доступ к свойству объекта (объекты изучим в главе 7)
const username = user.name || "Гость";

// Условное выполнение через &&:
// showUserMenu() — вызов функции (функции изучим в главе 4)
isLoggedIn && showUserMenu(); // вызовется только если isLoggedIn truthy
```

> ⚠️ **Важно:** `||` и значение `0`. Если нужно разрешить значение `0` как допустимое — используй `??` (оператор нулевого слияния из главы 1):
> ```js
> const count = 0;
> count || 10   // 10  — плохо! 0 считается falsy, заменяется на 10
> count ?? 10   // 0   — хорошо! ?? проверяет только null/undefined
> ```

## Тернарный оператор

### Откуда взялся и зачем

Тернарный оператор — единственный в JS оператор с тремя операндами (отсюда название: "тернарный" = "трёхчленный"). Это сокращение `if/else` для случаев когда нужно вернуть одно из двух значений.

### Синтаксис

```js
условие ? значение_если_true : значение_если_false
```

**Аналогия:** развилка на дороге. Налево или направо — только одно из двух, без третьего варианта.

### Плохой пример → Хороший пример

```js
// ❌ Многословно для простого выбора значения:
let label;
if (age >= 18) {
  label = "Взрослый";
} else {
  label = "Ребёнок";
}

// ✅ Тернарный — одна строка, то же самое:
const label = age >= 18 ? "Взрослый" : "Ребёнок";
```

### Когда НЕ использовать

Тернарный — только для простых двух вариантов. Никогда не вкладывай тернарный в тернарный:

```js
// ❌ Вложенный тернарный — нечитаемо:
const result = a > b ? a > c ? a : c : b > c ? b : c;

// ✅ Лучше if/else — понятно:
let result;
if (a > b && a > c)  result = a;
else if (b > c)      result = b;
else                 result = c;
```

> Вложенные тернарные сбивают с толку даже опытных разработчиков. Это нормально — просто используй `if/else` для сложной логики.

<!-- добавлено: 2026-03-16 -->
### Лишний тернарный: `? true : false`

Если условие **уже возвращает boolean** (true/false), тернарный не нужен — он ничего не добавляет:

```js
// ❌ Лишний тернарный — условие уже boolean:
return age >= 18 ? true : false   // "если true верни true, если false верни false"

// ✅ Достаточно:
return age >= 18                  // уже true или false
```

Все операторы сравнения (`>`, `<`, `>=`, `<=`, `===`, `!==`) возвращают boolean. Результат можно сразу вернуть или присвоить:

```js
const isAdult = age >= 18;        // ✅ isAdult = true или false
const hasAccess = role === "admin"; // ✅ hasAccess = true или false
```

**Когда тернарный НУЖЕН** — когда возвращаешь **не boolean**, а что-то другое:

```js
age >= 18 ? "взрослый" : "ребёнок"   // строки — нужен тернарный
age >= 18 ? 200 : 0                   // числа — нужен тернарный
age >= 18                             // boolean — НЕ нужен
```

> **Правило:** если после `?` стоит `true`, а после `:` стоит `false` — убирай тернарный, оставляй само условие.
<!-- /добавлено: 2026-03-16 -->

<!-- добавлено: 2026-03-07 -->
### Тернарный — выражение, if/else — инструкция

Это важное различие, которое объясняет **когда можно использовать тернарный, а когда нельзя**.

**Выражение (expression)** — это код, который **возвращает значение**. Его можно присвоить переменной, передать в функцию, использовать внутри другого выражения.

**Инструкция (statement)** — это код, который **выполняет действие**, но не возвращает значение. Его нельзя присвоить переменной.

```js
// ✅ Тернарный — это ВЫРАЖЕНИЕ. Возвращает значение, можно присвоить:
const msg = age >= 18 ? "Доступ открыт" : "Доступ закрыт";
console.log(age >= 18 ? "да" : "нет"); // можно передать в функцию

// ❌ if/else — это ИНСТРУКЦИЯ. Нельзя присвоить:
const msg = if (age >= 18) { "да" } else { "нет" }; // SyntaxError!
```

**Где это важно на практике:**

```js
// Внутри шаблонной строки — нужно выражение:
console.log(`Статус: ${isActive ? "активен" : "неактивен"}`); // ✅ тернарный
console.log(`Статус: ${if (isActive) { "активен" }}`);        // ❌ нельзя

// При передаче аргумента в функцию:
greet(isVIP ? "Уважаемый клиент" : "Гость"); // ✅ тернарный
greet(if (isVIP) { "Уважаемый клиент" });     // ❌ нельзя
```

**Правило:** если нужно **получить значение** — тернарный. Если нужно **выполнить разный код** (несколько строк, вызовы функций) — `if/else`.
<!-- /добавлено: 2026-03-07 -->

> 🔍 **Мини-проверка:** что выведет код?
> ```js
> const n = -5;
> const result = n > 0 ? "положительное" : "не положительное";
> console.log(result);
> ```
> <details><summary>Ответ</summary>
>
> `"не положительное"`. `-5 > 0` — false, берётся правая часть тернарного оператора.
> </details>

## switch — выбор из конкретных значений

### Откуда взялось и зачем

`switch` появился как альтернатива длинной цепочке `else if` когда все условия проверяют **одну переменную** на конкретные значения. Читается намного чище когда вариантов 5 и больше.

### Синтаксис

```js
switch (выражение) {
  case значение1:
    // код
    break;
  case значение2:
    // код
    break;
  default:
    // выполнится если ни один case не совпал
}
```

### Ловушка: забытый break

Без `break` выполнение "проваливается" в следующий `case`:

```js
// ❌ Забытый break — провал:
const day = 1;

switch (day) {
  case 1:
    console.log("Понедельник"); // ✅ выполнится
  case 2:
    console.log("Вторник");     // ✅ тоже выполнится! break забыт
  case 3:
    console.log("Среда");       // ✅ и это тоже!
}
// Вывод: "Понедельник", "Вторник", "Среда"

// ✅ С break — только нужный case:
switch (day) {
  case 1:
    console.log("Понедельник");
    break;
  case 2:
    console.log("Вторник");
    break;
}
```

> **Механика: почему без `break` выполнение "проваливается"**
>
> `switch` работает так: находит совпавший `case` и **прыгает на эту строку**. Дальше он просто выполняет весь код ниже **строка за строкой** — как чтение книги с закладки до конца.
> Он НЕ проверяет следующие `case` — он их просто **перешагивает** и выполняет код под ними.
> `break` — это команда "стоп, закончи чтение". Без неё движок продолжает читать и выполнять всё подряд до конца `switch` или до первого `break`.
> Именно поэтому без `break` после `case 1` выполнится и код из `case 2`, и из `case 3` — движок просто не знает, что пора остановиться.

### "Провал" используется намеренно — для группировки

```js
const day = "понедельник";

switch (day) {
  case "понедельник":
  case "вторник":
  case "среда":
  case "четверг":
  case "пятница":
    console.log("Рабочий день"); // для всех пяти
    break;
  case "суббота":
  case "воскресенье":
    console.log("Выходной");
    break;
  default:
    console.log("Неизвестный день");
}
```

### switch vs if/else — когда что выбрать

| Ситуация | Что лучше |
|----------|-----------|
| Одна переменная, много конкретных значений | `switch` |
| Диапазоны (`>18`, `<60`) | `if/else` |
| Сложные условия с `&&` или `\|\|` | `if/else` |
| 2–3 варианта | `if/else` или тернарный |

> 💡 **На заметку:** `switch` сравнивает значения через строгое равенство `===`. Это значит `switch ("1")` и `case 1:` — не совпадут (разные типы).

<!-- добавлено: 2026-03-07 -->
### return vs console.log в функциях с условиями

Когда ты пишешь функцию с `switch` или `if/else`, важно понимать разницу между `console.log` и `return`:

- **`console.log()`** — выводит текст в консоль **для отладки**. Функция при этом возвращает `undefined`.
- **`return`** — **возвращает значение** вызывающему коду. Функцию можно использовать дальше.

```js
// ❌ С console.log — функция возвращает undefined:
function getDayType(day) {
  switch (day) {
    case "суббота":
    case "воскресенье":
      console.log("Выходной");  // просто печатает в консоль
      break;
    default:
      console.log("Рабочий день");
      break;
  }
}

const result = getDayType("суббота");
console.log(result); // undefined — функция ничего не вернула!

// ✅ С return — функция возвращает значение:
function getDayType(day) {
  switch (day) {
    case "суббота":
    case "воскресенье":
      return "Выходной";     // возвращает строку
    default:
      return "Рабочий день"; // return автоматически завершает функцию (break не нужен)
  }
}

const result = getDayType("суббота");
console.log(result); // "Выходной" — можно использовать дальше
```

> 💡 **На заметку:** `return` внутри `switch` автоматически выходит и из `case`, и из функции — `break` после `return` не нужен. Подробнее о функциях — в главе 4.

## Итог

```
КОНСТРУКЦИИ:
if (условие) {}                        → основная конструкция
if (условие) {} else {}               → два варианта
if (условие) {} else if {} else {}    → несколько вариантов
условие ? значение_true : значение_false → тернарный (краткая запись)
switch (значение) { case X: ... break; } → выбор из конкретных значений

ЛОГИЧЕСКИЕ ОПЕРАТОРЫ:
&&  → И (оба true)
||  → ИЛИ (хотя бы одно true)
!   → НЕ (инверсия)

FALSY (ведут себя как false в условии):
false   0   -0   0n   ""   null   undefined   NaN
ВСЁ ОСТАЛЬНОЕ — truthy (включая [], {}, "0", -1)
```

## Попробуй сам

**1. Предскажи вывод:**
```js
let x = 0;
if (x) {
  console.log("A");
} else {
  console.log("B");
}
```

<details>
<summary>Ответ</summary>
<p>Выведет `"B"`. Потому что `0` — falsy, условие `if (0)` ложно.</p>
</details>

**2. Что вернёт каждое выражение?**
```js
let user = { name: "", age: 0 };
let name = user.name || "Аноним";
let age = user.age ?? 18;
console.log(name, age);
```

<details>
<summary>Ответ</summary>
<pre>name = "Аноним" (потому что "" — falsy для ||)
age = 0          (потому что 0 — НЕ null/undefined для ??)
Выведет: "Аноним" 0</pre>
<p>Вот почему для значений где 0 или "" валидны — используй `??` а не `||`.</p>
</details>

**3. Исправь код** — скидки должны суммироваться:
```js
function getDiscount(user) {
  let discount = 0;
  if (user.isVip) {
    discount = 20;
  } else if (user.orders > 10) {
    discount = 10;
  }
  return discount;
}
// VIP с 15 заказами должен получить 30, а получает 20
```

<details>
<summary>Ответ</summary>
<p>Проблема: `else if` выполняет только ОДНУ ветку. Если VIP — вторая проверка не выполнится. Нужны отдельные `if`:</p>
<pre>if (user.isVip) discount += 20;
if (user.orders > 10) discount += 10;</pre>
<p>Два отдельных `if` проверяются независимо. Теперь VIP с 15 заказами получит 20 + 10 = 30.</p>
</details>
