## Зачем эта тема существует?

Программы часто выполняют одно и то же действие много раз: обработать каждый элемент списка, повторять запрос пока пользователь не введёт корректные данные, отрисовать каждый пиксель на экране. Без циклов пришлось бы копировать одинаковый код сотни раз — менять его стало бы невозможно.

Циклы появились в первых языках программирования (Fortran, 1957) и с тех пор остаются фундаментальной конструкцией. JavaScript унаследовал `for` и `while` из C (1972), а `for...of` добавил в ES6 (2015) для более удобного перебора коллекций.

## Цикл for — когда знаешь количество итераций

### Откуда взялось и зачем

`for` — самый старый и универсальный цикл. Появился в языке C и почти без изменений перешёл в JavaScript. Используется когда заранее известно сколько раз нужно повторить действие, или когда нужен доступ к индексу (порядковому номеру) элемента.

### Что это такое

`for` — конструкция из трёх частей: начало, условие, шаг. Тело цикла выполняется пока условие истинно.

**Аналогия:** бегун на стадионе. Он знает сколько кругов бежать (условие), считает каждый круг (счётчик), и останавливается когда добежал нужное количество.

```
┌─────────────────────────────────────────┐
│  for (let i = 0; i < 3; i++) { ... }   │
│        ▲          ▲        ▲            │
│        │          │        │            │
│   начало       условие    шаг           │
│   (один раз)  (перед      (после        │
│               каждой      каждой        │
│               итерацией)  итерации)     │
└─────────────────────────────────────────┘

Порядок выполнения:
начало → [условие → тело → шаг] → [условие → тело → шаг] → ... → условие=false → выход
```

### Синтаксис

```js
for (начало; условие; шаг) {
  // тело цикла — выполняется пока условие true
}
```

Три части:
1. **начало** — выполняется один раз перед циклом (`let i = 0`)
2. **условие** — проверяется перед каждой итерацией (`i < 5`)
3. **шаг** — выполняется после каждой итерации (`i++`)

### Как работает — по шагам

```js
for (let i = 0; i < 3; i++) {
  console.log(i);
}
// 0
// 1
// 2
```

| Итерация | i до | Условие i < 3 | Выполнение | i после |
|----------|------|---------------|------------|---------|
| 1 | 0 | true | `console.log(0)` | 1 |
| 2 | 1 | true | `console.log(1)` | 2 |
| 3 | 2 | true | `console.log(2)` | 3 |
| — | 3 | false | цикл заканчивается | — |

### Примеры

**Перебор массива по индексу:**

**Забегая вперёд:** массивы (`[...]`) — упорядоченные списки значений — подробно разбираются в главе 6. Здесь мы используем их только для демонстрации циклов. `.length` — свойство, возвращающее количество элементов. `fruits[i]` — доступ к элементу по индексу (нумерация с 0).

```js
const fruits = ["яблоко", "банан", "груша"];

for (let i = 0; i < fruits.length; i++) {
  console.log(i, fruits[i]);
}
// 0 "яблоко"
// 1 "банан"
// 2 "груша"
```

<details>
<summary>🔧 Механика: пошаговая трассировка</summary>
<pre>fruits.length = 3
i=0: 0 < 3? да → log(0, fruits[0]) → "0 яблоко" → i++ → i=1
i=1: 1 < 3? да → log(1, fruits[1]) → "1 банан"  → i++ → i=2
i=2: 2 < 3? да → log(2, fruits[2]) → "2 груша"  → i++ → i=3
i=3: 3 < 3? нет → выход</pre>
<p>Индексация с 0: первый элемент — `fruits[0]`, последний — `fruits[2]` (длина - 1).</p>
</details>

<!-- добавлено: 2026-03-09 -->
> **Важно: `<` vs `<=` при переборе массива**
>
> Индексы массива идут от `0` до `length - 1`. Поэтому:
> - `i < length` — проходит все элементы (0, 1, 2 при length=3)
> - `i <= length` — **ошибка!** Последняя итерация обратится к несуществующему элементу (`arr[3]` = `undefined`)
> - `length - 1` — последний индекс, используется чтобы начать с конца
>
> ```
> arr = ["a", "b", "c", "d"]    length = 4
> индексы:  0    1    2    3    последний = length - 1 = 3
>
> i < length      → i = 0, 1, 2, 3     ✓ все элементы
> i <= length     → i = 0, 1, 2, 3, 4  ✗ arr[4] = undefined!
> i = length - 1  → начинаем с 3       ✓ для обратного перебора
> ```
>
> `<=` используется когда перебираешь **числа**, а не индексы (например `i <= 100` для суммы от 1 до 100).

**Обратный порядок:**

```js
for (let i = fruits.length - 1; i >= 0; i--) {
  console.log(fruits[i]);
}
// "груша"
// "банан"
// "яблоко"
```

<details>
<summary>🔧 Механика: обратный перебор</summary>
<pre>fruits.length = 3, значит начальное i = 3 - 1 = 2
i=2: 2 >= 0? да → log(fruits[2]) → "груша"  → i-- → i=1
i=1: 1 >= 0? да → log(fruits[1]) → "банан"  → i-- → i=0
i=0: 0 >= 0? да → log(fruits[0]) → "яблоко" → i-- → i=-1
i=-1: -1 >= 0? нет → выход</pre>
<p>Начинаем с последнего индекса (`length - 1`), идём до 0 включительно.</p>
</details>

**Сумма чисел от 1 до N:**

```js
let sum = 0;
for (let i = 1; i <= 100; i++) {
  sum += i;  // то же что sum = sum + i
}
console.log(sum); // 5050
```

<details>
<summary>🔧 Механика: как накапливается сумма</summary>
<pre>sum = 0
i=1:   sum = 0 + 1 = 1
i=2:   sum = 1 + 2 = 3
i=3:   sum = 3 + 3 = 6
i=4:   sum = 6 + 4 = 10
...
i=100: sum = 4950 + 100 = 5050
i=101: 101 <= 100? нет → выход</pre>
<p>`sum += i` — это `sum = sum + i`. Каждую итерацию к sum прибавляется текущее значение i. За 100 итераций накопится сумма всех чисел от 1 до 100.</p>
</details>

### Плохой пример → Хороший пример

```js
// ❌ Плохо: создание переменной в глобальной области
var i;  // доступна за пределами цикла — загрязняет область видимости
for (i = 0; i < 5; i++) {
  console.log(i);
}
console.log(i); // 5 — переменная "утекла" из цикла

// ✅ Хорошо: let в инициализаторе — переменная живёт только внутри цикла
for (let i = 0; i < 5; i++) {
  console.log(i);
}
// console.log(i); // ReferenceError — i не существует вне цикла
```

<details>
<summary>🔧 Механика: почему i равно 5 после цикла с var?</summary>
<p>Пошагово — что происходит внутри:</p>
<pre>Шаг 1: i = 0  → 0 < 5? да  → console.log(0) → i++ → i стало 1
Шаг 2: i = 1  → 1 < 5? да  → console.log(1) → i++ → i стало 2
Шаг 3: i = 2  → 2 < 5? да  → console.log(2) → i++ → i стало 3
Шаг 4: i = 3  → 3 < 5? да  → console.log(3) → i++ → i стало 4
Шаг 5: i = 4  → 4 < 5? да  → console.log(4) → i++ → i стало 5
Шаг 6: i = 5  → 5 < 5? НЕТ → цикл ОСТАНАВЛИВАЕТСЯ</pre>
<p>На шаге 5 `i++` уже выполнился и `i` стало 5. На шаге 6 проверка `5 < 5` дала `false` — цикл прекратился, но `i` так и осталось 5.</p>
<p>С `var` переменная существует за пределами цикла — поэтому `console.log(i)` после цикла выводит 5.</p>
<p>С `let` переменная живёт только внутри `{ }` цикла — снаружи её просто нет, отсюда ReferenceError.</p>
</details>

<details>
<summary>❌ Частые заблуждения</summary>
- "Цикл `for` можно использовать только со счётчиком `i`" — имя переменной может быть любым: `j`, `index`, `count`. `i` — просто традиция (от слова "iterator").
- "`i++` и `++i` в цикле for — разные вещи" — в цикле `for` результат одинаковый, потому что возвращаемое значение шага нигде не используется. Разница есть только когда результат присваивается переменной.
- "Условие проверяется после выполнения тела" — нет, условие проверяется **перед** каждой итерацией. Если условие false с самого начала — тело не выполнится ни разу.
</details>

**🔍 Мини-проверка:** что выведет этот код? Подумай до запуска.

```js
for (let i = 10; i > 0; i -= 3) {
  console.log(i);
}
```

<details>
<summary>Ответ</summary>
<p>Выведет `10`, `7`, `4`, `1`. Пошагово:</p>
<pre>i=10: 10 > 0? да → log(10) → i -= 3 → i=7
i=7:  7 > 0? да  → log(7)  → i -= 3 → i=4
i=4:  4 > 0? да  → log(4)  → i -= 3 → i=1
i=1:  1 > 0? да  → log(1)  → i -= 3 → i=-2
i=-2: -2 > 0? нет → выход</pre>
</details>

## Цикл while — пока выполняется условие

### Откуда взялось и зачем

`while` старше `for` — это самая базовая форма цикла. В отличие от `for`, у `while` нет встроенного счётчика и шага. Он просто проверяет условие и выполняет тело.

Используется когда **заранее неизвестно** сколько итераций нужно: читать данные пока они есть, повторять пока пользователь не введёт корректное значение, ждать пока не будет выполнено условие.

### Что это такое

`while` выполняет тело цикла **пока** условие истинно. Проверка — перед каждой итерацией.

**Аналогия:** ты моешь посуду. Не считаешь тарелки — просто моешь пока есть грязные. Когда грязных не осталось — останавливаешься.

```
┌────────────┐
│ Проверить   │ ← условие false → выход из цикла
│ условие     │
└──────┬─────┘
       │ true
       ▼
┌────────────┐
│ Выполнить   │
│ тело цикла  │
└──────┬─────┘
       │
       └──→ обратно к проверке условия
```

### Синтаксис

```js
while (условие) {
  // тело цикла
  // ВАЖНО: нужно изменять условие, иначе бесконечный цикл!
}
```

### Примеры

**Простой счётчик:**

```js
let count = 0;
while (count < 5) {
  console.log(count);
  count++; // ВАЖНО: без этого — бесконечный цикл
}
// 0, 1, 2, 3, 4
```

<details>
<summary>🔧 Механика: while-счётчик пошагово</summary>
<pre>count=0: 0 < 5? да → log(0) → count++ → count=1
count=1: 1 < 5? да → log(1) → count++ → count=2
count=2: 2 < 5? да → log(2) → count++ → count=3
count=3: 3 < 5? да → log(3) → count++ → count=4
count=4: 4 < 5? да → log(4) → count++ → count=5
count=5: 5 < 5? нет → выход</pre>
<p>Без `count++` значение count навсегда осталось бы 0, условие `0 < 5` навсегда `true` — бесконечный цикл.</p>
</details>

**Поиск первого отрицательного числа:**

```js
const numbers = [5, 12, 3, -7, 8, -2];
let i = 0;

while (i < numbers.length && numbers[i] >= 0) {
  i++;
}

console.log(`Первое отрицательное: ${numbers[i]}`); // -7
```

<details>
<summary>🔧 Механика: поиск с двумя условиями</summary>
<pre>numbers = [5, 12, 3, -7, 8, -2], i = 0

i=0: 0 < 6 && numbers[0] >= 0 → 0<6? да, 5>=0? да → оба true → i++ → i=1
i=1: 1 < 6 && numbers[1] >= 0 → 1<6? да, 12>=0? да → оба true → i++ → i=2
i=2: 2 < 6 && numbers[2] >= 0 → 2<6? да, 3>=0? да → оба true → i++ → i=3
i=3: 3 < 6 && numbers[3] >= 0 → 3<6? да, -7>=0? НЕТ → false → выход

i = 3 → numbers[3] = -7 → "Первое отрицательное: -7"</pre>
<p>Условие `&&` — оба должны быть true. Как только `numbers[i] >= 0` стало false (нашли отрицательное), цикл останавливается.</p>
</details>

### Плохой пример → Хороший пример

```js
// ❌ Плохо: while для простого перебора от 0 до N
let i = 0;
while (i < 10) {
  console.log(i);
  i++;
}
// Три строчки на управление циклом вместо одной

// ✅ Хорошо: для такой задачи лучше for
for (let i = 0; i < 10; i++) {
  console.log(i);
}
// Всё управление — в одной строке
```

**💡 На заметку:** Используй `while` когда не знаешь количество итераций заранее. Если количество известно — `for` нагляднее.

### Ловушка: бесконечный цикл

```js
// ❌ ОПАСНО — бесконечный цикл, программа зависнет
let i = 0;
while (i < 5) {
  console.log(i);
  // Забыли i++ — i всегда 0, условие всегда true → зависание
}
```

<details>
<summary>🔧 Механика: почему зависает</summary>
<pre>i=0: 0 < 5? да → log(0) → (i++ нет, i всё ещё 0)
i=0: 0 < 5? да → log(0) → (i всё ещё 0)
i=0: 0 < 5? да → log(0) → ...
... навсегда — условие никогда не станет false</pre>
<p>Цикл не может закончиться, потому что ничто не меняет `i`. Условие `0 < 5` будет `true` бесконечно.</p>
</details>

**⚠️ Важно:** если программа "зависла" — скорее всего бесконечный цикл. В браузере нажми `Ctrl+Shift+I` → Console → остановить скрипт. В Node.js — `Ctrl+C`.

## do...while — выполнить хотя бы один раз

### Что это такое

`do...while` — вариант `while`, который сначала выполняет тело, а потом проверяет условие. Гарантирует хотя бы одну итерацию.

<details>
<summary>🔧 Механика: почему do...while гарантирует хотя бы одно выполнение?</summary>
<p>Сравни порядок шагов:</p>
<pre>while:      [условие] → тело → [условие] → тело → ...
             ↑ если false с самого начала — тело НЕ выполнится

do...while: тело → [условие] → тело → [условие] → ...
             ↑ тело выполняется ДО первой проверки — всегда хотя бы раз</pre>
<p>Разница только в том, ГДЕ стоит первая проверка: до или после тела.</p>
</details>

```
┌────────────┐
│ Выполнить   │ ← тело выполняется ДО проверки
│ тело цикла  │
└──────┬─────┘
       │
       ▼
┌────────────┐
│ Проверить   │ ← условие false → выход
│ условие     │
└──────┬─────┘
       │ true
       └──→ обратно к телу
```

### Синтаксис

```js
do {
  // тело — выполнится хотя бы раз
} while (условие);  // точка с запятой обязательна!
```

### Пример

```js
// Генерируем случайные числа пока не выпадет больше 0.9
let randomNumber;
do {
  randomNumber = Math.random(); // случайное число от 0 до 1
  console.log(randomNumber);
} while (randomNumber <= 0.9);
// Тело выполнится хотя бы раз, даже если первое число > 0.9
```

<details>
<summary>🔧 Механика: пример с Math.random()</summary>
<pre>Допустим Math.random() выдаёт: 0.34, 0.72, 0.95

Итерация 1: randomNumber = 0.34 → log(0.34) → 0.34 <= 0.9? да → повтор
Итерация 2: randomNumber = 0.72 → log(0.72) → 0.72 <= 0.9? да → повтор
Итерация 3: randomNumber = 0.95 → log(0.95) → 0.95 <= 0.9? нет → выход

А если первое число 0.99?
Итерация 1: randomNumber = 0.99 → log(0.99) → 0.99 <= 0.9? нет → выход
Тело всё равно выполнилось 1 раз! (в отличие от while, который бы не начал)</pre>
</details>

<details>
<summary>❌ Частые заблуждения</summary>
- "`do...while` и `while` — одно и то же" — нет. `while` может не выполнить тело ни разу (если условие сразу false). `do...while` всегда выполнит хотя бы раз.
- "`do...while` используется часто" — на практике встречается редко. В 95% случаев подходит обычный `while` или `for`.
</details>

**🔍 Мини-проверка:** что выведет этот код? Подумай до запуска.

```js
let x = 10;
do {
  console.log(x);
  x++;
} while (x < 5);
```

<details>
<summary>Ответ</summary>
<p>Выведет `10`. Пошагово:</p>
<pre>x=10: выполняем тело → log(10) → x++ → x=11
      проверяем: 11 < 5? нет → выход</pre>
<p>Тело выполнилось 1 раз, хотя условие `x < 5` было false с самого начала. Обычный `while (x < 5)` при `x = 10` не выполнил бы тело ни разу.</p>
</details>

## for...of — перебор элементов коллекции

### Откуда взялось и зачем

До ES6 (2015) для перебора массива приходилось писать `for (let i = 0; i < arr.length; i++)` — громоздко, легко ошибиться с индексом. `for...of` создали специально для простого перебора **значений** любой итерируемой коллекции (массивы, строки, Map, Set).

### Что это такое

`for...of` перебирает **значения** коллекции. Не нужен индекс, не нужен счётчик — просто "для каждого элемента из коллекции".

**Аналогия:** раздача карт за столом. Берёшь верхнюю карту, кладёшь игроку, берёшь следующую — пока колода не кончится. Номер карты в колоде тебя не интересует.

### Синтаксис

```js
for (const element of iterable) {
  // element — текущий элемент
}
```

### Примеры

**Перебор массива:**

```js
const fruits = ["яблоко", "банан", "груша"];

for (const fruit of fruits) {
  console.log(fruit);
}
// "яблоко"
// "банан"
// "груша"
```

<details>
<summary>🔧 Механика: for...of под капотом</summary>
<pre>Итерация 1: fruit = "яблоко" (первый элемент) → log("яблоко")
Итерация 2: fruit = "банан"  (второй элемент) → log("банан")
Итерация 3: fruit = "груша"  (третий элемент)  → log("груша")
Элементов больше нет → выход</pre>
<p>Цикл сам берёт следующий элемент из массива. Не нужен индекс, не нужен `length`.</p>
</details>

**Перебор строки (строка — тоже итерируемое):**

```js
for (const char of "привет") {
  console.log(char);
}
// "п", "р", "и", "в", "е", "т"
```

<details>
<summary>🔧 Механика: строка как последовательность символов</summary>
<pre>"привет" — 6 символов
Итерация 1: char = "п" → log("п")
Итерация 2: char = "р" → log("р")
Итерация 3: char = "и" → log("и")
Итерация 4: char = "в" → log("в")
Итерация 5: char = "е" → log("е")
Итерация 6: char = "т" → log("т")
Символов больше нет → выход</pre>
<p>Строка в JS — итерируемый объект. `for...of` перебирает её посимвольно.</p>
</details>

**С массивом объектов:**

**Забегая вперёд:** `{ name, price }` — деструктуризация объекта (гл. 7). Объект `{ name: "Хлеб", price: 50 }` — структура данных с именованными свойствами (гл. 7). Здесь деструктуризация позволяет сразу извлечь нужные значения из объекта в переменные.

```js
const products = [
  { name: "Хлеб", price: 50 },
  { name: "Молоко", price: 80 },
];

for (const { name, price } of products) {
  console.log(`${name}: ${price} руб.`);
}
// "Хлеб: 50 руб."
// "Молоко: 80 руб."
```

<details>
<summary>🔧 Механика: деструктуризация в for...of</summary>
<pre>products = [{ name: "Хлеб", price: 50 }, { name: "Молоко", price: 80 }]

Итерация 1: текущий элемент = { name: "Хлеб", price: 50 }
  деструктуризация: const { name, price } = { name: "Хлеб", price: 50 }
  → name = "Хлеб", price = 50
  → log("Хлеб: 50 руб.")

Итерация 2: текущий элемент = { name: "Молоко", price: 80 }
  деструктуризация: const { name, price } = { name: "Молоко", price: 80 }
  → name = "Молоко", price = 80
  → log("Молоко: 80 руб.")</pre>
<p>`const { name, price }` — извлекает свойства `name` и `price` из объекта в отдельные переменные. Каждая итерация — новая область видимости, новые `const`.</p>
</details>

### Плохой пример → Хороший пример

```js
const colors = ["красный", "зелёный", "синий"];

// ❌ Плохо: for с индексом когда индекс не нужен
for (let i = 0; i < colors.length; i++) {
  console.log(colors[i]);
}
// Лишний код: объявление i, проверка длины, обращение по индексу

// ✅ Хорошо: for...of — чисто и понятно
for (const color of colors) {
  console.log(color);
}
```

<details>
<summary>🔧 Механика: сравнение двух подходов</summary>
<pre>colors = ["красный", "зелёный", "синий"]

❌ for с индексом (6 операций на каждой итерации):
  i=0: проверить 0<3, взять colors[0], log, i++  → "красный"
  i=1: проверить 1<3, взять colors[1], log, i++  → "зелёный"
  i=2: проверить 2<3, взять colors[2], log, i++  → "синий"

✅ for...of (2 операции на каждой итерации):
  color = "красный" → log  → "красный"
  color = "зелёный" → log  → "зелёный"
  color = "синий"   → log  → "синий"</pre>
<p>Результат одинаковый, но `for...of` проще: не надо помнить про индекс, длину и `colors[i]`.</p>
</details>

### for...of vs обычный for — когда что

| Ситуация | Что использовать |
|----------|-----------------|
| Нужен индекс элемента | `for` |
| Нужно перебрать каждый N-й элемент | `for` |
| Просто перебрать все значения | `for...of` |
| Нужен `break` при переборе | `for` или `for...of` (оба поддерживают) |

<details>
<summary>❌ Частые заблуждения</summary>
- "`for...of` работает с любым объектом" — нет. Работает только с **итерируемыми** (iterable): массивы, строки, Map, Set. Обычный объект `{}` — не итерируемый (для него есть `for...in`).
- "`const` в `for...of` — ошибка, ведь переменная меняется" — нет. На каждой итерации создаётся **новая** переменная `element`. Каждая итерация — новая область видимости.
</details>

<details>
<summary>🔧 Механика: как const работает в for...of?</summary>
<p>Движок JS разворачивает `for (const x of [1, 2, 3])` примерно так:</p>
<pre>Итерация 1: { const x = 1; console.log(x); }  ← своя область видимости
Итерация 2: { const x = 2; console.log(x); }  ← ДРУГАЯ область видимости
Итерация 3: { const x = 3; console.log(x); }  ← ЕЩЁ ДРУГАЯ область</pre>
<p>Каждая итерация — как отдельный блок `{ }` с новым `const`. Переменная не "меняется" — каждый раз создаётся заново.</p>
</details>

**🔍 Мини-проверка:** что выведет этот код? Подумай до запуска.

```js
const word = "Code";
let result = "";
for (const char of word) {
  result = char + result;
}
console.log(result);
```

<details>
<summary>Ответ</summary>
<p>Выведет `"edoC"`. Пошагово:</p>
<pre>word = "Code", result = ""
char = "C": result = "C" + ""    = "C"
char = "o": result = "o" + "C"   = "oC"
char = "d": result = "d" + "oC"  = "doC"
char = "e": result = "e" + "doC" = "edoC"</pre>
<p>Ключ: `result = char + result` — новый символ ставится ПЕРЕД старым результатом, поэтому строка переворачивается.</p>
</details>

## for...in — перебор ключей объекта

### Откуда взялось и зачем

`for...in` существует с первых версий JavaScript. Он перебирает **ключи** (имена свойств) объекта. Нужен для динамического перебора свойств когда ключи неизвестны заранее.

### Что это такое

`for...in` проходит по всем **перечисляемым** свойствам объекта, включая унаследованные из прототипа (прототипы разберём позже в продвинутых темах).

**Забегая вперёд:** объекты `{ key: value }` — структуры данных с именованными свойствами — подробно в главе 7. Здесь показан `for...in` для полноты картины по циклам. Если пока непонятно — вернись сюда после главы 7.

### Синтаксис

```js
for (const key in object) {
  // key — имя свойства (строка)
  // object[key] — значение свойства
}
```

### Пример

```js
const user = {
  name: "Игорь",
  age: 25,
  city: "Москва",
};

for (const key in user) {
  console.log(`${key}: ${user[key]}`);
}
// "name: Игорь"
// "age: 25"
// "city: Москва"
```

<details>
<summary>🔧 Механика: for...in пошагово</summary>
<pre>user = { name: "Игорь", age: 25, city: "Москва" }

Итерация 1: key = "name"  → user["name"]  = "Игорь"  → log("name: Игорь")
Итерация 2: key = "age"   → user["age"]   = 25       → log("age: 25")
Итерация 3: key = "city"  → user["city"]  = "Москва" → log("city: Москва")
Свойств больше нет → выход</pre>
<p>`key` — всегда строка (имя свойства). `user[key]` — доступ к значению по ключу (скобочная нотация, потому что `user.key` искало бы свойство с буквальным именем "key").</p>
</details>

<!-- добавлено: 2026-03-09 -->
> **Точка vs скобки: почему `user[key]`, а не `user.key`?**
>
> Точка — это **буквальное имя**. Скобки — **подставляют значение переменной**.
>
> ```js
> const user = { name: "Игорь", age: 25 };
> const key = "name";
>
> user.name   // "Игорь" — ищет свойство с именем name
> user["name"] // "Игорь" — то же самое, строка "name"
> user[key]   // "Игорь" — переменная key = "name", подставляется → user["name"]
> user.key    // undefined — ищет свойство с именем key, а такого нет!
> ```
>
> **Аналогия:** объект — шкафчик с ящиками. Точка — ты **говоришь** название ящика: `user.name` = "открой ящик name". Скобки — ты даёшь **бумажку** с названием: `user[key]` = "прочитай бумажку key, там написано name, открой ящик name".
>
> Поэтому в `for...in` используют скобки: на каждой итерации `key` содержит **разное имя свойства**, и нужно подставить его значение.

### Плохой пример → Хороший пример

```js
const arr = ["a", "b", "c"];

// ❌ Плохо: for...in для массива
for (const key in arr) {
  console.log(key);    // "0", "1", "2" — индексы как СТРОКИ!
  console.log(arr[key]); // "a", "b", "c" — работает, но неправильный инструмент
}
// Проблемы: ключи — строки, порядок не гарантирован, унаследованные свойства могут попасть

// ✅ Хорошо: for...of для массива
for (const item of arr) {
  console.log(item); // "a", "b", "c"
}
```

<details>
<summary>🔧 Механика: for...in vs for...of на массиве</summary>
<pre>arr = ["a", "b", "c"]

for...in (перебирает КЛЮЧИ):
  key = "0" (строка!) → arr["0"] = "a"
  key = "1" (строка!) → arr["1"] = "b"
  key = "2" (строка!) → arr["2"] = "c"
  ← ключи — строки, не числа!

for...of (перебирает ЗНАЧЕНИЯ):
  item = "a"
  item = "b"
  item = "c"
  ← сразу значения, без промежуточных шагов</pre>
<p>Массив — это объект, где индексы 0, 1, 2 — это ключи (свойства). `for...in` выдаёт их как строки `"0"`, `"1"`, `"2"`. `for...of` выдаёт сами значения.</p>
</details>

**⚠️ Важно:** **никогда не используй `for...in` для массивов.** Он перебирает ключи (индексы как строки), порядок не гарантирован, и могут попасть свойства из прототипа. Для массивов — `for...of` или `for`.

<details>
<summary>❌ Частые заблуждения</summary>
- "`for...in` и `for...of` — одно и то же" — нет. `for...in` перебирает **ключи** объекта. `for...of` перебирает **значения** итерируемой коллекции. Это разные инструменты для разных задач.
- "`for...in` перебирает только собственные свойства" — нет, он также перебирает **унаследованные** перечисляемые свойства. Для проверки используют `object.hasOwnProperty(key)`.
</details>

## break и continue — управление циклом

### Откуда взялось и зачем

Иногда цикл нужно прервать досрочно (нашли что искали) или пропустить одну итерацию (элемент не подходит). Для этого существуют два ключевых слова: `break` и `continue`.

### break — выйти из цикла досрочно

`break` немедленно прекращает выполнение цикла. Код после цикла продолжит работать.

```js
// Найти первый чётный элемент и остановиться:
const numbers = [1, 3, 4, 7, 8, 10];

for (const n of numbers) {
  if (n % 2 === 0) {
    console.log(`Нашли: ${n}`); // "Нашли: 4"
    break; // выходим из цикла — 7, 8, 10 уже не проверяем
  }
}
```

<details>
<summary>🔧 Механика: что именно пропускает break?</summary>
<p>В цикле `for (let i = 0; i < 10; i++)` при `break`:</p>
<pre>Шаг: условие → тело → ... → встретили break
                                   ↓
                             НЕМЕДЛЕННЫЙ выход
                             i++ НЕ выполняется
                             условие НЕ проверяется
                             → переход к коду ПОСЛЕ цикла</pre>
<p>Пример:</p>
<pre>for (let i = 0; i < 5; i++) {
  if (i === 2) break;
  console.log(i);
}
// i=0: 0<5? да → 2===0? нет → log(0) → i++ → i=1
// i=1: 1<5? да → 2===1? нет → log(1) → i++ → i=2
// i=2: 2<5? да → 2===2? ДА → break → ВЫХОД (i++ не выполнится)
// Вывод: 0, 1</pre>
</details>

### continue — пропустить текущую итерацию

`continue` прекращает текущую итерацию и переходит к следующей.

<details>
<summary>🔧 Механика: continue в for — выполняется ли i++?</summary>
<p>**Да!** В отличие от `break`, `continue` НЕ пропускает шаг (`i++`). Вот что происходит:</p>
<pre>Шаг: условие → тело → ... → встретили continue
                                   ↓
                             остаток тела ПРОПУЩЕН
                             но i++ ВЫПОЛНЯЕТСЯ ← важно!
                             → обратно к проверке условия</pre>
<p>Если бы `continue` пропускал `i++`, получился бы бесконечный цикл — `i` никогда бы не менялось.</p>
<p>**В `while` — другое дело!** В `while` нет встроенного шага, поэтому `continue` просто возвращается к условию. Если счётчик увеличивается в теле после `continue` — он будет пропущен:</p>
<pre>let i = 0;
while (i < 5) {
  if (i === 2) continue; // ❌ БЕСКОНЕЧНЫЙ ЦИКЛ: i++ ниже не выполнится!
  console.log(i);
  i++;
}</pre>
</details>

```js
// Вывести только нечётные числа:
for (let i = 1; i <= 10; i++) {
  if (i % 2 === 0) continue; // пропускаем чётные — переход к i++, потом к проверке
  console.log(i);
}
// 1, 3, 5, 7, 9
```

### Плохой пример → Хороший пример

```js
// ❌ Плохо: вложенный if вместо continue
for (let i = 0; i < 10; i++) {
  if (i % 2 !== 0) {
    if (i % 3 !== 0) {
      console.log(i); // числа не делящиеся ни на 2, ни на 3
    }
  }
}

// ✅ Хорошо: continue убирает вложенность
for (let i = 0; i < 10; i++) {
  if (i % 2 === 0) continue; // пропускаем чётные
  if (i % 3 === 0) continue; // пропускаем кратные 3
  console.log(i); // 1, 5, 7
}
```

<details>
<summary>🔧 Механика: continue как фильтр</summary>
<pre>i=0: 0%2===0? да → continue (пропущен)
i=1: 1%2===0? нет → 1%3===0? нет → log(1) ✓
i=2: 2%2===0? да → continue (пропущен)
i=3: 3%2===0? нет → 3%3===0? да → continue (пропущен)
i=4: 4%2===0? да → continue (пропущен)
i=5: 5%2===0? нет → 5%3===0? нет → log(5) ✓
i=6: 6%2===0? да → continue (пропущен)
i=7: 7%2===0? нет → 7%3===0? нет → log(7) ✓
i=8: 8%2===0? да → continue (пропущен)
i=9: 9%2===0? нет → 9%3===0? да → continue (пропущен)</pre>
<p>`continue` работает как фильтр: неподходящие значения отсеиваются на ранних проверках, до `console.log` доходят только прошедшие все проверки.</p>
</details>

**⚠️ Важно:** `break` и `continue` работают только с ближайшим циклом. Во вложенных циклах `break` выйдет только из внутреннего цикла, а не из внешнего.

### Вложенные циклы и break

```js
// break выходит только из внутреннего цикла:
for (let i = 0; i < 3; i++) {
  for (let j = 0; j < 3; j++) {
    if (j === 1) break; // выход только из внутреннего for
    console.log(i, j);
  }
}
// 0 0
// 1 0
// 2 0
// (j=1 и j=2 пропущены для каждого i, но внешний цикл продолжается)
```

<details>
<summary>🔧 Механика: почему break выходит только из ближайшего цикла?</summary>
<p>Пошаговая трассировка:</p>
<pre>i=0: начинаем внутренний цикл
  j=0: j===1? нет → log(0, 0) → j++
  j=1: j===1? ДА → break → выход из ВНУТРЕННЕГО цикла
  → возвращаемся во ВНЕШНИЙ цикл → i++ → i=1
i=1: начинаем внутренний цикл ЗАНОВО (j снова с 0)
  j=0: j===1? нет → log(1, 0) → j++
  j=1: j===1? ДА → break → выход из внутреннего
  → возвращаемся во внешний → i++ → i=2
i=2: то же самое → log(2, 0)
i=3: 3<3? нет → внешний цикл завершён</pre>
<p>`break` "видит" только тот цикл, в котором он находится. Для внешнего цикла `break` во внутреннем — просто конец внутреннего цикла, ничего особенного.</p>
</details>

**🔍 Мини-проверка:** что выведет этот код? Подумай до запуска.

```js
let sum = 0;
for (let i = 1; i <= 10; i++) {
  if (i === 6) break;
  if (i % 2 === 0) continue;
  sum += i;
}
console.log(sum);
```

<details>
<summary>Ответ</summary>
<p>Выведет `9`. Разберём по итерациям:</p>
<pre>i=1: не break, не чётное → sum=1
i=2: не break, чётное → continue
i=3: не break, не чётное → sum=4
i=4: не break, чётное → continue
i=5: не break, не чётное → sum=9
i=6: break → выход. Итого: 1+3+5 = 9</pre>
</details>

## Вложенные циклы

### Что это такое

Цикл внутри цикла. Внутренний цикл полностью выполняется на каждой итерации внешнего. Общее количество итераций — произведение итераций внешнего и внутреннего циклов.

### Пример: таблица умножения

```js
for (let i = 1; i <= 3; i++) {
  for (let j = 1; j <= 3; j++) {
    console.log(`${i} × ${j} = ${i * j}`);
  }
}
// 1 × 1 = 1
// 1 × 2 = 2
// 1 × 3 = 3
// 2 × 1 = 2
// 2 × 2 = 4
// ...
// 3 × 3 = 9
```

```
Итерации:
Внешний i=1: внутренний j=1,2,3  (3 итерации)
Внешний i=2: внутренний j=1,2,3  (3 итерации)
Внешний i=3: внутренний j=1,2,3  (3 итерации)
                          Всего: 3 × 3 = 9 итераций
```

<details>
<summary>🔧 Механика: полная трассировка вложенного цикла</summary>
<pre>i=1, j=1: log("1 × 1 = 1") → j++ → j=2
i=1, j=2: log("1 × 2 = 2") → j++ → j=3
i=1, j=3: log("1 × 3 = 3") → j++ → j=4 → 4<=3? нет → внутренний цикл закончен
  → i++ → i=2, внутренний цикл начинается заново с j=1
i=2, j=1: log("2 × 1 = 2") → j=2
i=2, j=2: log("2 × 2 = 4") → j=3
i=2, j=3: log("2 × 3 = 6") → j=4 → выход из внутреннего
  → i++ → i=3
i=3, j=1: log("3 × 1 = 3") → j=2
i=3, j=2: log("3 × 2 = 6") → j=3
i=3, j=3: log("3 × 3 = 9") → j=4 → выход из внутреннего
  → i++ → i=4 → 4<=3? нет → выход из внешнего</pre>
<p>Ключевой момент: при каждом новом `i` переменная `j` создаётся заново с начального значения 1.</p>
</details>

**⚠️ Важно:** вложенные циклы быстро увеличивают количество операций. Цикл 1000 × 1000 = миллион операций. Три вложенных цикла по 1000 = миллиард. Избегай глубокой вложенности без необходимости.

## Когда какой цикл использовать?

| Ситуация | Лучший выбор |
|----------|--------------|
| Перебор массива, нужен индекс | `for` |
| Перебор массива, индекс не нужен | `for...of` |
| Перебор ключей объекта | `for...in` |
| Неизвестное кол-во итераций | `while` |
| Нужно выполнить хотя бы раз | `do...while` |

**Забегая вперёд:** для массивов также есть методы `.forEach()`, `.map()`, `.filter()` и другие (гл. 6). Они часто удобнее циклов для конкретных задач: трансформация, фильтрация, поиск. Но для понимания циклов нужно знать `for` и `for...of` — они фундаментальнее.

## Итог

```
ЦИКЛЫ:

for (let i = 0; i < n; i++) {}    → счётчик, нужен индекс
while (условие) {}                 → пока условие true
do {} while (условие)              → хотя бы одна итерация
for (const item of массив) {}     → перебор значений (итерируемое)
for (const key in объект) {}      → перебор ключей объекта

УПРАВЛЕНИЕ:
break     → выйти из цикла досрочно
continue  → пропустить текущую итерацию, перейти к следующей

КОГДА ЧТО:
Знаешь количество / нужен индекс  → for
Не знаешь количество               → while
Просто перебрать значения           → for...of
Перебрать ключи объекта             → for...in
```

## Попробуй сам

**1. Предскажи вывод** (не запуская):
```js
for (let i = 5; i >= 1; i--) {
  if (i === 3) continue;
  console.log(i);
}
```

<details>
<summary>Ответ</summary>
<pre>5, 4, 2, 1 (тройка пропущена через continue)</pre>
</details>

**2. Что выведет этот код?**
```js
const arr = [10, 20, 30];
for (let i = 0; i <= arr.length; i++) {
  console.log(arr[i]);
}
```

<details>
<summary>Ответ</summary>
<pre>10, 20, 30, undefined
Последняя итерация: i=3, arr[3] не существует → undefined.
Исправление: i < arr.length (без =)</pre>
</details>

**3. Напиши цикл** который соберёт в строку все символы слова "JavaScript" кроме гласных (a, i):
```js
let result = "";
// твой код — используй for...of и continue
console.log(result); // "JvScrpt"
```

<details>
<summary>Ответ</summary>

```js
let result = "";
for (const char of "JavaScript") {
  if (char === "a" || char === "i") continue;
  result += char;
}
console.log(result); // "JvScrpt"
```
</details>
