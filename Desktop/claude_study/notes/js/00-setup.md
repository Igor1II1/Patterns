# Запуск кода и вывод в консоль

> [[js-fundamentals|← Назад к оглавлению JS]]
> Читай эту главу **перед** любым практическим заданием.

## Зачем это знать?

Прежде чем писать JS — нужно понять как его **запустить** и как **увидеть результат**. Без этого задания превращаются в написание кода в пустоту.

---

## Два способа запустить JS

### Способ 1: Node.js (терминал) — используем в учёбе

**Что такое Node.js:** это среда выполнения JS вне браузера. Позволяет запускать JS-файлы прямо из терминала — как обычные программы.

**Как запустить:**

```bash
node exercises/block-0/01-variables/script.js
```

Или если уже находишься в папке с файлом:

```bash
node script.js
```

**Результат:** всё что вывел `console.log` — появится в терминале.

> Именно этот способ используем для всех практических заданий в учёбе.

---

### Способ 2: Браузер (DevTools) — для HTML + JS

**Когда нужен:** когда JS подключён к HTML-странице, или когда хочешь быстро что-то проверить прямо в браузере.

**Как открыть консоль браузера:**
- Windows/Linux: `F12` или `Ctrl + Shift + J`
- Mac: `Cmd + Option + J`

**Как подключить JS к HTML:**

```html
<!-- index.html -->
<!DOCTYPE html>
<html>
  <body>
    <script src="script.js"></script>
  </body>
</html>
```

Открой `index.html` в браузере → нажми `F12` → вкладка **Console** → видишь результат.

## console.log — вывод в консоль

### Что это?

`console.log()` — встроенная функция JS для вывода значений в консоль. Это главный инструмент разработчика для проверки того что происходит в коде.

### Зачем нужен?

Программа выполняется "невидимо". Ты пишешь `let x = 5 + 3` — но не видишь результата. `console.log` даёт возможность "подглядеть" что происходит внутри.

```js
let x = 5 + 3;
console.log(x); // 8 — теперь видим результат в консоли
```

### Как использовать?

```js
// Вывести одно значение:
console.log("Привет");        // Привет
console.log(42);              // 42
console.log(true);            // true

// Вывести несколько значений сразу:
console.log("Имя:", "Игорь"); // Имя: Игорь
console.log(1, 2, 3);        // 1 2 3

// Вывести переменную:
let name = "Игорь";
console.log(name);            // Игорь

// Вывести выражение:
console.log(2 + 2);           // 4
console.log(5 > 3);           // true

// Вывести строку с переменной (интерполяция):
let age = 25;
console.log(`Мне ${age} лет`); // Мне 25 лет

// Вывести объект или массив:
let user = { name: "Игорь", age: 25 };
console.log(user);            // { name: 'Игорь', age: 25 }
```

### console.log — не единственный

| Метод | Что делает | Когда использовать |
|-------|-----------|-------------------|
| `console.log()` | Обычный вывод | Всегда — основной инструмент |
| `console.error()` | Вывод ошибки (красный) | Когда произошла ошибка |
| `console.warn()` | Предупреждение (жёлтый) | Когда что-то подозрительно |
| `console.table()` | Таблица для массивов/объектов | Когда нужно красиво вывести данные |

```js
console.log("Обычное сообщение");
console.error("Что-то пошло не так!");
console.warn("Это подозрительно");
console.table([{ name: "Игорь", age: 25 }, { name: "Иван", age: 30 }]);
```

## Типичный рабочий процесс

```
1. Пишешь код в script.js
2. Добавляешь console.log() чтобы увидеть результат
3. Запускаешь: node script.js
4. Видишь результат в терминале
5. Исправляешь → запускаешь снова
```

---

## Пример полного цикла

```js
// script.js
const firstName = "Игорь";
const age = 27;

console.log(firstName);                          // Игорь
console.log(age);                                // 27
console.log(`Меня зовут ${firstName}, мне ${age} лет`); // Меня зовут Игорь, мне 27 лет
```

Запускаешь в терминале:
```bash
node script.js
```

Видишь в терминале:
```
Игорь
27
Меня зовут Игорь, мне 27 лет
```
