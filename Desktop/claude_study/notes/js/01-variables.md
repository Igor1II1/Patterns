## Что такое переменная?

**Переменная** — это именованное место в памяти компьютера, в котором хранится значение.

Представь оперативную память как огромный склад с ячейками. Каждая ячейка имеет адрес — например `0x4F2A`. Программа постоянно работает с этими ячейками: кладёт туда данные, достаёт их. Но запомнить адрес `0x4F2A` невозможно. Поэтому придумали переменные — **человеческие имена для ячеек памяти**.

```js
let userName = "Игорь";
//  ↑ имя       ↑ значение
```

Теперь вместо адреса `0x4F2A` ты пишешь `userName` — и получаешь `"Игорь"`.

**Зачем нужны переменные:**
- Хранить данные между операциями
- Давать данным понятные имена (код становится читаемым)
- Переиспользовать одно значение в разных местах программы
- Изменять значение в одном месте и видеть изменение везде

```js
// Без переменной — непонятно что значит 0.2
let result = 100 * 0.2;  // * это умножение (подробнее — в разделе "Операторы" ниже)

// С переменной — сразу ясно что 0.2 это ставка налога
const taxRate = 0.2;
let total = 100 * taxRate;
```

## Как устроена память компьютера (для понимания переменных)

Чтобы понять переменные, нужно сначала понять где они живут — в **оперативной памяти (RAM)**.

### Два вида памяти

У компьютера есть два основных хранилища данных:

| | Жёсткий диск (HDD/SSD) | Оперативная память (RAM) |
|---|---|---|
| **Аналогия** | Шкаф с папками | Рабочий стол |
| **Скорость** | Медленный | Очень быстрый |
| **Объём** | Большой (500 ГБ — 2 ТБ) | Маленький (8—32 ГБ) |
| **При выключении** | Данные сохраняются | Данные стираются |
| **Что хранит** | Файлы, программы, фото | То, с чем работает прямо сейчас |

### Зачем нужна оперативная память?

Процессор работает очень быстро. Если бы он каждый раз лез за данными на жёсткий диск — он бы постоянно ждал. Оперативная память решает эту проблему: нужные данные загружаются туда заранее, и процессор берёт их мгновенно.

**Ключевое свойство:** оперативная память **временная**. Выключил компьютер — всё из неё стёрлось. Поэтому файлы хранятся на диске, а в RAM попадают только когда ты их открываешь.

### Как это связано с переменными?

Когда ты пишешь `let name = "Игорь"` — JavaScript просит у компьютера кусочек оперативной памяти, кладёт туда строку `"Игорь"` и запоминает, что этот кусочек называется `name`.

```
Оперативная память:
┌─────────┬───────────┐
│  name   │  "Игорь"  │  ← переменная занимает место в RAM
├─────────┼───────────┤
│  age    │  25       │
├─────────┼───────────┤
│  ...    │  ...      │
└─────────┴───────────┘
```

Переменная — это по сути **именованная ячейка в оперативной памяти**.

### Как переменная находит своё значение?

Переменная хранит **не само значение напрямую**, а работает через специальную таблицу — **scope** (область видимости). Это как картотека в библиотеке:

```
Таблица scope (область видимости):
┌────────────┬──────────────────┐
│  Имя       │  Адрес в памяти  │
├────────────┼──────────────────┤
│  userName  │  → адрес 0x4F2A  │  → там лежит "Игорь"
│  age       │  → адрес 0x4F3B  │  → там лежит 25
└────────────┴──────────────────┘
```

Когда ты пишешь `console.log(userName)`:
1. Движок JS ищет `"userName"` в таблице текущего scope
2. Находит адрес в памяти (`0x4F2A`)
3. Читает значение по этому адресу → `"Игорь"`

Когда делаешь `userName = "Олег"`:
1. Находит адрес `"userName"` в таблице
2. Записывает новое значение `"Олег"` по адресу (или выделяет новый адрес и обновляет таблицу)

Ты напрямую с адресами не работаешь — JS всё скрывает за именами. В некоторых языках (C, C++) можно работать с адресами напрямую, но в JS такой необходимости нет.

### А если в переменной хранится много значений?

Простые значения (`25`, `"Игорь"`, `true`) хранятся прямо в ячейке. Но что если нужно хранить список из 100 элементов? Не создавать же 100 переменных.

Для этого существуют **составные структуры** — массивы и объекты (главы 6–7). Переменная тогда хранит **не сами данные, а адрес контейнера** в памяти:

```
Таблица scope:
  "fruits" → адрес 1000

Память:
  адрес 1000: [массив, длина 3]
                 0 → адрес 2001
                 1 → адрес 2002
                 2 → адрес 2003

  адрес 2001: "яблоко"
  адрес 2002: "банан"
  адрес 2003: "груша"
```

Аналогия: переменная `fruits` — это **адрес шкафа**. Шкаф один, но в нём 3 ящика (0, 1, 2). В каждом ящике лежит своё значение. Ты знаешь адрес шкафа (имя переменной) и номер ящика (индекс) — этого достаточно чтобы найти любое значение:

- `fruits` — адрес шкафа
- `fruits[0]` — открой ящик 0 в этом шкафу
- `fruits.length` — сколько ящиков в шкафу

> **Забегая вперёд:** массивы и объекты подробно разбираются в главах 6–7. Здесь важно понять принцип: одна переменная может хранить адрес структуры с множеством значений.

Когда программа завершается — все её переменные удаляются из RAM. Именно поэтому, если ты хочешь сохранить данные между запусками программы, нужны файлы, базы данных или `localStorage` (про это будет позже).

## Объявление переменной: var, let, const

**Объявление** — это момент когда ты говоришь JavaScript: "создай переменную с таким именем".

В JavaScript три ключевых слова для объявления:

```js
var   name = "старый способ";
let   age  = 25;
const PI   = 3.14;
```

---

## var — устаревший способ (но знать обязательно)

### Что такое var?

`var` — это первый и долгое время единственный способ объявить переменную в JavaScript (до 2015 года). Сейчас он считается устаревшим и не используется в новом коде.

### Зачем знать var?

- На собеседованиях спрашивают разницу между `var`, `let`, `const`
- В старом коде и библиотеках он встречается повсеместно
- Понимание проблем `var` объясняет почему появились `let` и `const`

### Как работает var?

```js
var message = "Привет";
console.log(message); // "Привет"

var message = "Хай"; // можно объявить повторно — нет ошибки!
console.log(message); // "Хай"
```

### Проблема 1: Функциональная область видимости

> **Забегая вперёд:** здесь используются `function` (глава 4) и `if` (глава 2). Пока не нужно понимать их синтаксис — просто смотри на поведение переменной `x`. `function example() { ... }` — это блок кода с именем, а `if (true) { ... }` — код который выполняется при условии (здесь условие всегда true).

`var` не признаёт блоки `{}` (if, for, while) как границы своей видимости. Он живёт во **всей функции** где объявлен.

```js
function example() {
  if (true) {
    var x = 10; // думаешь что x живёт только в if
  }
  console.log(x); // 10 — var "вытек" за пределы if!
}
```

Это неинтуитивно и ведёт к багам — переменная живёт дольше чем ожидалось.

### Проблема 2: Hoisting (поднятие) с undefined

JavaScript перед выполнением кода "поднимает" объявление `var` в начало функции, но без значения:

```js
console.log(name); // undefined — не ошибка! Переменная "есть", но пустая
var name = "Игорь";
console.log(name); // "Игорь"
```

JavaScript как будто переписал твой код так:

```js
var name;           // поднял объявление наверх
console.log(name);  // undefined
name = "Игорь";     // присвоение осталось на месте
console.log(name);  // "Игорь"
```

Это скрывает ошибки — ты используешь переменную до того как задал ей значение, и не получаешь никакого предупреждения.

> **Механика: как это работает внутри**
>
> JS выполняет код в **две фазы**, а не в одну:
>
> **Фаза 1 — разбор (parsing).** Движок пробегает весь код и регистрирует объявления:
> - `var name` -- регистрирует переменную `name`, сразу присваивает ей `undefined`
> - `let age` -- регистрирует переменную `age`, но **не инициализирует** (помечает как "нельзя трогать")
> - `function foo()` -- регистрирует функцию **целиком**, с телом
>
> **Фаза 2 — выполнение (execution).** Код запускается строка за строкой.
>
> ```
> Исходный код:           Фаза 1 (разбор):         Фаза 2 (выполнение):
> ─────────────           ─────────────────         ────────────────────
> console.log(x);         var x = undefined         console.log(x) → undefined
> var x = 5;              (let y зарегистр.)        x = 5
> console.log(y);                                   console.log(y) → ReferenceError!
> let y = 10;                                       (до этой строки не дойдёт)
> ```
>
> Вот почему `var` дает `undefined`, а `let` выбрасывает ошибку -- разная инициализация в фазе 1.

### Hoisting функций — забегая вперёд

> **Что такое функция:** именованный блок кода который можно вызвать по имени. Подробно изучается в **главе 4**. Пока знай только два вида записи:
> ```js
> function sayHi() { ... }        // function declaration — объявление функции
> const sayBye = function() { ... } // function expression — функция как значение переменной
> ```

При hoisting они ведут себя по-разному:

**`function declaration`** — поднимается **полностью**, включая тело. Можно вызвать до объявления:

```js
sayHi(); // ✅ "Привет" — работает!
function sayHi() { console.log("Привет"); }
```

**`function expression`** — это просто переменная (`const`/`let`). Ведёт себя как обычная переменная при hoisting:

```js
sayBye(); // ❌ ReferenceError — const в TDZ
const sayBye = function() { console.log("Пока"); };
```

**Сравнение:**

| | Поднимается? | Можно вызвать до объявления? |
|---|---|---|
| `function declaration` | ✅ полностью | ✅ да |
| `function expression` (const/let) | ✅ да (TDZ) | ❌ нет |
| `var` с функцией | ✅ (undefined) | ❌ ошибка при вызове |

**🔍 Мини-проверка:** что выведет этот код? Подумай до запуска.

```js
console.log(a);
var a = 10;
console.log(a);
```

<details>
<summary>Ответ</summary>
<p>Выведет `undefined`, потом `10`.</p>
<pre>Фаза 1 (разбор): var a = undefined (поднял наверх)
Фаза 2 (выполнение):
  console.log(a) → undefined (переменная есть, но значение ещё не задано)
  a = 10
  console.log(a) → 10</pre>
</details>

## let — современная изменяемая переменная

### Что такое let?

`let` — ключевое слово для объявления переменной, значение которой **можно изменить** в будущем.

### Зачем нужен let?

Когда значение должно меняться по ходу программы: счётчики, текущее состояние, результат вычислений которые обновляются.

```js
let score = 0;
score = 10;   // положили новое значение
score = 20;   // заменили ещё раз — let это позволяет
console.log(score); // 20
```

### Как работает let?

```js
let age = 25;
age = 26; // можно изменить — это нормально
console.log(age); // 26
```

### Блочная область видимости

`let` живёт только в том блоке `{}` где объявлен. **Блок** — это любые фигурные скобки.

```js
// Просто блок — фигурные скобки сами по себе
{
  let x = 10; // x живёт только здесь
  console.log(x); // 10 ✅
}
console.log(x); // ❌ ReferenceError: x is not defined
```

> **Забегая вперёд:** `if` (условие) подробно разбирается в главе 2. Пока просто знай: `if (true) { ... }` — код внутри `{}` выполнится. Здесь важно увидеть что `let` не "вытекает" наружу.

```js
if (true) {
  let message = "Привет";
  console.log(message); // "Привет" ✅
}
console.log(message); // ❌ ReferenceError — вышли из блока
```

### Hoisting у let — Temporal Dead Zone (TDZ)

`let` тоже поднимается в начало блока, но **не инициализируется**. Промежуток от начала блока до строки объявления называется **Temporal Dead Zone (Зона временной мёртвости)**.

```js
console.log(name); // ❌ ReferenceError: Cannot access 'name' before initialization
let name = "Игорь";
```

**Почему это хорошо?** Ты сразу получаешь ошибку вместо скрытого бага с `undefined`. JavaScript говорит: "ты обратился к переменной до её создания".

> **Механика: как это работает внутри**
>
> Разберём `let x = 5` пошагово:
>
> ```
> {                         // начало блока
>   // -------- TDZ для x начинается здесь --------
>   console.log(x);        // Шаг 1: движок ищет x
>                           //   -> x найден (зарегистрирован в фазе разбора)
>                           //   -> но статус: "uninitialized" (не инициализирован)
>                           //   -> ОШИБКА: ReferenceError
>   // -------- TDZ для x заканчивается здесь ------
>   let x = 5;             // Шаг 2: статус x меняется на "initialized", значение = 5
>   console.log(x);        // Шаг 3: x инициализирован -> 5 ✅
> }
> ```
>
> Ключевой момент: переменная **существует** с начала блока (движок про неё знает),
> но **доступ запрещён** до строки `let x = ...`. Это и есть TDZ.

**🔍 Мини-проверка:** что выведет этот код? Подумай до запуска.

```js
let x = 5;
{
  let x = 20;
  console.log(x);
}
console.log(x);
```

<details>
<summary>Ответ</summary>
<p>Выведет `20`, потом `5`.</p>
<pre>Внешний x = 5
Блок { } создаёт НОВУЮ область видимости
  Внутренний x = 20 (это ДРУГАЯ переменная, не внешняя)
  console.log(x) → 20 (берёт ближайший x — внутренний)
Выходим из блока — внутренний x уничтожен
console.log(x) → 5 (внешний x не менялся)</pre>
</details>

## const — постоянная (нельзя переназначить)

### Что такое const?

`const` — ключевое слово для объявления **константы**: переменной которую нельзя переназначить после создания.

### Зачем нужен const?

- Защитить значение от случайного изменения
- Сделать код понятнее: читая `const`, разработчик сразу знает — это значение не изменится
- Упростить отладку — меньше мест где данные могут поменяться

```js
const MAX_USERS = 100;     // лимит пользователей — не должен меняться
const API_URL = "/api/v1"; // адрес API — фиксированный
const taxRate = 0.2;       // ставка налога
```

### Как работает const?

```js
const PI = 3.14159;
PI = 3; // ❌ TypeError: Assignment to constant variable
```

`const` запрещает **переназначение** — нельзя написать `const_переменная = новое_значение`.

### Важнейший нюанс: const и объекты

`const` запрещает переназначение переменной, но **не замораживает содержимое** объекта или массива.

```js
const user = { name: "Игорь", age: 25 };

// ✅ Изменить свойство — можно
user.name = "Иван";
user.age = 26;
console.log(user); // { name: "Иван", age: 26 }

// ✅ Добавить свойство — можно
user.role = "admin";

// ✅ Удалить свойство — можно
delete user.age;

// ❌ Переназначить переменную — нельзя
user = { name: "Борис" }; // TypeError
```

**Аналогия:** `const` — это постоянный ярлык наклеенный на коробку. Ярлык не переклеить на другую коробку. Но содержимое коробки можно менять как угодно.

> **Механика: как это работает внутри**
>
> Переменная хранит не сам объект, а **адрес** (ссылку) на него в памяти.
> `const` запрещает менять адрес, но не содержимое по этому адресу.
>
> ```
> const user = { name: "Игорь" };
>
> Память:
>   user --> [адрес 0x1] --> { name: "Игорь" }
>
> user.name = "Иван";          // меняем содержимое по адресу 0x1
>   user --> [адрес 0x1] --> { name: "Иван" }   ✅ адрес не менялся
>
> user = { name: "Борис" };    // пытаемся сменить адрес на 0x2
>   user --> [адрес 0x2]  ???   ❌ TypeError! const запрещает менять адрес
> ```
>
> Итого: `const` фиксирует **стрелку** (куда указывает переменная), а не **коробку** (что лежит по адресу).

**🔍 Мини-проверка:** какие строки вызовут ошибку?

```js
const user = { name: "Игорь", age: 25 };
user.name = "Олег";        // 1
user.city = "Москва";      // 2
user = { name: "Борис" };  // 3
```

<details>
<summary>Ответ</summary>
<p>Ошибку вызовет только строка 3 (`TypeError: Assignment to constant variable`).</p>
<p>Строки 1 и 2 работают нормально — const запрещает переназначение переменной (сменить адрес), но НЕ запрещает менять содержимое объекта по этому адресу.</p>
</details>

### const с массивами

> **Забегая вперёд:** массивы подробно разбираются в главе 6. Пока знай: `[1, 2, 3]` — это список значений, а `.push(4)` — добавляет элемент в конец списка.

```js
const numbers = [1, 2, 3];

numbers.push(4);   // ✅ можно изменять содержимое массива
numbers[0] = 99;   // ✅ можно менять элементы

numbers = [4, 5];  // ❌ нельзя переназначить переменную
```

## Соглашения об именовании переменных

### Правила синтаксиса (обязательные)

Имена переменных в JS:
- Могут содержать: буквы, цифры, `_`, `$`
- Не могут начинаться с цифры
- Нельзя использовать зарезервированные слова (`let`, `const`, `if`, `return` и т.д.)
- JavaScript чувствителен к регистру: `userName` и `username` — разные переменные!

```js
let userName = "Игорь"; // ✅
let user_name = "Игорь"; // ✅ но не принято в JS
let 1user = "Игорь";    // ❌ нельзя начинать с цифры
let let = 5;             // ❌ let — зарезервированное слово
```

### Соглашения об именовании (общепринятые)

В JavaScript-сообществе договорились о единых стилях именования:

**1. camelCase** — для обычных переменных и функций (главный стиль JS)

```js
let firstName = "Игорь";      // ✅ первое слово строчное, остальные с большой
let userAge = 25;
let isLoggedIn = false;
let calculateTotal = ...;     // функция
```

**2. UPPER_SNAKE_CASE** — для констант с фиксированными значениями (конфигурация, "магические числа")

```js
const MAX_RETRY_COUNT = 3;    // ✅ понятно что это лимит, менять нельзя
const API_URL = "https://...";
const TAX_RATE = 0.2;
```

**3. PascalCase** — для классов и конструкторов (изучим позже в главе по ООП)

```js

```

## Операторы

### Что такое оператор?

**Оператор** — это символ (или набор символов) который выполняет действие над значениями. Значения, с которыми оператор работает, называются **операндами**.

```js
let result = 5 + 3;
//          ↑ ↑ ↑
//    операнд │ операнд
//        оператор
```

### Классификация операторов по числу операндов

Все операторы делятся по тому, сколько значений они принимают:

**Бинарный** (binary) — работает с **двумя** операндами. Большинство операторов бинарные:

```js
5 + 3        // два операнда: 5 и 3
a === b      // два операнда
a && b       // два операнда
```

**Унарный** (unary) — работает с **одним** операндом:

```js
-5           // минус перед одним числом — отрицание (не вычитание!)
!true        // ! перед одним значением → false
typeof x     // один операнд → возвращает тип ("string", "number"...)
x++          // один операнд → увеличить x на 1
```

Обрати внимание: `-` может быть и **бинарным** (вычитание) и **унарным** (отрицание) — зависит от контекста:

```js
10 - 3   // бинарный: два операнда, вычитание → 7
-5       // унарный: один операнд, отрицание → -5
```

**Тернарный** (ternary) — работает с **тремя** операндами. В JS он один:

```js
условие ? значение_если_true : значение_если_false

let age = 20;
let status = age >= 18 ? "взрослый" : "ребёнок";
//           ↑ операнд1   ↑ операнд2    ↑ операнд3
// Шаг 1: вычисляем условие → 20 >= 18 → true
// Шаг 2: условие true → берём значение слева от : → "взрослый"
// Результат: "взрослый"

let age2 = 12;
let status2 = age2 >= 18 ? "взрослый" : "ребёнок";
// Шаг 1: 12 >= 18 → false
// Шаг 2: условие false → берём значение справа от : → "ребёнок"
// Результат: "ребёнок"
```

**Формула-якорь:** унарный = 1, бинарный = 2, тернарный = 3 операнда.

**🔍 Мини-проверка:** определи тип каждого оператора (унарный, бинарный, тернарный):

```js
typeof "hello"       // ?
10 - 3               // ?
!false               // ?
x > 5 ? "да" : "нет" // ?
```

<details>
<summary>Ответ</summary>
<pre>typeof "hello"        → унарный (1 операнд: "hello")
10 - 3                → бинарный (2 операнда: 10 и 3)
!false                → унарный (1 операнд: false)
x > 5 ? "да" : "нет" → тернарный (3 операнда: x>5, "да", "нет")</pre>
</details>

### Арифметические операторы

Работают с числами — как калькулятор:

| Оператор | Что делает | Пример | Результат |
|----------|-----------|--------|-----------|
| `+` | Сложение | `5 + 3` | `8` |
| `-` | Вычитание | `10 - 4` | `6` |
| `*` | Умножение | `3 * 4` | `12` |
| `/` | Деление | `10 / 3` | `3.333...` |
| `%` | Остаток от деления | `10 % 3` | `1` |
| `**` | Возведение в степень | `2 ** 3` | `8` |

```js
let price = 100;
let tax = price * 0.2;      // 100 * 0.2 = 20 — умножение
let total = price + tax;     // 100 + 20 = 120 — сложение
let perPerson = total / 3;   // 120 / 3 = 40 — деление
let remainder = 10 % 3;      // Шаг 1: 10 / 3 = 3 (целая часть) + остаток 1
                              // Шаг 2: % возвращает остаток → 1
```

**Остаток от деления `%`** — полезен чаще чем кажется. Например, проверить чётное ли число:

```js
7 % 2
// Шаг 1: 7 / 2 = 3 (целая часть) + остаток 1
// Шаг 2: % возвращает остаток → 1
// Вывод: нечётное (остаток есть)

8 % 2
// Шаг 1: 8 / 2 = 4 (целая часть) + остаток 0
// Шаг 2: % возвращает остаток → 0
// Вывод: чётное (делится без остатка)

2 ** 3
// Шаг 1: 2 в степени 3 = 2 * 2 * 2 = 8
// Результат: 8
```

### Приоритет операторов <!-- добавлено: 2026-03-04 -->

**Что это:** как и в математике, операторы выполняются не слева направо, а по приоритету. Оператор с высшим приоритетом выполняется первым.

```js
2 + 3 * 4
// Шаг 1: 3 * 4 = 12  (умножение — высший приоритет)
// Шаг 2: 2 + 12 = 14 (сложение — ниже)
// Результат: 14 (не 20!)
```

**Ключевые уровни приоритета (от высшего к низшему):**

| Приоритет | Операторы | Пример |
|-----------|----------|--------|
| Высший | `()` скобки | `(2 + 3) * 4 = 20` |
| Высокий | `**` степень | `2 ** 3 = 8` |
| Средне-высокий | `*`, `/`, `%` | `6 / 2 * 3 = 9` |
| Средний | `+`, `-` | `5 - 2 + 1 = 4` |
| Низкий | `>`, `<`, `>=`, `<=` | `5 > 3 = true` |
| Ниже | `===`, `!==`, `==`, `!=` | `5 === 5 = true` |
| Ещё ниже | `&&` | `true && false = false` |
| Ещё ниже | `\|\|`, `??` | `false \|\| true = true` |
| Самый низкий | `=`, `+=`, `-=` и т.д. | `x = 5` |

```js
// Примеры:
let result = 2 + 3 * 4;         // 14 (умножение раньше сложения)
let result2 = (2 + 3) * 4;      // 20 (скобки меняют порядок)
let check = 5 + 3 > 2 * 4;      // 8 > 8 → false
//  Шаг 1: 5 + 3 = 8
//  Шаг 2: 2 * 4 = 8
//  Шаг 3: 8 > 8 → false
```

**Главное правило:** если сомневаешься — **ставь скобки**. Скобки всегда выполняются первыми и делают код более читаемым.

### Оператор присваивания `=`

Знак `=` в программировании — это **не "равно"**, а **"положи значение в переменную"**.

```js
let x = 5;
// Шаг 1: создаём переменную x
// Шаг 2: кладём в неё 5
// x: [5]

x = x + 3;
// Шаг 1: берём текущее значение x → 5
// Шаг 2: 5 + 3 = 8
// Шаг 3: кладём 8 обратно в x
// x: [8]  (5 выброшено, теперь там 8)
```

Читай `=` как стрелку влево: `x ← 5`.

### Составные операторы присваивания

Чтобы не писать `x = x + 3` каждый раз, есть сокращённая запись:

| Полная запись | Сокращение | Что делает |
|--------------|-----------|-----------|
| `x = x + 5` | `x += 5` | Прибавить 5 к x |
| `x = x - 3` | `x -= 3` | Вычесть 3 из x |
| `x = x * 2` | `x *= 2` | Умножить x на 2 |
| `x = x / 4` | `x /= 4` | Разделить x на 4 |
| `x = x % 3` | `x %= 3` | Остаток от деления x на 3 |

```js
let score = 0;
score += 10;  // score = 0 + 10 = 10
score += 5;   // score = 10 + 5 = 15
score -= 3;   // score = 15 - 3 = 12
score *= 2;   // score = 12 * 2 = 24
console.log(score); // 24
```

### Инкремент и декремент

Ещё более короткая запись для "прибавить 1" и "вычесть 1":

```js
let count = 0;
count++;  // count = 0 + 1 = 1
count++;  // count = 1 + 1 = 2
count--;  // count = 2 - 1 = 1
console.log(count); // 1
```

**Префиксный vs постфиксный** — разница важна когда используешь значение прямо в выражении:

```js
let a = 5;
let b = a++;  // постфиксный: СНАЧАЛА вернули старое значение (5) в b, ПОТОМ увеличили a
// Шаг 1: b = a → b = 5
// Шаг 2: a = a + 1 → a = 6
console.log(a); // 6
console.log(b); // 5

let c = 5;
let d = ++c;  // префиксный: СНАЧАЛА увеличили c, ПОТОМ вернули новое значение в d
// Шаг 1: c = c + 1 → c = 6
// Шаг 2: d = c → d = 6
console.log(c); // 6
console.log(d); // 6
```

### Операторы сравнения

Сравнивают два значения и возвращают `true` или `false`:

| Оператор | Что делает | Пример | Результат |
|----------|-----------|--------|-----------|
| `===` | Строго равно (тип + значение) | `5 === 5` | `true` |
| `!==` | Строго не равно | `5 !== "5"` | `true` |
| `>` | Больше | `5 > 3` | `true` |
| `<` | Меньше | `3 < 5` | `true` |
| `>=` | Больше или равно | `5 >= 5` | `true` |
| `<=` | Меньше или равно | `3 <= 5` | `true` |

> Почему `===` а не `==`? Подробно разобрано ниже в разделе "Операторы сравнения: == и ===".

### Логические операторы

Работают с булевыми значениями (`true` / `false`):

| Оператор | Название | Что делает | Пример | Результат |
|----------|---------|-----------|--------|-----------|
| `&&` | И (AND) | `true` если **оба** true | `true && false` | `false` |
| `\|\|` | ИЛИ (OR) | `true` если **хотя бы один** true | `true \|\| false` | `true` |
| `!` | НЕ (NOT) | Меняет true на false и наоборот | `!true` | `false` |

```js
let age = 25;
let hasTicket = true;

// && — пошагово:
age >= 18 && hasTicket
// Шаг 1: вычисляем левую часть → 25 >= 18 → true
// Шаг 2: true, нужно проверить правую → hasTicket → true
// Шаг 3: обе части true → результат true

age >= 18 && false
// Шаг 1: 25 >= 18 → true
// Шаг 2: проверяем правую → false
// Шаг 3: нашли ложное → результат false

false && hasTicket    // false — до hasTicket даже не дошли!
// Шаг 1: false → ложное → СТОП (короткое замыкание, short-circuit)
// Шаг 2: правая часть НЕ вычисляется, сразу возвращается false

// || — пошагово:
age < 18 || hasTicket
// Шаг 1: вычисляем левую часть → 25 < 18 → false
// Шаг 2: false, нужно проверить правую → hasTicket → true
// Шаг 3: нашли истинное → результат true

true || hasTicket     // true — до hasTicket даже не дошли!
// Шаг 1: true → истинное → СТОП (short-circuit)
// Шаг 2: правая часть НЕ вычисляется, сразу возвращается true

// ! — переворачивает:
!true                // false — true стало false
!false               // true  — false стало true
!(age >= 18)
// Шаг 1: вычисляем внутри скобок → 25 >= 18 → true
// Шаг 2: !true → false
// Результат: false
```

> **Важно:** `||` и `&&` возвращают **конкретное значение**, а не всегда `true`/`false`:
> - `||` → возвращает **первое истинное** значение, или **последнее** если все ложные
> - `&&` → возвращает **первое ложное** значение, или **последнее** если все истинные

```js
// Именно поэтому || используют как "значение по умолчанию":
null || "Гость"
// Шаг 1: null → ложное → идём к правой части
// Шаг 2: "Гость" → истинное → возвращаем "Гость"
// Результат: "Гость"

"Игорь" || "Гость"
// Шаг 1: "Игорь" → истинное → СТОП, возвращаем "Игорь"
// Шаг 2: до "Гость" даже не дошли
// Результат: "Игорь"

0 || 42
// Шаг 1: 0 → ложное → идём дальше
// Шаг 2: 42 → истинное → возвращаем 42
// Результат: 42

null || undefined
// Шаг 1: null → ложное → идём дальше
// Шаг 2: undefined → ложное → это последнее, возвращаем его
// Результат: undefined
```

### !! — двойное отрицание (приведение к boolean)

`!!` — это два оператора `!` подряд. Двойное отрицание отрицания превращает любое значение в его булевый эквивалент.

**Как работает — по шагам:**

```js
!!null
// Шаг 1: !null  → null это falsy → true
// Шаг 2: !true  → false
// Итог: false

!!27
// Шаг 1: !27   → 27 это truthy → false
// Шаг 2: !false → true
// Итог: true

!!"привет"
// Шаг 1: !"привет" → truthy → false
// Шаг 2: !false    → true
// Итог: true

!!0
// Шаг 1: !0    → 0 это falsy → true
// Шаг 2: !true → false
// Итог: false
```

**Зачем это нужно:** это краткий способ преобразовать любое значение в `true` или `false`. Аналог `Boolean(значение)`:

```js
Boolean(null)  // false — официальный способ
!!null         // false — короткий способ, делает то же самое

Boolean("текст")  // true
!!"текст"         // true
```

**Формула-якорь:** `!!` = "является ли это значение истинным?" → возвращает `true` или `false`.

### Оператор нулевого слияния `??`

**Зачем нужен:** `||` возвращает правую часть для ЛЮБОГО ложного значения (`0`, `""`, `false`, `null`, `undefined`). Но иногда `0` или пустая строка — это валидные данные, и мы хотим вернуть правую часть **только если левая — именно `null` или `undefined`**. Для этого и существует `??`.

**Что делает:** возвращает правый операнд, если левый — `null` или `undefined`. Иначе возвращает левый.

```js
// Проблема с ||:
let userScore = 0;
let score = userScore || 100;  // 100 — НЕВЕРНО! 0 тоже ложный, хотя это валидный счёт

// Решение с ??:
let score2 = userScore ?? 100; // 0 — ВЕРНО! 0 не null/undefined, возвращаем его
```

```js
// Типичное использование — значение по умолчанию:
let userName = null;
let displayName = userName ?? "Гость";  // "Гость" — userName это null

let theme = undefined;
let currentTheme = theme ?? "light";    // "light" — theme это undefined

let age = 0;
let userAge = age ?? 18;               // 0 — age это 0, не null/undefined
```

**Сравнение `||` и `??` — пошагово:**

```js
0 || "default"
// Шаг 1: 0 → falsy → идём к правой части
// Шаг 2: "default" → истинное → возвращаем "default"
// Результат: "default"  (0 считается "ложным", хотя это валидное число!)

0 ?? "default"
// Шаг 1: 0 — это null? Нет. undefined? Нет.
// Шаг 2: 0 не null/undefined → возвращаем 0
// Результат: 0  ← правильно! 0 это реальное число

"" || "default"
// Шаг 1: "" → falsy (пустая строка) → идём к правой части
// Шаг 2: "default" → возвращаем "default"
// Результат: "default"  (пустая строка заменилась — может быть нежелательно)

"" ?? "default"
// Шаг 1: "" — это null? Нет. undefined? Нет.
// Шаг 2: "" не null/undefined → возвращаем ""
// Результат: ""  ← пустая строка сохранилась

null || "default"
// Шаг 1: null → falsy → правая часть
// Результат: "default"

null ?? "default"
// Шаг 1: null — это null? Да → идём к правой части
// Результат: "default"

undefined || "default"   // undefined → falsy → "default"
undefined ?? "default"   // undefined → это undefined → "default"
// Для null и undefined результат || и ?? одинаковый
```

**Правило:** если нужно "дефолтное значение когда данных нет" — используй `??`. Если "дефолтное значение когда что-то ложное" — используй `||`.

**Формула-якорь:** `??` = "только если null или undefined", `||` = "если что угодно ложное".

**🔍 Мини-проверка:** что вернёт каждое выражение?

```js
0 || 10       // ?
0 ?? 10       // ?
"" || "Гость" // ?
"" ?? "Гость" // ?
null ?? 42    // ?
```

<details>
<summary>Ответ</summary>
<pre>0 || 10        → 10    (0 falsy → идём к правой)
0 ?? 10        → 0     (0 не null/undefined → возвращаем 0)
"" || "Гость"  → "Гость" ("" falsy → идём к правой)
"" ?? "Гость"  → ""    ("" не null/undefined → возвращаем "")
null ?? 42     → 42    (null → идём к правой)</pre>
</details>

### Оператор нулевого присваивания `??=`

Присваивает значение **только если переменная** `null` **или** `undefined`:

```js
let user = null;
user ??= "Гость";     // user стал "Гость" (был null)
console.log(user);    // "Гость"

let name = "Игорь";
name ??= "Гость";     // ничего не изменилось (name уже есть)
console.log(name);    // "Игорь"
```

Это сокращение для: `user = user ?? "Гость"`.

### Оператор конкатенации строк `+`

Если хотя бы один операнд — строка, `+` склеивает строки вместо сложения:

```js
"Привет" + " " + "мир"
// Шаг 1: "Привет" + " " = "Привет " (обе строки — склеиваем)
// Шаг 2: "Привет " + "мир" = "Привет мир"
// Результат: "Привет мир"

"Возраст: " + 25
// Шаг 1: JS видит строку слева → правило: при + со строкой всё становится строкой
// Шаг 2: 25 → "25" (число стало строкой)
// Шаг 3: "Возраст: " + "25" = "Возраст: 25"
// Результат: "Возраст: 25"
```

## Область видимости (Scope)

> **Забегая вперёд:** в примерах ниже используются `function` (глава 4), `if` (глава 2) и `for` (глава 3). Не нужно понимать их синтаксис прямо сейчас — тут важно увидеть **как переменная ведёт себя** внутри и снаружи блоков `{}`. Когда дойдёшь до этих глав, вернись сюда — станет ещё понятнее.

### Что такое область видимости?

**Область видимости** — это часть кода, в которой переменная существует и доступна. За пределами своей области переменная не существует.

### Зачем нужна область видимости?

Без неё все переменные были бы глобальными. Представь большую программу — тысячи переменных с именами `x`, `name`, `data`. Они все конфликтовали бы друг с другом. Область видимости изолирует переменные и позволяет безопасно использовать одинаковые имена в разных частях кода.

### Типы областей видимости

**Глобальная** — переменная объявлена вне любых функций и блоков. Доступна везде.

```js
let globalVar = "я глобальная";

function show() {
  console.log(globalVar); // ✅ — видно изнутри функции
}

show();
console.log(globalVar); // ✅ — видно снаружи
```

**Функциональная** — переменная объявлена внутри функции. Доступна только внутри этой функции.

```js
function calculate() {
  let result = 42; // живёт только внутри calculate
  console.log(result); // ✅
}

calculate();
console.log(result); // ❌ ReferenceError — снаружи не существует
```

**Блочная** — переменная `let`/`const` объявлена внутри блока `{}`. Доступна только внутри этого блока.

> **Забегая вперёд:** `for` — это цикл (глава 3). Он повторяет блок кода несколько раз. `for (let i = 0; i < 3; i++)` означает: "начни с `i = 0`, повторяй пока `i < 3`, каждый раз увеличивай `i` на 1". Здесь важно не как работает цикл, а что переменная `i` объявлена через `let` и живёт только внутри `{}`.

```js
for (let i = 0; i < 3; i++) {
  console.log(i); // ✅ 0, 1, 2 — i доступна внутри цикла
}
console.log(i); // ❌ ReferenceError — i живёт только в for
```

### Вложенные области видимости

Внутренняя область видит переменные внешней — но не наоборот:

```js
let outer = "внешняя";

function inner() {
  let local = "локальная";
  console.log(outer); // ✅ — видим внешнюю переменную
  console.log(local); // ✅
}

inner();
console.log(outer); // ✅
console.log(local); // ❌ — не видим локальную снаружи
```

## Типы данных

### Что такое тип данных?

**Тип данных** — это категория значения, определяющая:
- Какие операции можно с ним делать
- Сколько памяти оно занимает
- Как оно хранится и сравнивается

Число и строка — разные типы. С числом можно сложить другое число. Со строкой — нет (ну, или результат будет не тем что ожидаешь).

### Сколько типов в JavaScript?

В JavaScript **8 типов данных**. Они делятся на две группы.

## Примитивные типы (7 штук)

**Примитив** — это простое одиночное значение. Примитивы хранятся **по значению** — при копировании создаётся независимая копия.

### string — строка

**Что это:** последовательность символов (текст).

**Зачем нужна:** хранить и работать с текстом — имена, сообщения, адреса, любые текстовые данные.

```js
let name = "Игорь";           // двойные кавычки
let city = 'Москва';          // одинарные кавычки — то же самое
let greeting = `Привет, ${name}!`; // шаблонная строка — можно вставлять выражения
```

Синтаксис `${...}` — это **интерполяция строк** (string interpolation).

**Интерполяция** — это встраивание переменных и выражений прямо внутрь строки. Вместо того чтобы "склеивать" строки через `+`, ты просто пишешь `${переменная}` в нужном месте.

```js
let name = "Игорь";
let age = 25;

// Старый способ — конкатенация через +:
let info1 = "Меня зовут " + name + ", мне " + age + " лет.";

// Современный способ — интерполяция:
let info2 = `Меня зовут ${name}, мне ${age} лет.`;
// Результат одинаковый: "Меня зовут Игорь, мне 25 лет."

// Внутри ${} можно писать любое выражение:
let math = `2 + 2 = ${2 + 2}`;   // "2 + 2 = 4"
let upper = `${name.toUpperCase()} — это имя`; // "ИГОРЬ — это имя"
```

**Формула-якорь:** интерполяция = `${выражение}` внутри обратных кавычек `` ` ``.

**Важное свойство:** строки **неизменяемы**. Нельзя изменить один символ — только создать новую строку.

```js
let str = "Привет";
str[0] = "п"; // ничего не произойдёт (в строгом режиме — ошибка)
str = "привет"; // так правильно — создаём новую строку
```

### number — число

**Что это:** числовой тип. В отличие от других языков, в JS нет отдельных типов для целых и дробных чисел — всё это `number`.

**Зачем нужен:** математические вычисления, счётчики, координаты, возраст, цены.

```js
let age = 25;         // целое число
let price = 99.99;    // дробное число
let negative = -10;   // отрицательное
let big = 1e6;        // 1000000 (научная нотация)
```

**Ловушка: проблема точности (floating point)** <!-- добавлено: 2026-03-04 -->

Числа в JS хранятся в формате IEEE 754 (64-битное число). Некоторые дроби не могут быть представлены точно в двоичной системе — это приводит к неожиданным результатам:

```js
console.log(0.1 + 0.2); // 0.30000000000000004 (не 0.3!)
console.log(0.1 + 0.2 === 0.3); // false ← ловушка!
```

Почему? Аналогия: попробуй записать 1/3 в виде десятичной дроби — 0.333333... (бесконечно). Точно так же 0.1 в двоичной системе — бесконечная дробь. Компьютер вынужден её округлить.

**Как правильно сравнивать дроби:**
```js
// Плохо:
0.1 + 0.2 === 0.3  // false ❌

// Хорошо — округлять перед сравнением:
Math.round((0.1 + 0.2) * 10) / 10 === 0.3  // true ✅

// Или использовать разницу (epsilon):
Math.abs(0.1 + 0.2 - 0.3) < 0.000001  // true ✅ — разница ничтожно мала
```

Это **не баг JS** — все языки с IEEE 754 (Python, Java, C#) ведут себя так же. Просто нужно знать об этом.

**Специальные числовые значения:**

```js
// Infinity — бесконечность
console.log(1 / 0);      // Infinity
console.log(-1 / 0);     // -Infinity
console.log(Infinity + 1); // Infinity

// NaN — Not a Number (не число)
// Появляется при некорректных математических операциях
console.log("abc" * 2);  // NaN — нельзя умножить строку на число
console.log(0 / 0);      // NaN
console.log(Math.sqrt(-1)); // NaN — корень из отрицательного числа

// ⚠️ Важно: NaN — это не тип данных! Это ЗНАЧЕНИЕ типа number.
// typeof NaN возвращает "number", потому что NaN — результат числовой операции.
console.log(typeof NaN); // "number" — не отдельный тип, а особое числовое значение

// Особенность NaN — он не равен самому себе!
console.log(NaN === NaN); // false — единственное значение в JS не равное себе
// Почему? NaN означает "результат некорректной операции".
// Разные некорректные операции дают "не число", и это разные "не числа":
// "abc" * 2 → NaN,  0 / 0 → NaN — это разные ошибки, сравнивать их бессмысленно.
// Поэтому создатели JS решили: NaN != NaN всегда.

> **Механика: почему NaN !== NaN**
>
> NaN -- это не конкретное значение, а заглушка: "тут была ошибка в вычислениях".
> Разные ошибки дают разные NaN:
> ```
> "hello" * 2  --> NaN   (ошибка: строка в умножении)
> 0 / 0        --> NaN   (ошибка: деление ноль на ноль)
> ```
> Оба результата -- NaN, но причины разные. Сравнивать их бессмысленно,
> как спрашивать "ошибка A == ошибка B?". Ответ: "не знаю" = `false`.
> Поэтому для проверки на NaN используй `Number.isNaN()`, а не `===`.

// Правильная проверка на NaN:
console.log(isNaN(NaN));        // true ✅ — глобальная функция
console.log(isNaN("abc"));      // true ⚠️  — сначала конвертирует: "abc" → NaN → true
console.log(Number.isNaN(NaN)); // true ✅ — строгая проверка
console.log(Number.isNaN("abc")); // false ✅ — строка не является NaN, не конвертирует
// Вывод: всегда используй Number.isNaN() — он не конвертирует, только проверяет тип

// === Все 4 проверки объекта Number === <!-- добавлено: 2026-03-04 -->

// 1. Number.isNaN() — только значение NaN (см. выше)

// 2. Number.isInteger() — целое число?
console.log(Number.isInteger(25));     // true  — целое
console.log(Number.isInteger(25.5));   // false — дробное
console.log(Number.isInteger("25"));   // false — строка, не number
console.log(Number.isInteger(null));   // false — null не number
console.log(Number.isInteger(NaN));    // false — NaN не целое
// Проверяет одновременно: тип (должен быть number) + целостность (нет дробной части)

// 3. Number.isFinite() — конечное число? (не Infinity и не NaN)
console.log(Number.isFinite(42));        // true  — обычное число
console.log(Number.isFinite(1 / 0));     // false — 1/0 = Infinity
console.log(Number.isFinite(-Infinity)); // false — отрицательная бесконечность
console.log(Number.isFinite(NaN));       // false — NaN тоже не конечное
console.log(Number.isFinite("42"));      // false — строка (в отличие от isNaN не конвертирует)
// Полезно когда делаешь деление и хочешь убедиться что результат — нормальное число

// 4. Number.isSafeInteger() — целое И в безопасном диапазоне?
// JS точно хранит целые числа только до 2^53 - 1 = 9007199254740991
console.log(Number.isSafeInteger(42));                // true
console.log(Number.isSafeInteger(9007199254740991));  // true  — максимально безопасное
console.log(Number.isSafeInteger(9007199254740992));  // false — уже за пределом точности
console.log(Number.isSafeInteger(25.5));              // false — дробное
// Нужен для финансовых расчётов и работы с большими ID — там важна точность

// Сравнение всех четырёх:
//                     42      Infinity   NaN    "42"   25.5
// Number.isNaN        false   false      true   false  false
// Number.isInteger    true    false      false  false  false
// Number.isFinite     true    false      false  false  false
// Number.isSafeInteger true   false      false  false  false
```

### undefined — значение не задано

**Что это:** значение которое JS присваивает автоматически когда значение не было задано явно.

```js
let x;              // объявлена, но не инициализирована

> **Забегая вперёд:** ниже — примеры с функциями (глава 4). Пока просто знай: `function name() { ... }` — это блок кода, который можно вызвать по имени. `return` — возвращает результат из функции.

function greet() {
  // нет return — функция ничего не возвращает

function show(name) {

### bigint — очень большие числа

```js
const bigNumber = 9007199254740991n; // n в конце делает его bigint
const huge = 123456789012345678901234567890n;

// С обычными числами нельзя смешивать

bigNumber + 1n;  // ✅

## Копирование по значению vs по ссылке

### Что это и зачем знать?

Это одно из ключевых отличий примитивов от объектов. Непонимание этого — источник самых частых и трудноуловимых багов у новичков.

### Примитивы: копирование по значению

Когда присваиваешь примитив другой переменной — создаётся **независимая копия**. Изменение одной переменной не влияет на другую.

```js
let a = 5;
let b = a; // b получает КОПИЮ значения 5

b = 10;
console.log(a); // 5 — не изменился!
console.log(b); // 10
```

В памяти это выглядит так:
```
a → [5]
b → [5]  ← отдельная ячейка памяти, другая копия
```

**Почему `a` осталась 5 после `b = 10`?**

Переменная — это коробка с одним отделением. Она может держать только **одно значение одновременно**.

Когда пишешь `b = 10` — ты не добавляешь 10 к тому что было. Ты **заменяешь** содержимое. Старое значение (5) просто исчезает из `b`.

```
let b = a;    →   b: [5]        ← положили копию пятёрки
b = 10;       →   b: [10]       ← 5 выброшено, положили 10
```

Пошагово:
```
После b = a:              После b = 10:

a: [5]                    a: [5]    ← коробка a не тронута
b: [5]  ← копия           b: [10]   ← в b пятёрка заменена на 10
```

Именно потому что `a` и `b` — **разные коробки** (копирование по значению), изменение `b` никак не затрагивает `a`.

> **Механика: как это работает внутри**
>
> Сравним примитив и объект шаг за шагом:
>
> ```
> ПРИМИТИВ (два независимых значения):
>   let a = 5;       a: [5]
>   let b = a;       a: [5]    b: [5]     ← скопировали число 5
>   b = 10;          a: [5]    b: [10]    ← a не затронута
>
> ОБЪЕКТ (две ссылки на одну коробку):
>   let obj1 = {x:1};    obj1 ---> { x: 1 }
>   let obj2 = obj1;     obj1 ---> { x: 1 } <--- obj2   ← обе смотрят туда же
>   obj2.x = 99;         obj1 ---> { x: 99 } <--- obj2  ← изменили содержимое
>   console.log(obj1.x); // 99 -- потому что объект ОДИН
> ```
>
> Примитив копируется **целиком** (два независимых числа).
> Объект копируется **по ссылке** (две стрелки на один объект).

### Объекты: копирование по ссылке

> **Забегая вперёд:** объекты подробно разбираются в главе 7. Пока знай: `{ name: "Игорь" }` — это набор свойств (ключ: значение), а `user.name` — обращение к свойству.

Когда присваиваешь объект другой переменной — обе переменные указывают на **один и тот же объект** в памяти.

```js
let user1 = { name: "Игорь" };
let user2 = user1; // user2 получает ССЫЛКУ на тот же объект

user2.name = "Иван"; // меняем через user2
console.log(user1.name); // "Иван" — объект один, изменился у обоих!
```

В памяти:
```
user1 → ссылка → [{ name: "Игорь" }] ← один объект
user2 → ссылка ↗
```

**Аналогия:** представь документ в облаке (Google Docs). Ты поделился ссылкой с другом. Теперь у вас обоих есть ссылка на один документ. Если друг изменил документ — ты тоже видишь изменение. Это копирование по ссылке.

### Почему `{a: 1} === {a: 1}` дает `false`?

<!-- добавлено: 2026-03-05 -->

Каждый раз когда ты пишешь `{}` или `[]` в коде -- JS создает **новый объект** в памяти. Даже если содержимое одинаковое -- это два разных объекта по двум разным адресам.

```js
{a: 1} === {a: 1}   // false -- два разных объекта в памяти
[1, 2] === [1, 2]   // false -- два разных массива в памяти
```

Оператор `===` для объектов сравнивает **ссылки** (адреса в памяти), а не содержимое:

```
{a: 1}  ->  адрес #1001  <- первый объект
{a: 1}  ->  адрес #1002  <- второй объект (другой адрес!)

#1001 === #1002  ->  false  (разные адреса)
```

Для примитивов `===` сравнивает **значения**:
```js
1 === 1         // true -- оба значения 1
"abc" === "abc" // true -- оба значения "abc"
```

> **Формула-якорь:** примитивы сравниваются по значению, объекты -- по ссылке (адресу в памяти).

Чтобы сравнить объекты **по содержимому** -- нужна своя функция (например через `JSON.stringify`).

---

**Как сделать независимую копию объекта:**

> **Забегая вперёд:** `...` (spread-оператор) и `Object.assign` разбираются в главе 7 (Объекты). Пока просто знай что такие способы существуют.

```js
const original = { name: "Игорь", age: 25 };

// Поверхностная копия (для вложенных объектов не работает)
const copy1 = { ...original };
const copy2 = Object.assign({}, original);

// Глубокая копия (копирует всё включая вложенные объекты)
const deepCopy = structuredClone(original);
```

## typeof — проверка типа

### Что это?

`typeof` — оператор который возвращает строку с названием типа значения.

### Зачем нужен?

Когда не знаешь какой тип придёт (из формы, с сервера, из функции) и нужно проверить перед операцией.

```js
typeof "строка"       // "string"
typeof 42             // "number"
typeof 3.14           // "number"
typeof true           // "boolean"
typeof undefined      // "undefined"
typeof null           // "object"    ← историческая ошибка, null не объект!
typeof []             // "object"    ← массив — это объект
typeof {}             // "object"
typeof function(){}   // "function"  ← особый случай! (см. ниже)
typeof Symbol()       // "symbol"
typeof 42n            // "bigint"
```

**Почему `typeof function(){}` возвращает `"function"`, а не `"object"`?** <!-- добавлено: 2026-03-04 -->

Функция в JS технически является объектом (помнишь строку `let fn = function(){} // function — тоже object`). Но `typeof` делает для неё исключение и возвращает `"function"` вместо `"object"`.

Почему? Создатели JS посчитали что функции достаточно особенные (их можно вызвать, у них есть `()`) чтобы отличать их от простых объектов.

```js
let fn = function() {};

// fn — это объект (под капотом)
typeof fn;          // "function"  ← typeof делает исключение для функций
fn instanceof Object; // true ✅   ← функция — это объект, instanceof это подтверждает

// Итог: функция = объект + возможность вызова
// typeof об этом не говорит, но instanceof показывает правду
```

**Запомни:** когда нужно проверить является ли что-то функцией — используй `typeof`. Когда нужно понять является ли что-то объектом (включая функции) — используй `instanceof Object`.

### Что такое точка `.` в коде?

Ты часто будешь видеть запись вида `Array.isArray(...)`, `Number.isNaN(...)`, `value.toString()`. Давай разберём что означает эта точка.

**Точка — это обращение к чему-то внутри объекта.** В JS почти всё является объектом (или ведёт себя как объект). Объект — это контейнер в котором хранятся данные и функции.

```
Array.isArray(...)
│     │
│     └── isArray — функция внутри Array
└── Array — встроенный объект JS
```

**Аналогия:** представь `Array` как отдел в компании. `isArray` — это конкретный сотрудник в этом отделе. Точка — это как сказать "возьми сотрудника `isArray` из отдела `Array`".

Полностью тема объектов разбирается в **главе 7 (Объекты)**. Пока просто знай: `X.Y` = "обратись к Y внутри X".

```js
// Примеры точечной нотации:
Array.isArray([1, 2, 3])  // функция isArray внутри Array
Number.isNaN(NaN)          // функция isNaN внутри Number
"привет".toUpperCase()     // функция toUpperCase внутри строки
(42).toString()            // функция toString внутри числа
```

```js

// Проверка на массив:

Array.isArray({});         // false ✅
Array.isArray("строка");   // false ✅

## Преобразование типов

### Что это?

Преобразование типов — изменение типа значения. Бывает явным (ты сам конвертируешь) и неявным (JavaScript делает сам).

### Явное преобразование

**В строку:**

```js
String(42)          // "42"
String(3.14)        // "3.14"
String(true)        // "true"
String(false)       // "false"
String(null)        // "null"
String(undefined)   // "undefined"
String([1, 2, 3])   // "1,2,3"

// Через toString()
(42).toString()     // "42"
(255).toString(16)  // "ff" — в шестнадцатеричной системе
```

**В число:**

```js
Number("42")        // 42  — строка содержит чистое число → конвертируется
Number("3.14")      // 3.14
Number("")          // 0   — пустая строка → 0 (сюрприз!)
Number("   ")       // 0   — строка из пробелов → пробелы обрезаются → пустая → 0
Number("   42   ")  // 42  — пробелы обрезаются → "42" → 42
Number("42abc")     // NaN — есть нечисловые символы → не число
Number("abc")       // NaN — вообще не число
Number(true)        // 1   — true → 1 (по логике: true = "да" = 1)
Number(false)       // 0   — false → 0 (false = "нет" = 0)
Number(null)        // 0   — null → 0 (запомни: null при конвертации даёт 0)
Number(undefined)   // NaN — undefined вообще не определено → не число
Number([])          // 0   — [] → "" → 0 (пустой массив → пустая строка → 0)
Number([3])         // 3   — [3] → "3" → 3 (массив из одного числа → строка → число)
Number([1,2])       // NaN — [1,2] → "1,2" → NaN (нельзя преобразовать "1,2" в число)

// parseInt и parseFloat — читают число с начала строки, останавливаются на нечисловом:
parseInt("42px")     // 42  — Шаг 1: читаем "4", "2" — числа; Шаг 2: "p" — не число, стоп → 42
parseInt("3.14")     // 3   — parseInt берёт только целую часть, точка — стоп
parseFloat("3.14px") // 3.14 — parseFloat понимает точку, стоп на "p"
parseInt("abc")      // NaN — начинается не с цифры → сразу NaN
```

**В булевый:**

Правило простое: **falsy** (ложных) значений в JS ровно 8. Всё остальное — **truthy** (истинное).

```js
// Falsy — ровно 8 значений, которые становятся false.
// Запомни их все — больше ложных значений нет!
Boolean(0)          // false — ноль, "пустое" число
Boolean(-0)         // false — отрицательный ноль (да, существует)
Boolean(0n)         // false — ноль типа bigint
Boolean("")         // false — пустая строка (ничего нет)
Boolean(null)       // false — явное "нет значения"
Boolean(undefined)  // false — значение не задано
Boolean(NaN)        // false — "не число", т.е. результат невалидной операции
Boolean(false)      // false — само по себе ложное

// Всё остальное — truthy (истинное), даже если выглядит "пустым":
Boolean(1)          // true  — любое ненулевое число
Boolean(-1)         // true  — отрицательное тоже truthy
Boolean("0")        // true  — строка "0" — это НЕ пустая строка, в ней есть символ!
Boolean("false")    // true  — строка "false" — тоже не пустая, в ней есть символы!
Boolean([])         // true  — пустой массив это объект, объект всегда truthy
Boolean({})         // true  — пустой объект тоже truthy
```

> **Ловушка для новичков:** `"0"` и `[]` — truthy! Пустая строка `""` — falsy, но строка `"0"` — нет. Пустой массив `[]` — truthy, хотя в нём ничего нет.

### Неявное преобразование — источник багов

JavaScript иногда автоматически конвертирует типы. Это удобно, но непредсказуемо.

```js
// + с одной строкой — всё становится строкой:

"5" + 2
// Шаг 1: JS видит строку "5" → правило: при + со строкой всё склеивается
// Шаг 2: 2 → "2" (число стало строкой)
// Шаг 3: "5" + "2" = "52"
// Результат: "52"  (не 7!)

2 + "5"
// Шаг 1: справа строка "5" → та же логика: 2 → "2"
// Шаг 2: "2" + "5" = "25"
// Результат: "25"

1 + 2 + "3"
// Шаг 1: вычисляем слева направо: 1 + 2 = 3  (оба числа — обычное сложение)
// Шаг 2: 3 + "3" → 3 → "3" → "3" + "3" = "33"
// Результат: "33"

"3" + 1 + 2
// Шаг 1: "3" + 1 → 1 → "1" → "3" + "1" = "31"
// Шаг 2: "31" + 2 → 2 → "2" → "31" + "2" = "312"
// Результат: "312"  (всё стало строкой с самого начала!)

// - * / конвертируют строки в числа (эти операторы умеют работать ТОЛЬКО с числами):

"5" - 2
// Шаг 1: JS видит - → этот оператор работает только с числами
// Шаг 2: "5" → 5 (строка стала числом)
// Шаг 3: 5 - 2 = 3
// Результат: 3

"5" * "2"
// Шаг 1: * работает только с числами → конвертируем оба операнда
// Шаг 2: "5" → 5, "2" → 2
// Шаг 3: 5 * 2 = 10
// Результат: 10

"10" / "2"
// Шаг 1: / → числа → "10" → 10, "2" → 2
// Шаг 2: 10 / 2 = 5
// Результат: 5

"5" - "3"
// Шаг 1: - → числа → "5" → 5, "3" → 3
// Шаг 2: 5 - 3 = 2
// Результат: 2

// Сравнение с == (конвертирует к числу перед сравнением):
0 == false      // false → 0, потом 0 == 0  → true
"" == false     // false → 0, "" → 0, потом 0 == 0  → true
"1" == 1        // "1" → 1, потом 1 == 1  → true
null == undefined // true — специальный случай (не конвертация, а отдельное правило)
null == 0       // false — null НЕ конвертируется в 0!
```

**🔍 Мини-проверка:** что вернёт каждое выражение?

```js
5 + "3"      // ?
"5" - 2      // ?
true + true  // ?
1 + 2 + "3"  // ?
"3" + 1 + 2  // ?
```

<details>
<summary>Ответ</summary>
<pre>5 + "3"      → "53"   (строка → конкатенация)
"5" - 2      → 3      (- конвертирует в числа: 5 - 2)
true + true  → 2      (true→1, 1+1=2)
1 + 2 + "3"  → "33"   (сначала 1+2=3, потом 3+"3"="33")
"3" + 1 + 2  → "312"  (сначала "3"+1="31", потом "31"+2="312")</pre>
</details>

## Операторы сравнения: == и ===

### Что такое == и ===?

Оба оператора сравнивают два значения. Разница — в том как они обращаются с разными типами.

### == — нестрогое равенство

**Что делает:** перед сравнением приводит типы к одному, потом сравнивает.

**Зачем существует:** исторически был первым. Сейчас использование `==` считается плохой практикой.

<!-- добавлено: 2026-03-04 -->
**Как работает конвертация:** JS приводит оба значения к **числу**, потом сравнивает. Пошагово:

```js
0 == false   // false → 0,   потом 0 == 0   → true
"" == false  // false → 0,   "" → 0,  потом 0 == 0   → true
"5" == 5     // "5" → 5,    потом 5 == 5    → true
1 == true    // true → 1,   потом 1 == 1    → true
null == undefined // true — специальный случай (см. ниже)
[] == false     // [] → "" → 0, false → 0 → true
```

### === — строгое равенство (всегда используй это)

**Что делает:** сравнивает значение И тип. Если типы разные — всегда `false`.

**Зачем нужен:** предсказуемое поведение без неожиданных преобразований.

```js
0 === false     // false — разные типы
"5" === 5       // false — разные типы
null === undefined // false — разные типы
5 === 5         // true ✅
"5" === "5"     // true ✅
```

> **Правило:** всегда используй `===` и `!==`. Использование `==` — красный флаг на код-ревью.

### Специальный случай: null и undefined при ==

<!-- добавлено: 2026-03-04 -->

`null` и `undefined` — особые значения. При `==` они **не конвертируются в числа**, как все остальные. Для них действует отдельное правило:

> `null == undefined` → **true** (только друг с другом)
> `null` и `undefined` со всем остальным через `==` → **false**

```js
null == undefined  // true  — особое правило JS, исключение
null == 0          // false — null НЕ конвертируется в 0!
null == false      // false — тоже нет!
null == ""         // false — тоже нет!

undefined == 0     // false — undefined тоже не конвертируется
undefined == false // false
undefined == ""    // false
```

**Почему так?** Создатель JS сделал специальное исключение: `null` и `undefined` "равны" только друг другу через `==`. Это отдельное правило, не результат конвертации.

**На практике:** если хочешь проверить "значение отсутствует" (null или undefined):

```js
if (value == null) { ... }  // сработает для null И для undefined
// это единственный случай когда == оправдан
```

## Взаимодействие с пользователем: alert, prompt, confirm

<!-- добавлено: 2026-03-04 -->

**Зачем нужны:** самый быстрый способ показать сообщение или получить ввод от пользователя прямо в браузере — без создания HTML-форм. Используются для тестирования и простых скриптов.

**Важно:** работают только в браузере. В Node.js их нет.

### Сводная таблица

| Функция | Что делает | Что возвращает | Типичная ошибка |
|---------|-----------|----------------|-----------------|
| `alert(msg)` | Показывает сообщение, кнопка OK | `undefined` | Ожидать что-то вернёт |
| `prompt(msg, default)` | Поле ввода текста | `string` или `null` | Забыть конвертировать в `Number()` |
| `confirm(msg)` | Вопрос OK/Отмена | `boolean` | — |

### alert — показать сообщение

```js
alert("Привет, мир!");
// Шаг 1: браузер показывает окно с текстом "Привет, мир!" и кнопкой OK
// Шаг 2: скрипт останавливается и ждёт
// Шаг 3: пользователь нажимает OK → скрипт продолжается
// Возвращает: undefined

alert(25);
// Шаг 1: 25 (number) → автоматически в строку "25"
// Шаг 2: показывает "25"
// Любое значение alert конвертирует в строку для отображения
```

### prompt — получить ввод

```js
let name = prompt("Как тебя зовут?");
// Шаг 1: показывает окно с полем ввода и текстом "Как тебя зовут?"
// Шаг 2: пользователь вводит текст → нажимает OK
// Шаг 3: prompt возвращает введённый текст как СТРОКУ
// Шаг 3а: если нажал "Отмена" → возвращает null

let age = prompt("Сколько тебе лет?", 18);
// Второй аргумент (18) — значение по умолчанию в поле ввода

console.log(typeof age); // "string" — prompt ВСЕГДА возвращает строку!

// ⚠️ Типичная ошибка:
let wrongAge = prompt("Возраст?");
wrongAge + 1;   // "251" ← ловушка! строка + число = конкатенация

// ✅ Правильно — сразу конвертируй:
let correctAge = Number(prompt("Возраст?"));
correctAge + 1; // 26 ✅ — теперь это сложение чисел
```

### confirm — задать вопрос да/нет

```js
let isAdult = confirm("Тебе есть 18 лет?");
// Шаг 1: показывает окно с вопросом и двумя кнопками: OK и Отмена
// Шаг 2: OK → возвращает true, Отмена → возвращает false

// Забегая вперёд: if/else — условная конструкция из главы 2.
// Пока просто смотри как confirm используется на практике:
if (isAdult) {
  alert("Добро пожаловать!");  // выполнится если true
} else {
  alert("Доступ закрыт");      // выполнится если false
}
```

**Ограничения диалогов:**
- Останавливают выполнение скрипта пока пользователь не ответит
- Внешний вид нельзя настроить — это браузерные диалоги
- В реальных проектах заменяются на красивые модальные окна (React/HTML)

## Итог раздела

```
ПЕРЕМЕННЫЕ:
  const  → везде по умолчанию, нельзя переназначить
  let    → когда значение изменится
  var    → устарел, не используй (знай для собеседований)

КЛЮЧЕВЫЕ ПРОБЛЕМЫ var:
  - функциональная область (вытекает из if/for)
  - hoisting с undefined (скрытые баги)

ОПЕРАТОРЫ:
  По числу операндов: унарный (1), бинарный (2), тернарный (3)
  Арифметические:    + - * / % **
  Присваивания:      = += -= *= /=
  Сравнения:         === !== > < >= <=
  Логические:        && || !
  Нулевое слияние:   ?? (только null/undefined) vs || (любое ложное)
  Нулевое присваивание: ??= (присвоить если null/undefined)
  Инкремент/декремент:  ++ --
  Тернарный:         условие ? если_true : если_false

БРАУЗЕРНЫЕ ДИАЛОГИ:
  alert("текст")           → показать сообщение
  prompt("вопрос", default) → получить строку от пользователя (или null)
  confirm("вопрос")        → получить true/false (OK/Отмена)

ОБЛАСТЬ ВИДИМОСТИ:
  - глобальная → везде
  - функциональная → только в функции
  - блочная → только в {} (для let/const)

8 ТИПОВ ДАННЫХ:
  Примитивы: string, number, boolean, null, undefined, bigint, symbol
  Объект: object (включает массивы и функции)

КОПИРОВАНИЕ:
  Примитивы → по значению (независимые копии)
  Объекты   → по ссылке (один объект, несколько имён)

ПРОВЕРКИ:
  typeof null → "object" (историческая ошибка!)
  Array.isArray([]) → true (правильная проверка)
  value === null → правильная проверка на null

СРАВНЕНИЕ:
  === всегда вместо == (строгое, без конвертации типов)
```

## Попробуй сам

Маленькие задания для закрепления прямо после прочтения. Открой Node.js и проверь.

**1. Предскажи вывод** (не запускай, сначала подумай):
```js
let a = "5";
let b = 2;
console.log(a + b);
console.log(a - b);
console.log(typeof (a - b));
```

<details>
<summary>Ответ</summary>
<pre>"52" (конкатенация: строка + число)
3    (вычитание: "5" → 5, потом 5 - 2)
"number" (результат вычитания — число)</pre>
</details>

**2. Заполни пропуски** — что нужно подставить вместо `???`:
```js
let score = ???;
let displayScore = score ?? 100;
console.log(displayScore); // должно вывести 0
```

<details>
<summary>Ответ</summary>
<p>`score = 0`. Потому что `??` возвращает правую часть только для null/undefined. `0` — не null и не undefined, поэтому `0 ?? 100` вернёт `0`.</p>
</details>

**3. Исправь баг:**
```js
const colors = ["красный", "синий"];
colors = ["зелёный"];
colors.push("жёлтый");
```

<details>
<summary>Ответ</summary>
<p>Строка 2 вызовет TypeError — const запрещает переназначение. Строка 3 работает нормально — push меняет содержимое, не переназначает переменную. Исправление: убрать строку 2 или заменить на `colors[0] = "зелёный"` если нужно изменить первый элемент.</p>
</details>
