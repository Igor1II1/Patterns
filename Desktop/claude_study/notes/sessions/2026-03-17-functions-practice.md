## Вопросы и объяснения

### Callback — ключевое объяснение
Игорь не понимал зачем передавать функцию в функцию. Помогла аналогия:
- Функция с коллбэком = универсальный инструмент
- Она знает КОГДА и С ЧЕМ делать, но ЧТО делать — решаешь ты снаружи
- Аналогия курьера: курьер один, но инструкции разные каждый раз
- Без коллбэка: отдельная функция на каждый случай (add, multiply, subtract...)
- С коллбэком: одна функция operate, бесконечное количество поведений

### Scope — Блок 2 ошибка
Не понял что функция increment() без своего let count работает с внешней переменной.
Ключевое: let внутри функции = новая переменная. Нет let = берёт внешнюю и МОЖЕТ МЕНЯТЬ.

### Рекурсия — типичные ошибки
- fibonacci: написал n=(n-1)+(n-2) — считал числа, не вызывал функцию. Нужно fibonacci(n-1)+fibonacci(n-2)
- sumRange: написал return sumRange(n-1) — забыл прибавить n. Нужно n + sumRange(n-1)
- flatten: return внутри цикла — выходит из функции на первом массиве

### Чистые функции
- addToArray назвал чистой (push мутирует входной массив)
- getRandom назвал чистой (разный выход при одинаковом входе)
- Два критерия: 1) одинаковый вход → одинаковый выход, 2) не меняет ничего снаружи

### Стрелочные функции в аргументах
Не понимал почему можно писать transform(5, n => n+1, n => n*2). Объяснение: функция — это значение, как число. Везде где можно написать 5, можно написать n => n+1.
