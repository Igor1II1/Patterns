# Сессия 2026-03-13 — Система повторений и создание упражнений

## Что прошли

Организационная сессия — без изучения новой темы. Полностью посвящена улучшению процесса обучения.

### 1. Проблема: забывание пройденных тем

Игорь заметил, что даже "сильные" темы начинают забываться без повторения. Практика покрывала только слабые места, а сильные концепции (например, switch/case из темы Условия) почти не повторялись.

### 2. Решение: система "5 слоёв повторения"

Создана комплексная система повторений, встроенная в каждую сессию:

**Слой 1 — Ежедневный прогрев (10-15 мин)**
- 3 карточки-минимумы из разных прошлых тем
- Интервалы: +1д → +3д → +7д → +14д → +30д → ЗАКРЕПЛЕНО
- Ошибки: 1-2 → назад на 1 шаг, 3+ → сброс к +1 дню

**Слой 2 — Вплетение в новые задания**
- Каждое задание новой темы намеренно использует 1-2 концепции из прошлого

**Слой 3 — Чередование 70/30**
- 70% новая тема + 30% микс из прошлых
- Последние 2-3 задания из 10 — микс

**Слой 4 — Еженедельный микс**
- Последний учебный день недели
- 5-7 заданий из РАЗНЫХ тем вперемешку

**Слой 5 — Практикум (каждые 3 темы)**
- 100% покрытие ВСЕХ пройденных концепций
- Теория + карточки + комплексные задачи + багфиксы

### 3. Правило 100% покрытия (правило 15)

Для каждой темы — 4 обязательных компонента:
- (A) Чеклист покрытия — таблица каждой концепции
- (B) Комплексная задача — всё из темы в одном задании
- (C) Карточки-минимумы — микро-задания для прогрева
- (D) Блокировка перехода — пока чеклист не 100%

### 4. Создание упражнений для ВСЕХ блоков

Запущены 5 параллельных агентов для создания упражнений на весь 6-месячный курс:
- Блок 0: чеклисты + дополнения для 0.4-0.10
- Блок 1: 11 тем (ООП → type-checking)

## Вопросы и объяснения

**В: Сильные места тоже нужно повторять?**
О: Да, обязательно. По кривой забывания Эббингауза даже хорошо выученный материал забывается без повторений. Система spaced repetition решает это: интервалы увеличиваются (+1д → +30д), и на каждом интервале тратится всего 2-3 минуты.

**В: Как будут проходить практикумы каждые 3 темы?**
О: Полная проверка ВСЕХ пройденных тем (не только последних 3): чеклист → теория → карточки → комплексные задачи → багфиксы → пока не 100%.

## Ключевые выводы

1. Без системы повторений даже сильные темы забываются
2. Повторение должно быть встроено в каждый день, а не только в отдельные сессии
3. 100% покрытие практикой — обязательное условие перехода к следующей теме
4. Игорь сам оценил: ООП нужно пройти заново (this, static, new, super — слабые)
