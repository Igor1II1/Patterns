## Вопросы и объяснения

### 1. Зачем нужны falsy и truthy?
Игорь спросил: "я все еще плохо понимаю что это и для чего falsy и truhtly нужны"

Falsy/truthy позволяют писать короткие проверки. Вместо длинного `if (user !== null && user !== undefined)` пишем `if (user)`. JS автоматически приводит значение к boolean.

Пример с корзиной: `if (cart)` — проверяет существует ли корзина ([] — truthy, значит ДА). Но `if (cart.length)` — проверяет есть ли элементы (0 — falsy, значит пустая).

### 2. Точечная нотация: свойства vs методы
Игорь заметил паттерн: `cart.length`, `console.log()`, `user.name` — всё через точку.

Правило:
- Свойство (без скобок) = данные: `cart.length`, `user.name`, `str.length`
- Метод (со скобками) = действие: `console.log()`, `str.includes()`, `Number.isNaN()`

Точка — это доступ к содержимому объекта. Подробнее в главах 5-7.

### 3. console.log — другие методы

Есть: console.warn(), console.error(), console.table(), console.time()/timeEnd(). Подробнее — в главе 9 (Ошибки). На текущем этапе достаточно знать console.log().

### 4. Тернарный оператор — выражение vs инструкция
Тернарный — ВЫРАЖЕНИЕ (expression): возвращает значение, можно присвоить.
if/else — ИНСТРУКЦИЯ (statement): выполняет код, не возвращает значение.

```js
const msg = age >= 18 ? "взрослый" : "ребёнок";  // ✅ работает
const msg = if (age >= 18) { "взрослый" };         // ❌ ошибка синтаксиса
```

### 5. switch: когда использовать
- switch — одна переменная, много конкретных значений
- if/else — диапазоны, сложные условия с && и ||
- switch сравнивает через === (строгое)
- Fall-through (провал без break) полезен для группировки одинаковых case

### 6. Short-circuit evaluation (подробно)
&& и || — ленивые, не вычисляют правую часть если не нужно.

&& возвращает ПЕРВОЕ falsy или ПОСЛЕДНЕЕ значение:
- `1 && 2 && 3` → 3 (все truthy, последний)
- `1 && 0 && 3` → 0 (первый falsy)

|| возвращает ПЕРВОЕ truthy или ПОСЛЕДНЕЕ значение:
- `0 || null || "найден"` → "найден" (первый truthy)
- `0 || null || undefined` → undefined (все falsy, последний)

?? проверяет только null и undefined (не все falsy):
- `0 ?? 10` → 0 (0 не null/undefined)
- `null ?? undefined ?? "дефолт"` → "дефолт"

### 7. return vs console.log в функциях
Ошибка Игоря: в 2.4 написал console.log внутри switch вместо return.
- console.log — печатает в консоль, функция возвращает undefined
- return — возвращает значение вызывающему коду
- Правило: если функция "должна возвращать" — нужен return

### 8. Независимые if vs else if (паттерн накопления)
Самое сложное в сессии — задание 2.7 (3 попытки).

else if = только ОДНА ветка сработает (взаимоисключающие):
```js
if (a) { return 20; }
else if (b) { return 10; }  // если a=true, сюда не попадём
```

Отдельные if = каждое проверяется независимо:
```js
let discount = 0;
if (user.vip) discount += 20;        // проверяется
if (cart.total > 5000) discount += 10; // тоже проверяется
if (discount > 30) discount = 30;      // ограничение
return discount;
```

### 9. Как проверить что значение — число
Игорь пытался написать `Number.isNAN()` — неверный регистр.
Правильно: `Number.isNaN(value)` — проверяет строго на NaN.
Для проверки "является ли числом": `typeof value === "number" && !Number.isNaN(value)`

### 10. Как вызвать функцию-выражение
Вопрос по заданию 2.3: как вызвать `const status = isOnline ? ...`
Ответ: нужно сначала определить status, потом вызвать функцию:
```js
function isOnline(status) {
    return status ? "в сети" : "не в сети";
}
console.log(isOnline(true));  // "в сети"
```

## Дополнительная информация

### Паттерн аккумулятора с ограничением
Часто встречается в реальном коде:
```js
let total = 0;
if (condition1) total += value1;
if (condition2) total += value2;
if (total > MAX) total = MAX;  // или Math.min(total, MAX)
return total;
```

### Когда использовать какой инструмент
| Задача | Инструмент |
|--------|-----------|
| Да/нет выбор | if/else или тернарный |
| Одно значение, много вариантов | switch |
| Составное условие (И, ИЛИ) | if с && и || |
| Накопление (суммирование) | отдельные if + переменная |
| Значение по умолчанию | \|\| или ?? |
