## Вопросы и объяснения

### Почему {a:1} === {a:1} дает false?
Каждый `{}` или `[]` создает новый объект в памяти с уникальным адресом. Оператор `===` для объектов сравнивает адреса (ссылки), не содержимое. Два объекта с одинаковым содержимым -- два разных адреса -- false.

### Как JSON.stringify позволяет сравнить объекты?
1. `JSON.stringify([1,2])` -> `'[1,2]'` (объект -> строка)
2. Теперь сравниваем строки, а не объекты
3. Строки -- примитивы, `===` сравнивает их посимвольно
4. JSON.stringify -- "мост": из мира ссылок в мир значений

### Почему в shallowEqual последняя ветка else возвращает false?
4 комбинации типов для двух аргументов:
- if: оба НЕ объекты -> сравни ===
- else if: a объект, b НЕ объект -> false (разные типы)
- else if: оба объекты -> JSON.stringify
- else: a НЕ объект, b объект -> false (единственная оставшаяся комбинация)

### Почему const позволяет менять свойства объекта?
`const` фиксирует ссылку (адрес), не содержимое. Нельзя переприсвоить переменную (`user = {}`), но можно менять содержимое объекта (`user.name = "Иван"`). Аналогия: приклеенная бирка на коробке -- бирку не переклеить, но содержимое менять можно.

### Все типы данных -- примитивы кроме объектов?
Да. 7 примитивов (string, number, boolean, null, undefined, bigint, symbol) + 1 объектный тип (object).

## Дополнительная информация

### Три способа независимой копии объекта
```js
const copy1 = { ...original };              // spread (поверхностная)
const copy2 = Object.assign({}, original);   // Object.assign (поверхностная)
const copy3 = structuredClone(original);     // глубокая копия
```
Поверхностная копия не копирует вложенные объекты (они остаются ссылками). Глубокая -- копирует всё.

### NaN -- единственное значение не равное самому себе
```js
NaN === NaN          // false (!)
Number.isNaN(NaN)    // true -- правильный способ проверки
isNaN("abc")         // true -- ненадёжный (сначала конвертирует)
Number.isNaN("abc")  // false -- строгая проверка
```
