## Что в главе 01-variables.md

### Переменная — что это
Именованное место в памяти. `let userName = "Игорь"` — это человеческое имя для адреса ячейки `0x4F2A`.

**Зачем:** хранить данные, давать им понятные имена, переиспользовать, изменять в одном месте.

### var / let / const

| | var | let | const |
|---|---|---|---|
| Переназначение | ✅ | ✅ | ❌ |
| Перефъявление | ✅ | ❌ | ❌ |
| Область видимости | функция | блок `{}` | блок `{}` |
| Hoisting | `undefined` | TDZ ошибка | TDZ ошибка |

**Правило:** по умолчанию `const`. Если нужно менять — `let`. `var` не используем.

### Hoisting (всплытие)
- `var` всплывает со значением `undefined` — поэтому не падает, но даёт неожиданный результат
- `let`/`const` всплывают, но попадают в TDZ (Temporal Dead Zone) — при обращении до объявления выбрасывается `ReferenceError`
- Function Declaration (`function foo(){}`) всплывает целиком, поэтому вызов до объявления работает
- Function Expression (`const foo = function(){}`) не всплывает — ведёт себя как `const`

### Типы данных

**7 примитивных:**
- `number` — числа (включая `NaN`, `Infinity`)
- `string` — строки
- `boolean` — `true` / `false`
- `null` — намеренное отсутствие значения
- `undefined` — значение не было присвоено
- `bigint` — очень большие числа
- `symbol` — уникальные идентификаторы

**1 объектный:** `object` (объекты, массивы, null — баг JS)

### typeof
```js
typeof null        // "object" ← историческая ошибка JS
typeof []          // "object" ← нужно Array.isArray()
typeof function(){} // "function"
typeof NaN         // "number" ← NaN — это тип number
```

### Классификация операторов
- **Унарный** — 1 операнд: `-5`, `!true`, `typeof x`, `x++`
- **Бинарный** — 2 операнда: `5 + 3`, `a === b`, `a && b`
- **Тернарный** — 3 операнда: `условие ? a : b`

Знак `-` может быть и унарным (отрицание) и бинарным (вычитание) — зависит от контекста.

### == vs ===
- `===` (строгое) — сравнивает значение И тип. Всегда использовать это.
- `==` (нестрогое) — приводит типы перед сравнением, много сюрпризов:
  - `0 == false` → `true`
  - `null == undefined` → `true`
  - `NaN == NaN` → `false` (NaN не равен ничему, даже себе)

Проверка NaN: `Number.isNaN(value)` или `Object.is(value, NaN)`.

### Оператор ?? (nullish coalescing)
```js
// Проблема ||: считает 0 и "" ложными
0 || "default"   // "default" — НЕВЕРНО если 0 валидный счёт

// Решение ??:  срабатывает только на null/undefined
0 ?? "default"   // 0 — ВЕРНО
"" ?? "default"  // "" — ВЕРНО
null ?? "default" // "default"
```

**Правило-якорь:** `??` = только null или undefined, `||` = любое ложное.

### Оператор ??= (nullish assignment)
```js
let user = null;
user ??= "Гость";  // user стал "Гость" — был null
// Эквивалентно: user = user ?? "Гость"

let name = "Игорь";
name ??= "Гость";  // ничего не изменилось — name уже есть
```

### Копирование по значению vs по ссылке

**Примитивы** копируются по значению — создаётся независимая копия:
```js
let a = 5;
let b = a;  // b получает КОПИЮ значения 5
b = 10;     // a не изменилась — у неё своя копия
```

**Объекты** копируются по ссылке — обе переменные указывают на одно место в памяти:
```js
let obj1 = { count: 0 };
let obj2 = obj1;  // obj2 указывает на ТОТ ЖЕ объект
obj2.count = 5;
console.log(obj1.count); // 5 — obj1 тоже изменился!
```

Независимая копия объекта:
```js
const copy1 = { ...obj1 };           // spread (поверхностная)
const copy2 = Object.assign({}, obj1); // assign (поверхностная)
const copy3 = JSON.parse(JSON.stringify(obj1)); // глубокая
```

### alert / prompt / confirm
```js
alert("Сообщение");              // показывает окно, ждёт OK
const name = prompt("Имя?");    // возвращает строку или null (если Отмена)
const ok = confirm("Продолжить?"); // возвращает true или false
```

## Ключевые выводы

1. `const` по умолчанию, `let` когда нужно менять, `var` — никогда
2. `typeof null === "object"` — баг JS, проверяй отдельно
3. `===` вместо `==` — всегда
4. `??` вместо `||` когда `0` и `""` — валидные значения
5. Объекты передаются по ссылке — осторожно с мутациями
