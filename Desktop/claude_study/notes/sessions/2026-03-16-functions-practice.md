## Вопросы и объяснения

### Когда i — индекс, а когда само число?
- Есть массив → i = индекс, данные = arr[i]
- Нет массива → i = само число, работаешь с i напрямую

### Лишний тернарный `? true : false`
- Операторы сравнения (`>=`, `===` и т.д.) уже возвращают boolean
- `age >= 18 ? true : false` → лишний, достаточно `age >= 18`
- Тернарный нужен когда возвращаешь НЕ boolean: `age >= 18 ? "взрослый" : "ребёнок"`
- Добавлено в учебник 02-conditions.md

### Коллбэк — передаём функцию, не значение
- `doTwice("привет")` — ошибка, строку нельзя вызвать
- `doTwice(function() { console.log("привет") })` — правильно
- fn в параметрах — это функция, вызываем через fn()

### return внутри цикла
- return завершает ВСЮ функцию, не только итерацию
- Для накопления: `result = result + str` (присваивание) внутри цикла
- `return result` — только ПОСЛЕ цикла

### Паттерн аккумулятора строки с разделителем
```js
let result = ""
for (let i = 0; i < times; i++) {
    if (i > 0) result = result + separator  // разделитель перед 2-м и далее
    result = result + str                    // слово на каждой итерации
}
```
Разделитель добавляется на СЛЕДУЮЩЕЙ итерации, перед новым словом. `i > 0` — чтобы не было лишнего разделителя в начале.

### !score vs typeof для проверки типа
- `!score` — проверка на falsy (0, "", null, undefined, NaN, false)
- `typeof score !== "number"` — проверка типа
- Для "это не число?" нужен typeof, не !

### typeof функции = "function", не "object"
- `typeof function(){}` → `"function"` — одно из исключений typeof

### Number.isNaN vs typeof для проверки числа
- `typeof value === "number"` — проверяет тип, но NaN проходит (typeof NaN === "number")
- `Number.isNaN(value)` — ловит только NaN
- Полная проверка: `typeof value === "number" && !Number.isNaN(value)`
- `Number.isInteger()` решает обе проблемы сразу (для целых чисел)

### null == undefined
- `null == undefined` → true (нестрогое)
- `null === undefined` → false (строгое, разные типы)
- typeof null === "object" — баг 1995 года

## Ключевые выводы

1. В теле цикла работай с `i`, не с `n`. Формула: n — граница, i — текущее
2. return внутри цикла — выход из функции. Для накопления — присваивание
3. Коллбэк = передать функцию, не значение. fn() — вызов, fn — передача
4. Если условие уже boolean — тернарный `? true : false` лишний
5. Для проверки типа — typeof, не ! (falsy)
