## Вопросы и объяснения

### 1. Зачем `fruits.length - 1`?
В массиве `["яблоко", "банан", "груша"]` длина = 3, но индексы 0, 1, 2. Последний индекс = длина - 1. Используется когда нужно начать перебор с конца: `for (let i = fruits.length - 1; i >= 0; i--)`.

### 2. Почему `i < arr.length`, а не `i <= arr.length`?
- `i < length` — проходит индексы 0, 1, 2 (при length=3) — все элементы
- `i <= length` — пройдёт ещё 3, а `arr[3]` = undefined (нет такого элемента)
- `<=` используется для перебора чисел (1 до 100), а не индексов

### 3. Строка — это массив?
Нет. Строка — примитив (`typeof "hello" === "string"`). Массив — объект (`typeof [] === "object"`). У них похожий синтаксис (индексы, length), но это разные типы. Строка неизменяемая, массив — изменяемый.

### 4. Точка vs скобки (user.key vs user[key])
- `user.name` — буквально ищет свойство "name"
- `user[key]` — берёт значение переменной key, подставляет
- Аналогия: точка = говоришь название ящика. Скобки = даёшь бумажку с названием
- Поэтому в `for...in` используют скобки — `key` каждый раз другой

### 5. Почему `for...of` использует const, а `for` — let?
`for...of` создаёт **новую** переменную на каждой итерации (можно const). Обычный `for` — одна переменная `i` живёт весь цикл и меняется через `i++` (нужен let).

### 6. Переворот строки через `char + result`
`result = char + result` — каждый раз создаётся новая строка, где новая буква стоит СЛЕВА от старого содержимого. Объяснено через аналогию "листок бумаги" — старый выбрасывается, на новом пишешь: новая буква + то что было.

### 7. Можно ли `if` без фигурных скобок?
Да, если после `if` одна строка: `if (n > 7) break;`. Для нескольких строк — скобки обязательны.

## Дополнительная информация

### Выбор цикла (шпаргалка)
| Ситуация | Цикл |
|----------|------|
| Пройтись по значениям массива | `for...of` |
| Нужен индекс элемента | `for` |
| Перебрать ключи объекта | `for...in` |
| Не знаешь сколько итераций | `while` |
| Хотя бы одно выполнение | `do...while` |
