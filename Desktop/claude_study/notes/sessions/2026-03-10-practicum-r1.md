## Вопросы и объяснения

### "Почему typeof function(){} возвращает "function"?"
Игорь сам нашёл ответ в учебнике. typeof делает исключение для функций — возвращает "function" хотя технически функция это object. instanceof подтверждает: `fn instanceof Object // true`.

### "while — для поиска ложного значения?"
Нет. while просто повторяет пока условие true. Назначение определяешь ты, не цикл. Когда использовать:
- for — знаешь сколько раз
- while — не знаешь сколько раз
- do...while — то же что while, но минимум 1 раз

### "Откуда arr взялось? Мы его не объявляли"
arr — параметр функции. Это "коробка" которая заполняется при вызове. `analyzeArray([0, "hello"])` → arr = [0, "hello"]. Подробно в главе 4 (Функции).

### "Вложенный if — можно ли if внутри if?"
Да. Внутри любого блока { } можно писать любой код. Внешний if работает как фильтр — внутренний выполняется только когда внешнее условие true.

### typeof / instanceof / Array.isArray — три инструмента
- typeof — "какой тип?" ("string", "number", "object")
- instanceof — "это экземпляр класса X?" ([] instanceof Array → true)
- Array.isArray() — "это массив?" (самый надёжный)

## Дополнительная информация

### Сборка строки по частям (конкатенация с накоплением)
Паттерн который встречается постоянно:
```js
let result = "";              // начальное значение
result = result + "часть1";   // "часть1"
result = result + ", часть2"; // "часть1, часть2"
result = result + ", часть3"; // "часть1, часть2, часть3"
```
Главное: всегда `result = result + ...`, никогда `result = новоеЗначение + ...` (это затрёт предыдущее).

### console.log после цикла vs внутри
```js
// Внутри — выводит на КАЖДОЙ итерации:
while (i > 0) { i -= 2; console.log(i); }  // 3, 1, -1

// После — выводит ОДИН раз финальное значение:
while (i > 0) { i -= 2; }
console.log(i);                              // -1
```
