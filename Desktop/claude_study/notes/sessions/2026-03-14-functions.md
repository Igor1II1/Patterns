## Вопросы и объяснения

### 1. Когда hoisting нежелателен?
Вопрос: в учебнике написано "когда hoisting нежелателен" — есть ли примеры?
Ответ: Function Declaration поднимается наверх, и можно вызвать до объявления. В больших файлах это создаёт путаницу:
- Видишь вызов calculate(10, 20) на строке 5, а определение на строке 150
- С Function Expression код читается как книга: сначала определил, потом вызвал
- Также для условного создания функций (разная логика в if/else) и передачи как аргумент

### 2. Что такое this (кратко)?
this — ссылка на объект, от имени которого вызвана функция. Как "я" — зависит от кто говорит.
user.sayHi() → this = user (то что ПЕРЕД точкой).

### 3. Что такое коллбэк?
Коллбэк — функция, переданная как аргумент в другую функцию. Та вызовет её когда нужно.
Аналогия: оставляешь номер телефона в ресторане, тебе перезвонят когда столик свободен.

### 4. Передача объектов в функции — ключевое объяснение
Примитивы (числа, строки) — передаётся КОПИЯ. Функция меняет копию, оригинал цел.
Объекты — передаётся ДУБЛИКАТ КЛЮЧА от комнаты. Оба ключа открывают одну дверь.
- obj.age = 25 — зашёл в комнату, повесил табличку → user тоже видит
- obj = {} — выбросил свой ключ, получил новый → оригинальная комната цела

### 5. Что такое rest-параметр (...args)?
Три точки перед именем = собери все аргументы в массив.
Аналогия: стоишь с коробкой и говоришь "кидайте сколько хотите" — все мячи в коробке.
sum(1,2,3) → numbers = [1, 2, 3]

### 6. Разница rest и spread
- ...в параметрах функции = REST (собрать в массив)
- ...при вызове/в массиве = SPREAD (развернуть из массива)
first(...nums) → first(10, 20, 30) → args[0] = 10
first(nums) → first([10,20,30]) → args[0] = [10,20,30] (весь массив как один аргумент)

### 7. Механика коллбэков пошагово
repeat(3, i => console.log(i)):
1. n = 3, action = стрелочная функция
2. Цикл i=0: action(0) → console.log(0)
3. Цикл i=1: action(1) → console.log(1)
4. Цикл i=2: action(2) → console.log(2)
Ключевое: action — обычная переменная, в которой лежит функция. Вызов через ().

### 8. Параметр vs аргумент
- Параметр = имя-заглушка в определении (name)
- Аргумент = реальное значение при вызове ("Игорь")

### 9. Четыре "вида" стрелочных функций
Это не четыре отдельных вида, а одна функция с сокращением записи:
- 0 параметров → () =>
- 1 параметр → n => (без скобок)
- 2+ параметра → (a, b) =>
- 1 выражение → без {}, без return
- 2+ строк → с {}, с return
