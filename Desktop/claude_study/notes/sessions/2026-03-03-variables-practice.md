## Вопросы и объяснения

### Что такое конкатенация строк?

**Конкатенация** (от лат. concatenatio — сцепление) — склейка строк в одну.

В JS оператор `+` при работе со строками не складывает, а склеивает:

```js
"Привет" + " " + "мир"   // "Привет мир"
"Возраст: " + 25          // "Возраст: 25" — число стало строкой
```

Если **хотя бы один** операнд — строка, `+` переключается в режим конкатенации и преобразует второй операнд в строку.

```js
const a = "5";
const b = 2;
console.log(a + b); // "52" — не 7! "5" + 2 → "52"
console.log(a - b); // 3  — вычитание конвертирует в числа
console.log(a * b); // 10 — умножение конвертирует в числа
```

Правило: `+` — единственный оператор который "заражается" строкой. Все остальные (`-`, `*`, `/`, `%`) конвертируют в число.

### null — примитив или объект?

`null` — это **примитивный тип**. Несмотря на то что `typeof null` возвращает `"object"` — это историческая ошибка языка (баг из 1995 года).

```js
typeof null // "object" ← ложь! Это баг, не факт
```

null — примитив, как string, number, boolean. Означает "значения нет" — явное, намеренное отсутствие значения.

Это зафиксировано в спецификации JS как известная ошибка которую нельзя исправить без поломки старого кода.

### Что такое !! (двойное отрицание)?

`!!` — два оператора `!` подряд. Превращает любое значение в его булевый эквивалент (`true` или `false`).

```js
!!null     // false (null — falsy)
!!27       // true  (число — truthy)
!!"привет" // true  (строка — truthy)
!!0        // false (0 — falsy)
!!""       // false (пустая строка — falsy)
```

Пошагово: `!!null` → `!null` = `true` → `!true` = `false`

Аналог: `Boolean(null)` — делает то же самое, но `!!` короче.

```js
Boolean(null)  // false
!!null         // false — эквивалентно
```

Формула-якорь: `!!` = "является ли это значение истинным?"

### Почему 4+2+4 — не тернарный оператор?

Ошибка в задании 1.4: написал `4+2+4` как пример тернарного оператора.

**Тернарный** — строго оператор `? :`. Он **один** в JS. Принимает три операнда:
```
условие ? значение_если_true : значение_если_false
```

`4+2+4` — это два **бинарных** оператора `+` применённых подряд:
- `4 + 2` → бинарный, два операнда
- `6 + 4` → ещё один бинарный, два операнда

Три числа в выражении ≠ тернарный оператор.

Правильный пример тернарного:
```js
age >= 18 ? "взрослый" : "маленький"
```

### 1.2 — typeof и преобразование типов ✅
```js
typeof null           // "object" — историческая ошибка
typeof []             // "object" — массив это объект
typeof function(){}   // "function"
typeof undefined      // "undefined"
typeof NaN            // "number"

"5" + 2  // "52" — конкатенация
"5" - 2  // 3   — числовое вычитание
"5" * 2  // 10  — числовое умножение
```

### 1.3 — Hoisting ✅
```js
console.log(x); // undefined — var поднят, но без значения
console.log(y); // ReferenceError — let в TDZ

var x = 10;
let y = 20;

sayHi();  // "Привет" — function declaration поднимается полностью
sayBye(); // ReferenceError — const в TDZ

function sayHi() { console.log("Привет"); }
const sayBye = function() { console.log("Пока"); };
```

### 1.4 — Виды операторов ⚠️
```js
typeof "hello"  // унарный (1 операнд)
5 + 3           // бинарный (2 операнда)
-10             // унарный (1 операнд, отрицание)
!false          // унарный (1 операнд)
a > b ? "да" : "нет" // тернарный (3 операнда)
x++             // унарный (1 операнд)
10 % 3          // бинарный (2 операнда)
!!null          // унарный (оператор ! применён дважды)
```
Собственные примеры:
```js
// тернарный:
age >= 18 ? "взрослый" : "маленький"
// бинарный:
10 * 2
// унарный:
-8
```
