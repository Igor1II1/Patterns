### 1.5 — Неявное преобразование типов
Тема: как JS автоматически конвертирует типы в разных контекстах.
Ключевое: `+` со строкой = конкатенация; `-`, `*`, `/` = конвертация в число.

### 1.6 — `??` vs `||`
```js
function getDisplayName(user) {
    return user.nickname ?? user.email ?? "Аноним"
}
```
Ключевое: `||` заменяет любое falsy (включая `""`, `0`, `false`). `??` заменяет только `null`/`undefined`.
Правило: если `0`, `false`, `""` — валидные значения → используй `??`.

### 1.7 — `??=` (нулевое присваивание)
```js
function initSettings(settings) {
    settings.theme ??= "light";
    settings.language ??= "ru";
    settings.fontSize ??= 14;
    settings.notifications ??= true;
    return settings;
}
```
Ключевое: `??=` присваивает значение ТОЛЬКО если переменная `null` или `undefined`.
`||=` заменил бы `0` и `false` (falsy). `??=` — нет.

### 1.8 — `getType()` (typeof + исключения)
```js
function getType(value) {
    if (value === null) return "null";
    if (Array.isArray(value)) return "array";
    return typeof value;
}
```
Ключевое: `typeof` врёт в двух случаях — `null → "object"` и `[1,2,3] → "object"`.
Порядок важен: сначала проверь `null`, потом массив, потом `typeof`.

### 1.9 — `==` vs `===` + `checkNaN`
```js
console.log(0 == false);        // true — приводит типы к числу
console.log(0 === false);       // false — разные типы
console.log(null == undefined); // true — особое правило JS
console.log(NaN == NaN);        // false — NaN ≠ NaN (единственное исключение)

function checkNaN(value) {
    return Number.isNaN(value);
}
```
Ключевое: `===` всегда предсказуем. `==` конвертирует типы перед сравнением.
NaN единственная вещь в JS которая не равна сама себе.

### 1.10 — `isValidAge()` (валидация числа)
```js
function isValidAge(value) {
    if (Number.isInteger(value) && value >= 0 && value <= 150) {
        return true;
    } else {
        return false;
    }
}
// Или короче: return Number.isInteger(value) && value >= 0 && value <= 150;
```
Ключевое: `Number.isInteger()` проверяет и тип и целостность одновременно.
Диапазон — только через `&&`: `0 < value <= 150` в JS НЕ работает как в математике.

## Вопросы и объяснения

### Почему `typeof NaN === "number"`?

`NaN` означает "Not a Number" но является ЗНАЧЕНИЕМ типа `number`. Это исторически: операция вроде `"abc" * 2` должна вернуть что-то типа number — но результат невалиден. Поэтому JS возвращает `NaN` — специальное числовое значение означающее "ошибочный результат числовой операции".

```js
typeof NaN      // "number" ← NaN это значение типа number
NaN === NaN     // false   ← единственное значение в JS не равное себе
Number.isNaN(NaN) // true  ← правильная проверка на NaN
```

### Почему `0.1 + 0.2 !== 0.3`?

Числа в JS хранятся в формате IEEE 754 (64-битное число с плавающей точкой). Некоторые десятичные дроби нельзя точно представить в двоичном виде — как `1/3` нельзя записать точно в десятичном.

```js
0.1 + 0.2  // → 0.30000000000000004
// Решение для сравнения:
Math.abs(0.1 + 0.2 - 0.3) < 0.0000001  // true
// Решение для отображения:
(0.1 + 0.2).toFixed(2)  // "0.30"
```

### `Number.isNaN()` vs глобального `isNaN()`

| | `Number.isNaN(x)` | `isNaN(x)` |
|---|---|---|
| Что делает | Строго: `true` только если значение = `NaN` | Сначала конвертирует в число, потом проверяет |
| `isNaN("abc")` | `false` (строка ≠ NaN) | `true` (конвертирует "abc" → NaN) |
| `isNaN("")` | `false` | `false` ("" → 0) |
| Рекомендуется | ✅ Да | ❌ Нет — неожиданные результаты |

```js
Number.isNaN("abc")  // false — строка, не NaN
isNaN("abc")         // true  — "abc" → NaN → true (неожиданно!)

Number.isNaN(NaN)    // true  ✅
isNaN(NaN)           // true  ✅ (тут совпадают)
```

### Что такое примитив?

Примитив — это простое значение, хранящееся и копируемое напрямую. В JS 7 примитивных типов:

```
string     → "hello"
number     → 42, 3.14, NaN, Infinity
boolean    → true, false
null       → null
undefined  → undefined
bigint     → 9007199254740991n
symbol     → Symbol("id")
```

Всё остальное — объекты (Object, Array, Function, Map, Set...).
Разница ключевая: примитивы копируются по значению, объекты — по ссылке.

### Почему `0 < value <= 150` не работает в JS?

JS вычисляет слева направо:
```
0 < value <= 150
→ (0 < value) <= 150
→ true <= 150  (или false <= 150)
→ 1 <= 150     (true конвертируется в 1)
→ true          (всегда!)
```

Правильно:
```js
value >= 0 && value <= 150
```

## Ключевые выводы

1. `??` и `||` — разные операторы. Выбор зависит от того, что считается "пустым" значением.
2. `NaN` — это число. Проверять через `Number.isNaN()`, не через `=== NaN` и не через `isNaN()`.
3. `typeof` врёт для `null` и массивов — всегда проверяй их вручную ДО `typeof`.
4. `===` всегда лучше `==`. Исключение — `null == undefined` в старом коде.
5. Диапазоны в JS — только через `&&`. Математическая нотация не работает.
6. Примитивы = простые значения. Объекты = ссылки. Это важно при копировании.

## Дополнительная информация

### Паттерн "ранний возврат" вместо if/else
```js
// Длинно (но работает):
function isValidAge(value) {
    if (condition) return true;
    else return false;
}

// Короче — вернуть само условие:
function isValidAge(value) {
    return Number.isInteger(value) && value >= 0 && value <= 150;
}
```
Условие `&&` уже возвращает `true` или `false` — `if/else` не нужен.

### Цепочка `??` для fallback-значений
```js
// Классический паттерн: попробовать несколько источников по приоритету
const name = user.nickname ?? user.email ?? user.phone ?? "Аноним";
```
Читается как: "возьми nickname, если нет — email, если нет — phone, если нет — 'Аноним'".
