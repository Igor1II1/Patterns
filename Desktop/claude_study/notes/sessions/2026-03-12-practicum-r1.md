## Вопросы и объяснения

### typeof + Number.isNaN -- зачем два фильтра?
typeof ловит нечисловые типы ("отлично" -> "string" !== "number").
Но typeof NaN === "number" (ловушка JS!) -- поэтому нужен Number.isNaN отдельно.
Два фильтра работают в паре: typeof для строк, Number.isNaN для NaN.

### Закон де Моргана
При инвертировании условий операторы меняются местами:
- !(A && B) = !A || !B
- !(A || B) = !A && !B

Практическое применение: "валидный email = includes('@') && includes('.')"
Инвертируем для ошибки: "невалидный = !includes('@') || !includes('.')"

### Паттерн необязательного фильтра
```js
if (filters.minPrice !== undefined && el.price < filters.minPrice) continue;
```
Часть 1: "фильтр задан?" (undefined -> false -> пропуск всей проверки)
Часть 2: "товар подходит?" (проверяется только если фильтр задан)
false && (что угодно) = false -> continue не срабатывает -> товар проходит

## Дополнительная информация

### Паттерны из R1 (запомнить)
- **Аккумулятор**: `total += el.amount` (переменная копит значения в цикле)
- **Поиск максимума**: `if (el.amount > maxAmount)` (сравниваем с текущим рекордом)
- **Необязательный фильтр**: `!== undefined && условие` (проверяй только если задано)
- **Continue для фильтрации**: continue пропускает СОВПАВШЕЕ, оставляет остальное
- **Независимые if**: для фильтров. **if/else if**: для взаимоисключающих вариантов
