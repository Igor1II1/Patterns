# Как проходит каждая сессия

---

## Старт сессии

Напиши: **"начинаем, 14:30"** — с текущим временем

Claude прочитает все нужные файлы сам (включая `repetition-schedule.md`) и скажет:
- что проходили в прошлый раз
- что нужно повторить СЕГОДНЯ (по расписанию интервалов)
- что делаем сегодня
- сколько часов уже в обучении

> Если время не указал — Claude спросит сам.

---

## Структура занятия (обычная сессия — новая тема)

```
1. ПРОГРЕВ (10-15 мин) — Слои 1 + 4
   ├── 3 карточки-минимумы по расписанию из repetition-schedule.md
   │   (выбираются по интервалам: +1д, +3д, +7д, +14д, +30д)
   └── 2-3 теоретических вопроса по прошлой теме

2. ТЕОРИЯ (30-40 мин)
   └── Читаешь главу учебника (js/*.md)
   └── Claude объясняет непонятные места
   └── Задаёшь вопросы

3. ТЕОРЕТИЧЕСКИЙ ОПРОС (10-15 мин)
   └── Claude задаёт вопросы по НОВОЙ теме
   └── Ты отвечаешь как на собеседовании
   └── Результаты → в статистику

4. ПРАКТИКА (60-90 мин) — Слои 2 + 3
   ├── Основные задания по новой теме (🟢 → 🟡 → 🔴)
   │   Каждое задание вплетает 1-2 концепции из прошлых тем (Слой 2)
   ├── Последние 2-3 задания = микс с прошлыми темами (Слой 3: 70/30)
   ├── Claude сверяет чеклист покрытия (все концепции главы)
   ├── Дополнительные задания на пробелы (если чеклист не 100%)
   └── КОМПЛЕКСНАЯ ЗАДАЧА — всё из темы в одном задании

5. ПРОВЕРКА ПОКРЫТИЯ
   └── Claude показывает чеклист → 100%? Переходим. Нет? Ещё задания.

6. ИТОГИ (5 мин)
   └── Говоришь "закрываем"
   └── Claude обновляет все файлы + repetition-schedule.md
   └── Показывает статистику сессии
```

---

## Структура сессии-практикума (каждые 3 темы — Слой 5)

```
1. ПОЛНЫЙ ЧЕКЛИСТ
   └── Claude показывает ВСЕ концепции из ВСЕХ пройденных тем

2. ТЕОРЕТИЧЕСКИЙ ОПРОС (15-20 мин)
   └── Вопросы по ВСЕМ пройденным темам (не только слабые)

3. КАРТОЧКИ-МИНИМУМЫ (20-30 мин)
   └── Микро-задания по каждой концепции из чеклиста
       Быстро, 2-3 минуты каждое

4. КОМПЛЕКСНЫЕ ЗАДАЧИ (60-90 мин)
   └── 3-5 больших заданий, каждое комбинирует концепции из РАЗНЫХ тем

5. БАГФИКСЫ (20-30 мин)
   └── Код с ошибками из всех тем — найди и исправь

6. ПРОВЕРКА ПОКРЫТИЯ
   └── Все концепции в чеклисте покрыты? → ✅ или ещё задания

7. ИТОГИ
```

---

## Структура еженедельного микса (Слой 4)

Последний учебный день недели (обычно суббота), 30-40 минут:

```
ЕЖЕНЕДЕЛЬНЫЙ МИКС
├── 5-7 заданий из РАЗНЫХ тем вперемешку
├── Задания НЕ по порядку тем (случайный микс)
├── Акцент на темы которые давно не повторялись
└── Результат → обновить repetition-schedule.md
```

---

## 5 слоёв повторения — как они работают вместе

```
┌─────────────────────────────────────────────────────────┐
│ СЛОЙ 1: Ежедневный прогрев (каждая сессия, 10-15 мин)  │
│   3 карточки по расписанию интервалов                   │
│   +1д → +3д → +7д → +14д → +30д → ЗАКРЕПЛЕНО           │
├─────────────────────────────────────────────────────────┤
│ СЛОЙ 2: Вплетение в новые задания (каждая сессия)       │
│   Каждое задание использует 1-2 концепции из прошлого   │
├─────────────────────────────────────────────────────────┤
│ СЛОЙ 3: Чередование 70/30 (каждая сессия)               │
│   70% новая тема + 30% микс из прошлых                  │
├─────────────────────────────────────────────────────────┤
│ СЛОЙ 4: Еженедельный микс (1 раз в неделю)              │
│   5-7 заданий из всех пройденных тем вперемешку         │
├─────────────────────────────────────────────────────────┤
│ СЛОЙ 5: Практикум (каждые 3 темы)                       │
│   100% покрытие ВСЕХ концепций всех пройденных тем      │
└─────────────────────────────────────────────────────────┘
```

## Интервалы повторения

```
После изучения темы:
  +1 день   → 2 карточки (свежее закрепление)
  +3 дня    → 1 карточка (первый тест памяти)
  +7 дней   → 1 карточка (недельная проверка)
  +14 дней  → 1 карточка (двухнедельная проверка)
  +30 дней  → 1 карточка (месячная проверка)
  Дальше    → только в еженедельных миксах и практикумах

Если ошибся при повторении:
  1-2 ошибки  → назад на 1 интервал (14д → 7д)
  3+ ошибок   → сброс к +1 день
```

## График теоретических проверок

| Момент | Что происходит | Сколько вопросов |
|--------|---------------|-----------------|
| **Начало каждой сессии** | Прогрев по расписанию (repetition-schedule.md) | 3 карточки + 2-3 вопроса |
| **После изучения новой темы** | Полный опрос по теме | Все вопросы темы |
| **Последний день недели** | Еженедельный микс | 5-7 заданий из разных тем |
| **Каждые 3 темы** | Практикум с 100% покрытием | Все концепции |
| **Конец Блока 0** | Финальный экзамен по всем 10 темам | Все вопросы |

## Ключевые фразы

| Ты пишешь | Claude делает |
|-----------|--------------|
| `"начинаем, 14:30"` | Читает файлы + repetition-schedule, фиксирует время, говорит план |
| `"готово"` | Проверяет твой код |
| `"проверь теорию"` | Полный опрос по текущей теме |
| `"проверь теорию по [теме]"` | Опрос по конкретной теме |
| `"сводный опрос"` | 10–15 вопросов по всем пройденным темам |
| `"микс"` | Еженедельный микс — задания из разных тем |
| `"закрываем, 17:00"` | Обновляет все файлы + repetition-schedule + итоги |
| `"трекер"` | Показывает статистику из time-tracker.md |

## Как проходит теоретический опрос

```
Claude: Что такое замыкание?

Ты: [отвечаешь своими словами, как на собеседовании]

Claude: ✅ Верно — назвал главное.
        Добавлю один нюанс: ...

        Следующий вопрос: ...
```

Оценки:
- ✅ Ответил полно и точно
- ⚠️ Ответил частично (не хватило деталей)
- ❌ Не знал / ответил неверно
- 💬 Попросил подсказку

Всё фиксируется в `progress.md`.

---

## Как задавать вопросы

**Хорошо:**
- "Почему стрелочная функция не имеет своего this?"
- "В каком случае мне нужен reduce вместо forEach?"
- "Объясни почему мой код не работает" + показываешь код

**Не очень:**
- "Объясни всё про функции" — слишком широко
- "Это правильно?" без показа кода

## Что Claude обновляет в конце сессии

После **"закрываем, 17:00"**:

1. `progress.md` — теория (X/Y ✅) + практика (X/Y ✅) + % темы
2. `errors-journal.md` — конкретные ошибки этой сессии
3. `profile.md` — пройденные темы, слабые места
4. `theory-tests.md` — новые вопросы если нужно
5. `time-tracker.md` — время сессии, задания, КПД, общая статистика
6. `learning-profile.md` — анализ мышления и стиля обучения
7. `sessions/ГГГГ-ММ-ДД-тема.md` — конспект сессии (вопросы, примеры, выводы)
8. Учебник (`js/*.md` и др.) — если были доп. объяснения или неточности
9. `repetition-schedule.md` — обновить интервалы, отметить выполненные повторения

Итоги в чате:
```
📊 Итоги сессии — [Тема]

⏱️ Время:    X.X ч (14:30 — 17:00)
Теория:      X/Y вопросов (Z%)
Практика:    X/Y заданий (Z%)
КПД:         🟢 высокий

Слабые места: ...
Следующий раз: ...
Повторения выполнены: 0.1 (+14д ✅), 0.2 (+7д ✅)

Уровень: 🟡 X%
Всего часов в обучении: X.X ч
```

## Темп обучения

- Одна тема за сессию — нормально
- Не переходи к следующей теме пока чеклист покрытия не 100%
- Если что-то непонятно — спрашивай снова, другими словами
- Практика = 70% новое + 30% старое (всегда)
