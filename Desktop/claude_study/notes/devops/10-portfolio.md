## Зачем эта тема существует?

Ты написал код, задеплоил приложение, настроил CI/CD. Но работодатель об этом не знает. Портфолио — это мост между твоими навыками и трудоустройством. Хорошо оформленный GitHub-профиль, грамотное резюме и уверенное поведение на собеседовании — это то, что отличает кандидата, которого берут, от кандидата, которого не берут.

## README для проекта

### Что это такое?

README — первое, что видит работодатель, открывая твой репозиторий. У тебя есть 10 секунд, чтобы произвести впечатление. Хороший README показывает: что это за проект, какой стек, как запустить.

Fullstack-приложение для управления задачами в стиле Trello.


## GitHub-профиль

### Что это такое?

GitHub-профиль — это твое портфолио разработчика. Работодатели смотрят его перед собеседованием.

### Как работает?

```markdown
# Чеклист GitHub-профиля

- [ ] Фото (не аниме, нормальное фото)
- [ ] Bio: "Junior Fullstack Developer | Next.js, TypeScript, PostgreSQL"
- [ ] Pinned repositories: 3-6 лучших проектов
- [ ] README профиля (username/username репозиторий)
- [ ] Зеленые квадратики (contribution graph) — регулярная активность
- [ ] Никаких пустых репозиториев или "test123"
```

### README профиля

```markdown
# Привет, я Игорь

Junior Fullstack Developer.
Учусь, пишу код, строю проекты.

## Стек

- JavaScript / TypeScript
- React, Next.js
- Node.js, Prisma
- PostgreSQL
- Docker, Git

## Проекты

| Проект | Описание | Стек |
|--------|---------|------|
| [TaskBoard](https://github.com/...) | Доска задач | Next.js, Prisma, PostgreSQL |
| [WeatherApp](https://github.com/...) | Приложение погоды | React, API integration |
| [Portfolio](https://github.com/...) | Личный сайт | Next.js, Tailwind |

## Контакты
```

### Что должно быть в проектах

```
Минимум 3 проекта для Junior:

1. Fullstack-проект (главный)
   - Next.js + Prisma + PostgreSQL
   - Авторизация
   - CRUD-операции
   - Деплой на Vercel
   - Это твой "звездный" проект

2. Frontend-проект
   - React/Next.js
   - Работа с API
   - Адаптивный дизайн
   - Показывает навыки верстки и UI

3. Вспомогательный проект
   - Может быть простым
   - Показывает разнообразие навыков
   - CLI-инструмент, бот, библиотека
```

### Частые заблуждения

- "Количество репозиториев важнее качества" — нет, 3 хороших проекта лучше 30 пустых.
- "Форки считаются за проекты" — нет, работодатель видит что это форк.

```
Системно подхожу к обучению, разбираюсь в "почему", а не только "как".
```

### Правила резюме

```
1. Одна страница (максимум две)
2. Проекты > образование (для Junior без опыта)
3. Каждый проект — 2-3 пункта с конкретикой
4. Навыки — только те, которые можешь подтвердить на собеседовании
5. Никакой воды: "ответственный, коммуникабельный, стрессоустойчивый"
6. Ссылки на GitHub и живые демо обязательны
```

### Частые заблуждения

- "Нужно указать все технологии, которые хотя бы видел" — нет, указывай только то, в чем уверен. На собеседовании спросят.
- "Без опыта работы резюме бесполезно" — нет, проекты заменяют опыт для Junior. Работодатель хочет видеть, что ты умеешь делать.

### Мини-проверка

1. Что важнее в резюме Junior — опыт или проекты?
2. Сколько страниц должно быть в резюме?
3. Какие "навыки" не стоит указывать?

## Подготовка к собеседованию

### Что это такое?

Собеседование Junior-разработчика обычно включает: разговор о проектах, теоретические вопросы и live-coding задачу.

### Типичные вопросы

```
JavaScript:
- Чем let отличается от const и var?
- Что такое замыкание? Приведи пример.
- Как работает this?
- Что такое промисы? Чем async/await отличается от .then()?
- Что такое event loop?
- Как работает прототипное наследование?

React:
- Что такое виртуальный DOM?
- Чем useState отличается от useRef?
- Когда использовать useEffect?
- Что такое key в списках и зачем он нужен?
- Как избежать лишних ре-рендеров?
- Server Components vs Client Components в Next.js?

Backend:
- Что такое REST API? Какие HTTP-методы знаешь?
- Чем 401 отличается от 403?
- Что такое SQL-инъекция и как от нее защититься?
- Зачем нужны индексы в базе данных?
- Что такое ORM?
- Как работает JWT?

Общее:
- Расскажи про свой проект. Что было сложнее всего?
- Как бы ты реализовал [функционал]?
- Что такое Git? Какие команды используешь?
- Что такое Docker и зачем он нужен?
```

### Как отвечать на вопросы

```
Хорошо:
"Замыкание — это когда функция запоминает переменные из внешней
области видимости, даже после того как внешняя функция завершилась.
Например, в счетчике:

function createCounter() {
  let count = 0;
  return () => ++count;
}

Функция createCounter вернула внутреннюю функцию, и та помнит
переменную count. Это полезно для инкапсуляции состояния."

Плохо:
"Это когда переменная доступна из другой функции."
— Слишком размыто, без примера, без понимания "зачем".
```

### Live-coding советы

```
1. Проговаривай мысли вслух
   - "Сначала я хочу разобрать задачу..."
   - "Я думаю использовать массив потому что..."

2. Начни с простого решения, потом оптимизируй
   - Brute-force решение лучше, чем никакого

3. Тестируй граничные случаи
   - Пустой массив, null, отрицательные числа

4. Не бойся спрашивать
   - "Правильно ли я понимаю что входные данные..."
   - "Нужно ли обрабатывать случай когда..."

5. Если застрял — скажи
   - "Я знаю что нужно сделать X, но не помню точный синтаксис.
     Если бы мог загуглить — нашел бы за 30 секунд."
```

### Частые заблуждения

- "На собеседовании нужно знать все" — нет, Junior не знает все. Важнее показать мышление и способность учиться.
- "Live-coding — про идеальный код" — нет, это про мышление. Говори вслух, показывай как решаешь.
- "Нужно скрывать что учился самостоятельно" — наоборот, самообучение показывает мотивацию и дисциплину.

### Мини-проверка

1. Что важнее на собеседовании — идеальный ответ или ход мыслей?
2. Как отвечать если не знаешь ответ?
3. Какие три типа вопросов обычно на собеседовании Junior?

## Презентация проекта

### Что это такое?

На собеседовании часто просят рассказать о проекте. Это твой шанс показать навыки и подход к решению задач.

### Как работает?

```
Структура рассказа о проекте (2-3 минуты):

1. ЧТО это (10 секунд)
   "Я сделал доску управления задачами — аналог Trello."

2. ЗАЧЕМ (10 секунд)
   "Чтобы отработать fullstack-разработку: авторизацию, REST API,
   работу с базой данных."

3. СТЕК (15 секунд)
   "Next.js, TypeScript, Prisma с PostgreSQL, авторизация через
   NextAuth.js, деплой на Vercel."

4. ЧТО ИНТЕРЕСНОГО (30-60 секунд)
   "Самым сложным было реализовать drag-and-drop с обновлением
   порядка задач в базе. Использовал транзакции Prisma, чтобы
   атомарно обновлять порядок в двух колонках."

5. ЧТО БЫ УЛУЧШИЛ (15 секунд)
   "Сейчас бы добавил WebSocket для real-time обновлений и
   покрыл бы тестами критические пути."
```

### Что работодатель хочет услышать

```
- Понимание принятых решений (почему Prisma, а не Sequelize?)
- Осознание трудностей (что было сложно, как решил)
- Критический взгляд (что бы сделал иначе)
- Технический словарь (говоришь на языке разработчика)
```

## Итог

| Этап | Что делать |
|------|-----------|
| GitHub | 3+ проекта с README, скриншотами, демо |
| Резюме | 1 страница: навыки, проекты с конкретикой |
| Теория | Подготовить ответы на типичные вопросы |
| Live-coding | Практиковать задачи, говорить вслух |
| Проект | Уметь рассказать за 2-3 минуты |

Формула успеха Junior-разработчика:
Проекты (показывают навыки) + README (показывают профессионализм) + Собеседование (показывают мышление) = Оффер
