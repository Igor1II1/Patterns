# Проект 1: Библиотека книг

**Месяц:** Март 2026
**Папка:** `projects/month-1-library/`
**Срок:** Последняя неделя марта (после изучения всех тем блоков 0 и 1)

---

## Описание

Одностраничное веб-приложение для управления личной библиотекой. Данные хранятся в localStorage — приложение работает без сервера.

---

## Функциональность

### Обязательно (MVP)
- [ ] Добавить книгу: название, автор, год, жанр, статус (прочитана / читаю / хочу)
- [ ] Список всех книг в виде карточек
- [ ] Удалить книгу
- [ ] Поиск в реальном времени (по названию и автору)
- [ ] Фильтр по статусу (все / прочитана / читаю / хочу)
- [ ] Хранение в localStorage (данные сохраняются после перезагрузки)

### Дополнительно (если успеешь)
- [ ] Редактирование книги
- [ ] Сортировка (по году, по автору, по названию)
- [ ] Фильтр по жанру
- [ ] Счётчик книг по статусам
- [ ] Пустое состояние (когда список пуст — красивая заглушка)

---

## Технический стек

- **Только ванильный JS** — без фреймворков
- HTML + CSS (любой подход: flexbox, grid)
- localStorage для хранения данных

## Поэтапная разработка

### Этап 1: Структура и хранение (день 1)
1. Создать `index.html` с базовой разметкой (форма + список)
2. Написать класс `Book`
3. Написать класс `Library` с `add`, `remove`, `save`, `load`
4. Проверить что книги сохраняются и загружаются

### Этап 2: Отрисовка (день 2)
1. Написать `render.js` — функция `renderBooks(books)`
2. Отображать книги как карточки
3. Кнопка удаления работает
4. Форма добавления работает

### Этап 3: Поиск и фильтры (день 3)
1. Поиск при вводе (`input` событие)
2. Фильтр по статусу (кнопки или select)
3. Поиск и фильтр работают вместе

### Этап 4: Полировка (день 4)
1. Стили: аккуратные карточки, удобная форма
2. Валидация формы (не отправлять пустые поля)
3. Пустое состояние (заглушка когда книг нет)
4. Тестирование всех сценариев
