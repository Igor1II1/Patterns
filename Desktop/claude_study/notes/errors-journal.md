## JavaScript: Переменные — продвинутые операторы и валидация

### Математическая нотация диапазона (2026-03-04) 🔁 типичная ошибка новичка
Контекст: задание 1.10 — проверить что value находится в диапазоне 0..150.
```js

                              // (0 < value) даёт true/false, потом true <= 150 = всегда true!

```
> Вывод: в JS нет цепочки сравнений как в математике. Всегда пиши два условия через `&&`.

### Неполный if без else (2026-03-04)
Контекст: задание 1.10 — функция isValidAge возвращала `undefined` вместо `false`.
```js

     // забыли else — возвращает undefined для ложных случаев!
   }

✅ function isValidAge(value) {
     if (condition) return true;
     else return false;
     // или короче: return condition;  ← condition уже true/false
   }
```
> Вывод: если функция должна возвращать значение — убедись что КАЖДАЯ ветка возвращает что-то. Или сразу пиши `return условие;` — условие уже даёт true/false.
