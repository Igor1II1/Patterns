# Git: Отмена изменений — reset, revert, stash

← [[git/04-github-workflow]] | ← [[git-fundamentals]]

---

## Карта инструментов

Перед использованием — определи в какой стадии находятся изменения:

```
Рабочая папка → Staging → Репозиторий → Remote
     ↑               ↑          ↑           ↑
git restore    git restore  git reset   git revert
               --staged     git revert
```

---

## Ситуация 1: Хочу отменить изменения в файле (не в staging)

```bash
# Файл изменён но НЕ добавлен в staging (git add не делал)
git restore index.js

# Восстановить ВСЕ изменённые файлы
git restore .
```

**Осторожно:** это необратимо. Несохранённые изменения пропадут.

---

## Ситуация 2: Файл добавлен в staging (git add сделан)

```bash
# Убрать из staging (изменения в файле сохранятся)
git restore --staged index.js

# Убрать всё из staging
git restore --staged .

# После этого изменения снова в рабочей папке
# Можно добавить снова или отменить через git restore
```

---

## Ситуация 3: Хочу изменить последний коммит

Только если коммит **ещё не запушен**!

```bash
# Изменить сообщение последнего коммита
git commit --amend -m "Новое правильное сообщение"

# Добавить забытый файл в последний коммит
git add forgotten.js
git commit --amend --no-edit
```

## Ситуация 4: Хочу отменить последние N коммитов

### git reset — переместить HEAD назад

```bash
# --soft: коммит отменён, изменения остаются в staging
git reset --soft HEAD~1

# --mixed (по умолчанию): коммит отменён, изменения в рабочей папке
git reset --mixed HEAD~1
# или просто:
git reset HEAD~1

# --hard: коммит отменён, изменения УДАЛЕНЫ (опасно!)
git reset --hard HEAD~1
```

`HEAD~1` = 1 коммит назад, `HEAD~3` = 3 коммита назад.

**Когда что использовать:**
| Вариант | Коммит | Staging | Файлы | Когда |
|---------|--------|---------|-------|-------|
| `--soft` | ❌ | ✅ (остаются) | ✅ | Переделать коммит |
| `--mixed` | ❌ | ❌ | ✅ | Переделать код и коммит |
| `--hard` | ❌ | ❌ | ❌ | Полностью выбросить |

**Правило:** `--hard` только для незапушенных коммитов. После `--hard` восстановление сложно.

## Ситуация 5: Нужно отменить запушенный коммит

`git reset` переписывает историю — нельзя делать для запушенных коммитов (сломает синхронизацию с remote).

Вместо этого — `git revert`:

```bash
# Создаёт НОВЫЙ коммит, который отменяет указанный
git revert HEAD         # отменить последний коммит
git revert a3f2c1d     # отменить конкретный коммит

# Без автоматического открытия редактора
git revert HEAD --no-edit
```

`git revert` безопасен — история сохраняется, просто добавляется коммит-отмена.

```
Before: A → B → C (C содержит баг)
After:  A → B → C → D (D отменяет C)
```

## git stash — убрать работу на потом

Нужно срочно переключиться на другую задачу, но текущие изменения не готовы к коммиту.

```bash
# Убрать текущие изменения в stash
git stash

# Убрать с описанием
git stash push -m "Работа над поиском, не закончена"

# Список stash'ей
git stash list
# stash@{0}: On feature: Работа над поиском, не закончена
# stash@{1}: On main: Временные эксперименты

# Восстановить последний stash и удалить его
git stash pop

# Восстановить конкретный stash
git stash pop stash@{1}

# Восстановить без удаления из stash
git stash apply

# Удалить stash без восстановления
git stash drop stash@{0}

# Очистить все stash'и
git stash clear
```

### Типичный сценарий

```bash
# Работаешь над поиском...
# Менеджер: "Срочный баг в продакшене!"

git stash push -m "WIP: поиск по жанрам"
git switch main
git switch -c fix/critical-bug

# Чинишь баг, коммитишь, пушишь...

git switch feature/search
git stash pop
# Продолжаешь работать
```

## Восстановление "потерянных" коммитов

Даже после `git reset --hard` можно восстановить коммиты через `reflog`:

```bash
# История всех операций HEAD (включая reset, checkout, merge)
git reflog

# a3f2c1d HEAD@{0}: reset: moving to HEAD~1
# b4e3d2c HEAD@{1}: commit: feat: добавить поиск
# 9c8b7a6 HEAD@{2}: commit: feat: базовая структура

# Восстановить "потерянный" коммит
git reset --hard b4e3d2c
```

`reflog` хранится ~90 дней. После этого коммиты недоступны.

## Шпаргалка по ситуациям

| Что случилось | Команда |
|--------------|---------|
| Изменил файл, хочу вернуть | `git restore <file>` |
| Добавил в staging, хочу убрать | `git restore --staged <file>` |
| Опечатка в сообщении коммита | `git commit --amend -m "новое"` |
| Забыл добавить файл в коммит | `git add f.js` + `git commit --amend --no-edit` |
| Отменить последний коммит (не пушен) | `git reset HEAD~1` |
| Отменить и удалить изменения | `git reset --hard HEAD~1` |
| Отменить запушенный коммит | `git revert HEAD` |
| Убрать работу на потом | `git stash push -m "..."` |
| Восстановить из stash | `git stash pop` |
| Найти "потерянный" коммит | `git reflog` |
