## Что такое ООП?

**ООП (Объектно-Ориентированное Программирование)** — это способ организации кода, при котором программа строится из объектов. Каждый объект содержит данные (свойства) и поведение (методы).

**Зачем это нужно?**
- Код становится структурированным и читаемым
- Легче поддерживать и расширять
- Можно переиспользовать код через наследование

---

## 4 принципа ООП

### 1. Инкапсуляция

**Идея:** данные внутри объекта скрыты от внешнего мира. Доступ к ним — только через методы.

**Зачем:** защита данных от случайного изменения, контроль через валидацию.

```js
class Book {
  #year; // приватное поле — снаружи недоступно

  constructor(title, author, year) {
    this.title = title;
    this.author = author;
    this.#year = year;
  }

  getYear() {
    return this.#year; // читаем через метод
  }

  setYear(newYear) {
    if (newYear > 0) {
      this.#year = newYear; // меняем только если прошло валидацию
    }
  }
}

const book = new Book("1984", "Оруэлл", 1949);
console.log(book.#year); // Ошибка! Приватное поле
console.log(book.getYear()); // 1949 — OK
```

> **Ключевой момент:** `#` перед именем поля делает его приватным. Это синтаксис ES2022 — относительно новый стандарт JS.

> 🔧 **Механика: как работает `new` при создании объекта**
>
> Когда ты пишешь `new Book("1984", "Оруэлл", 1949)`, JS выполняет 4 шага:
> ```
> Шаг 1: Создаёт пустой объект              → {}
> Шаг 2: Привязывает прототип               → {}.__proto__ = Book.prototype
> Шаг 3: Выполняет constructor с this = {}   → { title: "1984", author: "Оруэлл", #year: 1949 }
> Шаг 4: Возвращает готовый объект           → book = { title, author, #year }
> ```
> Если constructor явно возвращает объект (`return {}`), `new` вернёт его вместо созданного.
> Если возвращает примитив или ничего — вернёт объект из шага 1.

### 2. Наследование

**Идея:** один класс может взять всё из другого класса и добавить своё.

**Зачем:** избегаем дублирования кода.

```js
class Book {
  constructor(title, author, year) {
    this.title = title;
    this.author = author;
    this.year = year;
  }

  getInfo() {
    return `${this.title} — ${this.author} (${this.year})`;
  }
}

// Ebook наследует всё из Book
class Ebook extends Book {
  constructor(title, author, year, fileFormat) {
    super(title, author, year); // вызываем конструктор родителя
    this.fileFormat = fileFormat;
  }

  getInfo() {
    return `${super.getInfo()}, формат: ${this.fileFormat}`; // расширяем метод родителя
  }
}

const ebook = new Ebook("Дюна", "Герберт", 1965, "epub");
console.log(ebook.getInfo()); // "Дюна — Герберт (1965), формат: epub"
```

> **Важно:** `super()` в конструкторе **обязателен** если класс наследует другой. Без него будет ошибка.

> 🔧 **Механика: как `extends` выстраивает наследование**
>
> Когда JS видит `class Ebook extends Book`, он делает две связки:
> ```
> 1. Ebook.prototype.__proto__  →  Book.prototype   (для методов экземпляров)
> 2. Ebook.__proto__            →  Book              (для static методов)
> ```
> Результат — цепочка прототипов:
> ```
> ebook → Ebook.prototype → Book.prototype → Object.prototype → null
> ```
> Поэтому `ebook` видит методы и из `Ebook`, и из `Book`, и из `Object`.

> 🔧 **Механика: как работает `super()` внутри**
>
> `super(title, author, year)` в конструкторе `Ebook` делает следующее:
> ```
> Шаг 1: Вызывает Book.constructor(title, author, year)
> Шаг 2: Book.constructor записывает this.title, this.author, this.year
> Шаг 3: Управление возвращается в Ebook.constructor
> Шаг 4: Теперь можно использовать this (до super() — нельзя, будет ReferenceError)
> ```
> `super.getInfo()` в методе — это вызов `Book.prototype.getInfo()` с текущим `this`.
> Без `super()` в конструкторе наследника объект `this` просто не создаётся — отсюда ошибка.

**Цепочка в нашем коде:**
```
Publication (базовый)
    └── Book (добавляет title, author, #year)
            └── Ebook (добавляет fileFormat)
```

### 3. Полиморфизм

**Идея:** разные объекты отвечают на одну и ту же команду по-своему.

**Зачем:** можно работать с разными типами объектов через единый интерфейс.

```js
const books = [
  new Book("1984", "Оруэлл", 1949),
  new Ebook("Дюна", "Герберт", 1965, "epub"),
  new Book("Мастер и Маргарита", "Булгаков", 1967),
];

// Один и тот же вызов — разный результат
books.forEach(book => console.log(book.getInfo()));

// "1984 — Оруэлл (1949)"
// "Дюна — Герберт (1965), формат: epub"  ← другой вывод!
// "Мастер и Маргарита — Булгаков (1967)"
```

> **Переопределение метода** (override) — когда дочерний класс пишет свою версию метода родителя. Это и есть полиморфизм в действии.

### 4. Абстракция

**Идея:** скрыть сложность, показать только нужное. Создать "шаблон" который нельзя использовать напрямую.

**Зачем:** задать контракт — "все наследники обязаны реализовать эти методы".

```js
class Publication {
  constructor() {
    // Запрещаем создавать Publication напрямую
    if (new.target === Publication) {
      throw new Error("Publication — абстрактный класс, используй наследников");
    }
  }

  // Абстрактный метод — обязан быть переопределён
  getInfo() {
    throw new Error("Метод getInfo() должен быть реализован в наследнике");
  }
}

// new Publication() — бросит ошибку
// new Book(...) — OK, Book реализует getInfo()
```

> **Важно для понимания:** в JavaScript нет настоящих абстрактных классов (в отличие от Java, C#, TypeScript). Это ручная имитация через `throw`. В TypeScript это решается ключевым словом `abstract` — изучишь позже.

## Дополнительно: что ещё важно знать

### `new.target`
Специальная переменная внутри конструктора. Содержит класс, для которого был вызван `new`.

```js
class A {
  constructor() {
    console.log(new.target.name);
  }
}
class B extends A {}

new A(); // "A"
new B(); // "B" — потому что new вызван для B
```

Именно это используется в `Publication` чтобы понять — создают `Publication` напрямую или наследника.

### Когда использовать классы?

Используй класс когда:
- Нужно создать **несколько объектов** одного типа
- Объекты имеют **общее поведение** (методы)
- Есть **иерархия** (книга → электронная книга → аудиокнига)

Не используй класс когда:
- Объект создаётся один раз (просто `const config = { ... }`)
- Нет поведения, только данные

---

### Типичные ошибки новичков

| Ошибка | Проблема | Правильно |
|--------|----------|-----------|
| Забыть `super()` в наследнике | ReferenceError | Всегда вызывай `super()` первым в конструкторе |
| Обращаться к `this.#privateField` снаружи | SyntaxError | Используй геттер `getField()` |
| Мутировать данные напрямую без валидации | Баги | Используй сеттер с проверкой |

## Итог главы

```
ООП = организация кода через объекты с 4 принципами:

Инкапсуляция  → скрыть данные, дать доступ через методы (#privateField)
Наследование  → взять готовое, добавить своё (extends / super)
Полиморфизм   → один метод, разное поведение (override getInfo())
Абстракция    → шаблон-контракт для наследников (new.target + throw)
```
