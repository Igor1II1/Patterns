## Зачем эта тема существует?

Почти любое приложение отображает списки: список товаров, пользователей, сообщений, задач. А еще интерфейс должен по-разному выглядеть в зависимости от условий: показать кнопку "Войти" или "Выйти", отобразить загрузку или ошибку. Эта глава о том, как правильно рендерить списки и управлять тем, что показывать на экране.

## 1. Рендеринг списков с map()

### Что это такое?

В React нет специальной конструкции для циклов (как `v-for` в Vue или `*ngFor` в Angular). Вместо этого используется обычный JavaScript-метод `map()`, который преобразует массив данных в массив JSX-элементов.

### Как работает?

```jsx
function FruitList() {
  const fruits = ['Яблоко', 'Банан', 'Вишня', 'Груша'];

  return (
    <ul>
      {fruits.map(fruit => (
        <li key={fruit}>{fruit}</li>
      ))}
    </ul>
  );
}

// React получит:
// <ul>
//   <li>Яблоко</li>
//   <li>Банан</li>
//   <li>Вишня</li>
//   <li>Груша</li>
// </ul>
```

С объектами:

```jsx
function UserList() {
  const users = [
    { id: 1, name: 'Игорь', role: 'admin' },
    { id: 2, name: 'Анна', role: 'user' },
    { id: 3, name: 'Петр', role: 'user' },
  ];

  return (
    <div>
      {users.map(user => (
        <div key={user.id} className="user-card">
          <h3>{user.name}</h3>
          <p>Роль: {user.role}</p>
        </div>
      ))}
    </div>
  );
}
```

**Важно:** `map()` возвращает новый массив. React умеет рендерить массивы JSX-элементов. Фигурные скобки `{}` в JSX позволяют вставлять JavaScript-выражения, включая массивы.

### Вынесение элемента в отдельный компонент

```jsx
// Хорошая практика: отдельный компонент для элемента списка
function UserCard({ user }) {
  return (
    <div className="user-card">
      <h3>{user.name}</h3>
      <p>Роль: {user.role}</p>
    </div>
  );
}

function UserList({ users }) {
  return (
    <div>
      {users.map(user => (
        <UserCard key={user.id} user={user} />
      ))}
    </div>
  );
}
```

## 2. Ключи (keys) -- зачем они нужны

### Что это такое?

`key` -- это специальный атрибут, который React использует для идентификации элементов в списке. Ключ говорит React: "это тот же самый элемент, что и в прошлый раз" (или "это новый элемент").

### Как работает?

Без ключей React не может эффективно обновлять список. Допустим, у нас список из трех элементов и мы добавили один в начало:

```
Без ключей React думает:
  Было:   [A, B, C]
  Стало:  [D, A, B, C]
  React:  "A изменился на D, B на A, C на B, и добавился C"
  Результат: перерисовал ВСЕ четыре элемента

С ключами React понимает:
  Было:   [A(1), B(2), C(3)]
  Стало:  [D(4), A(1), B(2), C(3)]
  React:  "A, B, C не изменились, просто добавился D в начало"
  Результат: вставил ОДИН элемент
```

### Что использовать как ключ?

```jsx
// Лучший вариант: уникальный id из данных
{users.map(user => (
  <UserCard key={user.id} user={user} />
))}

// Допустимо: любое уникальное стабильное значение
{emails.map(email => (
  <li key={email}>{email}</li>
))}
```

### Плохой пример -- Хороший пример

```jsx
// Плохо: индекс массива как ключ
{items.map((item, index) => (
  <li key={index}>{item.name}</li>
))}
// Почему плохо? При добавлении/удалении элемента индексы сдвигаются.
// React думает, что элементы изменились, хотя они просто сдвинулись.
// Это ломает анимации, состояние инпутов и производительность.

// Хорошо: уникальный id
{items.map(item => (
  <li key={item.id}>{item.name}</li>
))}
```

**Когда индекс допустим:** только если список статический (никогда не сортируется, не фильтруется, элементы не добавляются/удаляются) И не содержит компонентов с собственным state.

### Правила ключей

1. **Уникальный среди соседей** -- ключи должны быть уникальны внутри одного массива, но могут повторяться в разных списках
2. **Стабильный** -- ключ одного элемента не должен меняться между рендерами
3. **Не используй `Math.random()`** -- новый ключ каждый рендер = пересоздание всех элементов
4. **key не передается как prop** -- компонент не получит `props.key`

```jsx
// key не доступен внутри компонента!
function UserCard({ key, user }) {  // key будет undefined
  console.log(key); // undefined
}

// Если нужен id внутри -- передавай отдельно
<UserCard key={user.id} id={user.id} user={user} />
```

### Частые заблуждения

- "Ключи нужны для производительности" -- это часть правды. Ключи нужны прежде всего для **корректности**: правильной привязки state, корректных анимаций, правильного порядка DOM-элементов.
- "Без key код не будет работать" -- будет, но React покажет предупреждение в консоли и будет работать неоптимально.

## 3. Условный рендеринг: оператор &&

### Что это такое?

Условный рендеринг -- отображение разного контента в зависимости от условий. Оператор `&&` (логическое И) -- самый простой способ: показать элемент, если условие истинно.

### Как работает?

```jsx
function Notification({ count }) {
  return (
    <div>
      <h1>Уведомления</h1>
      {count > 0 && <p>У вас {count} новых сообщений</p>}
      {/* Если count > 0 -- покажет <p>, иначе -- ничего */}
    </div>
  );
}
```

Принцип работы: `condition && expression`. Если `condition` -- truthy, возвращается `expression`. Если falsy -- возвращается само `condition`.

### Плохой пример -- Хороший пример

```jsx
// Плохо: ловушка с числом 0
function MessageCount({ count }) {
  return (
    <div>
      {count && <p>{count} сообщений</p>}
      {/* Если count = 0, React отрендерит 0 на экране!
          Потому что 0 -- falsy, && вернет 0, а React отображает числа */}
    </div>
  );
}

// Хорошо: явное сравнение
function MessageCount({ count }) {
  return (
    <div>
      {count > 0 && <p>{count} сообщений</p>}
      {/* Теперь условие вернет false, а React не отображает false */}
    </div>
  );
}
```

**Правило:** с `&&` всегда убеждайся, что левая часть -- boolean (`true`/`false`), а не число или строка.

## 4. Условный рендеринг: тернарный оператор

### Что это такое?

Тернарный оператор `condition ? a : b` позволяет выбрать один из двух вариантов: если условие истинно -- покажи `a`, иначе -- покажи `b`.

### Как работает?

```jsx
function LoginButton({ isLoggedIn }) {
  return (
    <div>
      {isLoggedIn ? (
        <button>Выйти</button>
      ) : (
        <button>Войти</button>
      )}
    </div>
  );
}

// С переменными для читаемости
function UserGreeting({ user }) {
  const greeting = user
    ? `Привет, ${user.name}!`
    : 'Привет, гость!';

  return <h1>{greeting}</h1>;
}
```

### Плохой пример -- Хороший пример

```jsx
// Плохо: вложенные тернарники -- нечитаемо
function StatusBadge({ status }) {
  return (
    <span>
      {status === 'active'
        ? 'Активен'
        : status === 'pending'
          ? 'Ожидание'
          : status === 'blocked'
            ? 'Заблокирован'
            : 'Неизвестно'}
    </span>
  );
}

// Хорошо: вынести логику или использовать объект-маппинг
function StatusBadge({ status }) {
  const labels = {
    active: 'Активен',
    pending: 'Ожидание',
    blocked: 'Заблокирован',
  };

  return <span>{labels[status] || 'Неизвестно'}</span>;
}
```

## 5. Условный рендеринг: ранний return

### Что это такое?

Ранний return -- паттерн, при котором компонент возвращает результат досрочно, не доходя до основного JSX. Полезен для обработки состояний загрузки, ошибок, пустых данных.

### Как работает?

```jsx
function UserProfile({ userId }) {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    fetch(`/api/users/${userId}`)
      .then(res => res.json())
      .then(data => { setUser(data); setLoading(false); })
      .catch(err => { setError(err.message); setLoading(false); });
  }, [userId]);

  // Ранние return -- обрабатываем особые случаи первыми
  if (loading) {
    return <div className="spinner">Загрузка...</div>;
  }

  if (error) {
    return <div className="error">Ошибка: {error}</div>;
  }

  if (!user) {
    return <div>Пользователь не найден</div>;
  }

  // Основной рендер -- мы уверены, что user существует
  return (
    <div className="profile">
      <h1>{user.name}</h1>
      <p>{user.email}</p>
      <p>Возраст: {user.age}</p>
    </div>
  );
}
```

### Плохой пример -- Хороший пример

```jsx
// Плохо: всё через тернарники -- трудно читать
function DataView({ data, loading, error }) {
  return (
    <div>
      {loading ? (
        <Spinner />
      ) : error ? (
        <Error message={error} />
      ) : data ? (
        <div>
          <h1>{data.title}</h1>
          <p>{data.body}</p>
        </div>
      ) : (
        <p>Нет данных</p>
      )}
    </div>
  );
}

// Хорошо: ранние return -- линейная логика, легко читать
function DataView({ data, loading, error }) {
  if (loading) return <Spinner />;
  if (error) return <Error message={error} />;
  if (!data) return <p>Нет данных</p>;

  return (
    <div>
      <h1>{data.title}</h1>
      <p>{data.body}</p>
    </div>
  );
}
```

## 6. Комбинирование паттернов

### Как работает?

В реальном приложении все паттерны используются вместе:

```jsx
function TodoApp() {
  const [todos, setTodos] = useState([
    { id: 1, text: 'Изучить React', done: true },
    { id: 2, text: 'Написать проект', done: false },
    { id: 3, text: 'Пройти собеседование', done: false },
  ]);
  const [filter, setFilter] = useState('all'); // 'all', 'active', 'done'

  // Фильтрация
  const filteredTodos = todos.filter(todo => {
    if (filter === 'active') return !todo.done;
    if (filter === 'done') return todo.done;
    return true;
  });

  // Ранний return для пустого списка
  if (todos.length === 0) {
    return <p>Список задач пуст. Добавьте первую задачу!</p>;
  }

  return (
    <div>
      <h1>Задачи ({todos.length})</h1>

      {/* Условный рендеринг: показать счетчик только если есть выполненные */}
      {todos.some(t => t.done) && (
        <p>Выполнено: {todos.filter(t => t.done).length}</p>
      )}

      {/* Рендеринг списка с map и key */}
      <ul>
        {filteredTodos.map(todo => (
          <li
            key={todo.id}
            style={{ textDecoration: todo.done ? 'line-through' : 'none' }}
          >
            {todo.text}
            {/* Тернарный оператор для текста кнопки */}
            <button onClick={() => toggleTodo(todo.id)}>
              {todo.done ? 'Отменить' : 'Выполнить'}
            </button>
          </li>
        ))}
      </ul>

      {/* Сообщение если фильтр не дал результатов */}
      {filteredTodos.length === 0 && (
        <p>Нет задач с фильтром "{filter}"</p>
      )}
    </div>
  );
}
```

## Итог

- **map()** -- основной способ рендеринга списков. Преобразует массив данных в массив JSX
- **key** -- обязательный атрибут для элементов списка. Используй уникальный id из данных, не индекс
- **&&** -- показать элемент если условие true. Осторожно с числом 0 (используй `> 0`)
- **Тернарный оператор** -- выбрать один из двух вариантов. Не вкладывай тернарники друг в друга
- **Ранний return** -- лучший способ обработать loading/error/empty. Линейная логика читается проще
- **Фильтрацию и подготовку данных** делай до return, не в JSX
