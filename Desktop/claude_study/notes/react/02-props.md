# Глава 2. Props -- передача данных в компоненты

> [[react/01-intro-jsx|<-- Назад: Что такое React]] | [[react-fundamentals|<-- Назад к оглавлению React]] | [[react/03-use-state|Далее: useState -->]]

---

## Зачем эта тема существует?

Компоненты без данных бесполезны. Кнопка должна знать свой текст, карточка товара -- название и цену, аватар -- URL картинки. Props (от "properties" -- свойства) -- это механизм передачи данных от родительского компонента к дочернему. Без props каждый компонент был бы статичным и одинаковым.

Представь функцию без аргументов: она всегда делает одно и то же. Props -- это аргументы для компонентов. Они делают компонент гибким и переиспользуемым.

---

## 1. Props -- передача данных в компонент

### Что это такое?

Props -- это объект, который React передает в функцию-компонент. Это **входные данные** компонента: то, что он получает снаружи и использует для отображения.

### Как работает?

```jsx
// Родительский компонент передает props как атрибуты JSX
function App() {
  return (
    <div>
      <UserCard name="Игорь" age={25} />
      <UserCard name="Анна" age={30} />
    </div>
  );
}

// Дочерний компонент получает props как объект-аргумент
function UserCard(props) {
  return (
    <div className="card">
      <h2>{props.name}</h2>
      <p>Возраст: {props.age}</p>
    </div>
  );
}
```

React собирает все атрибуты в один объект:

```js
// <UserCard name="Игорь" age={25} />
// Компонент получит:
props = {
  name: "Игорь",
  age: 25
}
```

**Правило типов значений:**
- Строки передаются в кавычках: `name="Игорь"`
- Числа, булевы, массивы, объекты, функции -- в фигурных скобках: `age={25}`, `isAdmin={true}`, `onClick={handleClick}`

### Плохой пример -- Хороший пример

```jsx
// Плохо: два почти одинаковых компонента
function AdminButton() {
  return <button className="btn btn-red">Удалить</button>;
}

function UserButton() {
  return <button className="btn btn-blue">Сохранить</button>;
}

// Хорошо: один компонент, настраиваемый через props
function Button(props) {
  return (
    <button className={`btn btn-${props.color}`}>
      {props.text}
    </button>
  );
}

// Использование:
<Button color="red" text="Удалить" />
<Button color="blue" text="Сохранить" />
```

## 2. Props -- только для чтения

### Что это такое?

Props нельзя изменять внутри компонента. Это **фундаментальное правило** React. Компонент может читать свои props, но не может их перезаписывать.

### Как работает?

```jsx
// НЕЛЬЗЯ -- это сломает приложение
function UserCard(props) {
  props.name = 'Другое имя'; // ОШИБКА! Нельзя менять props
  return <h2>{props.name}</h2>;
}

// МОЖНО -- использовать props для чтения
function UserCard(props) {
  const displayName = props.name.toUpperCase(); // создаем новую переменную
  return <h2>{displayName}</h2>;
}
```

**Почему нельзя?** React рассчитывает на то, что компонент -- это **чистая функция**: одни и те же props всегда дают одинаковый результат. Если компонент начнет менять props, React не сможет корректно определить, когда нужна перерисовка.

**Аналогия:** props -- это как задание от начальника. Ты можешь прочитать задание и выполнить его, но не можешь переписать само задание. Если нужны изменения -- используй state (об этом в следующей главе).

### Частые заблуждения

- "Если props нельзя менять, как обновлять данные?" -- через state. Родитель меняет свой state, передает новые props дочернему компоненту, и тот автоматически перерисовывается.
- "props.name = 'x' выбросит ошибку" -- в обычном режиме нет, но в strict mode -- да. В любом случае, это нарушение контракта React.

## 3. Деструктуризация props

### Что это такое?

Деструктуризация -- это способ извлечь значения из объекта прямо в параметрах функции. Вместо `props.name` пишешь просто `name`. Код становится чище и короче.

### Как работает?

```jsx
// Без деструктуризации -- работает, но многословно
function UserCard(props) {
  return (
    <div>
      <h2>{props.name}</h2>
      <p>Возраст: {props.age}</p>
      <p>Город: {props.city}</p>
    </div>
  );
}

// С деструктуризацией -- чище и понятнее
function UserCard({ name, age, city }) {
  return (
    <div>
      <h2>{name}</h2>
      <p>Возраст: {age}</p>
      <p>Город: {city}</p>
    </div>
  );
}

// Использование одинаковое:
<UserCard name="Игорь" age={25} city="Москва" />
```

Деструктуризация -- это стандартная фича JavaScript (ES6), не специфичная для React. Но в React она используется повсеместно.

### Плохой пример -- Хороший пример

```jsx
// Плохо: props.xxx повсюду, трудно читать
function ProductCard(props) {
  const total = props.price * props.quantity;
  return (
    <div>
      <h3>{props.title}</h3>
      <p>Цена: {props.price} руб.</p>
      <p>Количество: {props.quantity}</p>
      <p>Итого: {total} руб.</p>
      {props.inStock && <span>В наличии</span>}
    </div>
  );
}

// Хорошо: деструктуризация, сразу видно какие props принимает компонент
function ProductCard({ title, price, quantity, inStock }) {
  const total = price * quantity;
  return (
    <div>
      <h3>{title}</h3>
      <p>Цена: {price} руб.</p>
      <p>Количество: {quantity}</p>
      <p>Итого: {total} руб.</p>
      {inStock && <span>В наличии</span>}
    </div>
  );
}
```

Дополнительный бонус: при деструктуризации сразу видно, какие props компонент ожидает. Это служит документацией.

## 4. Значения по умолчанию

### Что это такое?

Иногда prop не обязателен. Если родитель его не передал, компонент должен использовать разумное значение по умолчанию.

### Как работает?

```jsx
// Значения по умолчанию через деструктуризацию
function Button({ text = 'Нажми', color = 'blue', size = 'medium' }) {
  return (
    <button className={`btn btn-${color} btn-${size}`}>
      {text}
    </button>
  );
}

// Все props необязательны:
<Button />                          // text="Нажми", color="blue", size="medium"
<Button text="Отправить" />         // text="Отправить", color="blue", size="medium"
<Button text="Удалить" color="red" size="small" />  // все переданы явно
```

### Плохой пример -- Хороший пример

```jsx
// Плохо: проверки через || (может сломаться с 0 или "")
function Counter({ count }) {
  return <span>{count || 'Нет данных'}</span>;
  // Если count = 0, покажет "Нет данных" -- баг!
}

// Хорошо: значение по умолчанию в деструктуризации
function Counter({ count = 0 }) {
  return <span>{count}</span>;
  // count = 0 -- покажет 0 (правильно)
}

// Или через nullish coalescing, если default нужен условно:
function Counter({ count }) {
  return <span>{count ?? 'Нет данных'}</span>;
  // 0 покажет 0, а null/undefined покажет "Нет данных"
}
```

## 5. children -- особый prop

### Что это такое?

`children` -- это специальный prop, который содержит все, что находится **между** открывающим и закрывающим тегами компонента. Это позволяет создавать компоненты-обертки.

### Как работает?

```jsx
// Компонент-обертка
function Card({ children, title }) {
  return (
    <div className="card">
      <h2 className="card-title">{title}</h2>
      <div className="card-body">
        {children}
      </div>
    </div>
  );
}

// Использование -- содержимое между тегами становится children
function App() {
  return (
    <Card title="Профиль">
      <img src="avatar.jpg" alt="Аватар" />
      <p>Игорь, 25 лет</p>
      <button>Редактировать</button>
    </Card>
  );
}
```

`children` может быть чем угодно: текстом, JSX-элементами, компонентами, массивом:

```jsx
// Текст
<Button>Нажми меня</Button>
// children = "Нажми меня"

// JSX-элементы
<Card>
  <h2>Заголовок</h2>
  <p>Текст</p>
</Card>
// children = [<h2>Заголовок</h2>, <p>Текст</p>]

// Другой компонент
<Layout>
  <Header />
  <Content />
  <Footer />
</Layout>
```

### Плохой пример -- Хороший пример

```jsx
// Плохо: текст кнопки передается как prop
<Button text="Отправить" icon="send" />

function Button({ text, icon }) {
  return (
    <button>
      <Icon name={icon} />
      {text}
    </button>
  );
}

// Хорошо: используем children -- гибче и нагляднее
<Button>
  <Icon name="send" />
  Отправить
</Button>

function Button({ children }) {
  return <button>{children}</button>;
}

// Теперь кнопка может содержать что угодно:
<Button>Просто текст</Button>
<Button><Icon name="delete" /> Удалить</Button>
<Button><Spinner /> Загрузка...</Button>
```

### Частые заблуждения

- "children -- это особый механизм React" -- нет, `children` -- это обычный prop. Просто React автоматически помещает содержимое между тегами в `props.children`.
- "children всегда массив" -- нет. Один элемент -- это сам элемент. Несколько -- массив. Текст -- строка.

## 6. Типизация props (концепция)

### Что это такое?

JavaScript не проверяет типы props. Ты можешь передать число вместо строки, и ошибка всплывет только в рантайме. Для больших проектов это проблема. Существуют два подхода к типизации.

### Как работает?

**Подход 1: PropTypes (устаревший, но встречается)**

```jsx
import PropTypes from 'prop-types';

function UserCard({ name, age, isAdmin }) {
  return (
    <div>
      <h2>{name}</h2>
      <p>Возраст: {age}</p>
      {isAdmin && <span>Админ</span>}
    </div>
  );
}

UserCard.propTypes = {
  name: PropTypes.string.isRequired,  // обязательная строка
  age: PropTypes.number,               // необязательное число
  isAdmin: PropTypes.bool              // необязательный boolean
};

UserCard.defaultProps = {
  age: 18,
  isAdmin: false
};
```

**Подход 2: TypeScript (современный стандарт)**

```tsx
// TypeScript -- статическая проверка на этапе написания кода
type UserCardProps = {
  name: string;         // обязательно
  age?: number;         // необязательно (?)
  isAdmin?: boolean;    // необязательно
};

function UserCard({ name, age = 18, isAdmin = false }: UserCardProps) {
  return (
    <div>
      <h2>{name}</h2>
      <p>Возраст: {age}</p>
      {isAdmin && <span>Админ</span>}
    </div>
  );
}
```

TypeScript ловит ошибки **до запуска кода** -- прямо в редакторе. Это делает разработку быстрее и безопаснее. Сегодня TypeScript -- стандарт индустрии для React-проектов.

### Частые заблуждения

- "PropTypes достаточно" -- PropTypes проверяют типы только в рантайме и только в dev-режиме. TypeScript проверяет на этапе компиляции -- это надежнее.
- "TypeScript сложный и не стоит усилий" -- кривая обучения есть, но окупается очень быстро. IDE начинает подсказывать props, ловить опечатки и показывать документацию.

## Итог

- **Props** -- объект с данными, которые родитель передает дочернему компоненту. Это аргументы функции-компонента
- **Props только для чтения** -- компонент не может менять свои props. Это гарантия предсказуемости
- **Деструктуризация** -- способ писать `{ name, age }` вместо `props.name, props.age`. Делает код чище
- **Значения по умолчанию** -- задаются через `{ name = 'Гость' }` в деструктуризации
- **children** -- специальный prop, содержимое между тегами компонента. Позволяет создавать гибкие обертки
- **TypeScript** -- современный способ типизации props. Ловит ошибки до запуска программы
