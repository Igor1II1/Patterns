## Зачем эта тема существует?

Когда веб-приложение становится сложным -- десятки кнопок, формы, списки, модальные окна -- управлять DOM вручную через `document.createElement` и `addEventListener` становится кошмаром. Код превращается в спагетти: непонятно, какой обработчик что меняет, какой элемент от чего зависит.

React решает эту проблему. Вместо того чтобы вручную менять DOM, ты **описываешь**, как интерфейс должен выглядеть при данных данных. React сам разбирается, что именно нужно обновить на странице. Это как разница между тем, чтобы давать пошаговые инструкции ("возьми краску, закрась квадрат, потом нарисуй круг") и просто показать картинку ("вот так должно выглядеть").

## 1. React -- библиотека для построения интерфейсов

### Что это такое?

React -- это JavaScript-библиотека от Meta (Facebook) для создания пользовательских интерфейсов. Не фреймворк (как Angular), а именно библиотека -- она отвечает только за отображение (View), а остальное ты выбираешь сам.

Ключевая идея React: **UI = f(state)**. Интерфейс -- это функция от данных. Изменились данные -- интерфейс автоматически перерисовался.

### Как работает?

React работает через три шага:

1. Ты описываешь компонент (функция, которая возвращает разметку)
2. React вызывает эту функцию и получает описание интерфейса
3. React сравнивает это описание с тем, что сейчас на экране, и обновляет только изменившиеся части

```jsx
// Ты пишешь:
function Welcome() {
  return <h1>Привет, мир!</h1>;
}

// React превращает это в реальные DOM-элементы на странице
```

### Частые заблуждения

- "React -- это фреймворк" -- нет, это библиотека. Фреймворк диктует структуру всего приложения (Angular, Next.js). React отвечает только за UI.
- "React нужен для любого сайта" -- нет. Для простой статической страницы React избыточен. Он нужен, когда интерфейс интерактивный и сложный.

## 2. Virtual DOM

### Что это такое?

Virtual DOM (виртуальный DOM) -- это легковесная копия реального DOM, которую React хранит в памяти. Это обычные JavaScript-объекты, описывающие структуру страницы.

### Как работает?

Прямая работа с DOM -- медленная операция. Каждое изменение может вызвать перерасчет стилей, перекомпоновку и перерисовку страницы. React минимизирует эти дорогие операции через алгоритм **согласования (reconciliation)**:

1. При изменении данных React создает **новый** Virtual DOM
2. Сравнивает его с **предыдущим** Virtual DOM (diffing)
3. Вычисляет **минимальный набор изменений**
4. Применяет только эти изменения к реальному DOM

```
Данные изменились
       |
       v
Новый Virtual DOM  <-->  Старый Virtual DOM
       |                        |
       +--- diff (сравнение) ---+
       |
       v
Минимальные изменения --> Реальный DOM
```

**Аналогия:** представь, что ты редактируешь документ. Вместо того чтобы перепечатать весь текст заново, ты находишь конкретные места, которые нужно исправить, и меняешь только их. Virtual DOM -- это черновик, в котором ты сначала делаешь все правки, а потом одним махом переносишь их в чистовик.

### Плохой пример -- Хороший пример

```js
// Без React: ручное обновление DOM
// Плохо -- при каждом клике пересоздаем весь список
function updateList(items) {
  const ul = document.getElementById('list');
  ul.innerHTML = ''; // удаляем всё
  items.forEach(item => {
    const li = document.createElement('li');
    li.textContent = item;
    ul.appendChild(li);
  });
}
```

```jsx
// С React: описываем что должно быть, React сам обновит нужное
function ItemList({ items }) {
  return (
    <ul>
      {items.map(item => (
        <li key={item}>{item}</li>
      ))}
    </ul>
  );
}
```

### Частые заблуждения

- "Virtual DOM быстрее реального DOM" -- не совсем. Virtual DOM добавляет свой overhead. Но он позволяет **батчить** (группировать) изменения и применять их оптимально. Для сложных интерфейсов это выигрыш.

## 3. JSX -- синтаксис для описания интерфейса

### Что это такое?

JSX (JavaScript XML) -- это расширение синтаксиса JavaScript, которое позволяет писать HTML-подобную разметку прямо в JS-коде. JSX -- это **не** HTML. Это синтаксический сахар для вызовов `React.createElement()`.

### Как работает?

Компилятор (Babel или TypeScript) превращает JSX в обычные вызовы функций:

```jsx
// Ты пишешь JSX:
const element = <h1 className="title">Привет</h1>;

// Компилятор превращает в:
const element = React.createElement('h1', { className: 'title' }, 'Привет');
```

Результат -- обычный JavaScript-объект:

```js
{
  type: 'h1',
  props: {
    className: 'title',
    children: 'Привет'
  }
}
```

### Правила JSX

```jsx
// 1. Один корневой элемент (или Fragment)
// Плохо:
return (
  <h1>Заголовок</h1>
  <p>Текст</p>
);

// Хорошо:
return (
  <div>
    <h1>Заголовок</h1>
    <p>Текст</p>
  </div>
);

// Или через Fragment (не создает лишний DOM-элемент):
return (
  <>
    <h1>Заголовок</h1>
    <p>Текст</p>
  </>
);

// 2. Все теги должны быть закрыты
<img src="photo.jpg" />    // самозакрывающийся тег
<br />                      // не <br> как в HTML

// 3. className вместо class
<div className="container">...</div>   // не class=

// 4. htmlFor вместо for
<label htmlFor="email">Email</label>   // не for=

// 5. camelCase для атрибутов
<div onClick={handleClick} tabIndex={0}>...</div>

// 6. JavaScript-выражения в фигурных скобках
const name = 'Игорь';
<h1>Привет, {name}!</h1>           // переменная
<p>{2 + 2}</p>                      // выражение
<p>{isAdmin ? 'Админ' : 'Гость'}</p> // тернарный оператор
```

### Частые заблуждения

- "JSX -- это HTML внутри JavaScript" -- нет. JSX только **похож** на HTML. Это JavaScript-выражения, которые компилируются в вызовы функций.
- "Без JSX нельзя писать React" -- можно, через `React.createElement()`. Но JSX намного читаемее.

## 4. Компоненты -- строительные блоки React

### Что это такое?

Компонент -- это функция, которая возвращает JSX. Это переиспользуемый кусок интерфейса со своей логикой и разметкой. Компонент можно представить как LEGO-кубик: из множества маленьких компонентов собирается целая страница.

### Как работает?

```jsx
// Компонент -- это функция с большой буквы
function Greeting() {
  return <h1>Привет, мир!</h1>;
}

// Использование -- как HTML-тег
function App() {
  return (
    <div>
      <Greeting />
      <Greeting />
      <Greeting />
    </div>
  );
}
```

**Важно:** имя компонента всегда начинается с **большой буквы**. React различает `<div>` (HTML-элемент) и `<Greeting>` (компонент) именно по регистру первой буквы.

### Плохой пример -- Хороший пример

```jsx
// Плохо: всё в одной функции, ничего не переиспользуется
function App() {
  return (
    <div>
      <div className="header">
        <h1>Мой сайт</h1>
        <nav>
          <a href="/">Главная</a>
          <a href="/about">О нас</a>
        </nav>
      </div>
      <div className="content">
        <h2>Статья</h2>
        <p>Текст статьи...</p>
      </div>
      <div className="footer">
        <p>2026 Мой сайт</p>
      </div>
    </div>
  );
}

// Хорошо: разбито на компоненты
function Header() {
  return (
    <header className="header">
      <h1>Мой сайт</h1>
      <Navigation />
    </header>
  );
}

function Navigation() {
  return (
    <nav>
      <a href="/">Главная</a>
      <a href="/about">О нас</a>
    </nav>
  );
}

function Article() {
  return (
    <section className="content">
      <h2>Статья</h2>
      <p>Текст статьи...</p>
    </section>
  );
}

function Footer() {
  return (
    <footer className="footer">
      <p>2026 Мой сайт</p>
    </footer>
  );
}

function App() {
  return (
    <div>
      <Header />
      <Article />
      <Footer />
    </div>
  );
}
```

## 5. Рендеринг -- как React попадает на страницу

### Что это такое?

Рендеринг -- это процесс, при котором React превращает дерево компонентов в реальные DOM-элементы и вставляет их на страницу.

### Как работает?

Точка входа -- файл `index.js` (или `main.jsx` в Vite):

```jsx
// index.js (Create React App) или main.jsx (Vite)
import React from 'react';
import ReactDOM from 'react-dom/client';
import App from './App';

// Находим контейнер в HTML
const root = ReactDOM.createRoot(document.getElementById('root'));

// Рендерим React-приложение в этот контейнер
root.render(<App />);
```

А в HTML-файле (`index.html`) есть всего один элемент:

```html
<!DOCTYPE html>
<html>
<head>
  <title>Мое приложение</title>
</head>
<body>
  <div id="root"></div>
  <!-- Всё React-приложение окажется внутри этого div -->
</body>
</html>
```

### Что происходит при рендеринге

1. `ReactDOM.createRoot()` создает "корень" React внутри DOM-элемента
2. `root.render(<App />)` вызывает компонент `App`
3. `App` возвращает JSX-дерево
4. React рекурсивно вызывает все вложенные компоненты
5. Результат -- полное дерево React-элементов
6. React превращает это дерево в реальные DOM-элементы

## 6. import/export -- модульная система

### Что это такое?

React-приложение состоит из множества файлов. Каждый компонент обычно живет в отдельном файле. `import` и `export` позволяют связывать файлы между собой.

### Как работает?

```jsx
// --- Header.jsx ---
// Именованный экспорт
export function Header() {
  return <header><h1>Мой сайт</h1></header>;
}

// --- Footer.jsx ---
// Экспорт по умолчанию
function Footer() {
  return <footer><p>2026</p></footer>;
}
export default Footer;

// --- App.jsx ---
// Именованный импорт -- имя должно совпадать, фигурные скобки обязательны
import { Header } from './Header';

// Импорт по умолчанию -- имя можно выбрать любое, без скобок
import Footer from './Footer';
// или: import MyFooter from './Footer';  -- тоже сработает

function App() {
  return (
    <div>
      <Header />
      <Footer />
    </div>
  );
}

export default App;
```

### Плохой пример -- Хороший пример

```jsx
// Плохо: всё в одном файле
// App.jsx -- 500 строк, 10 компонентов

// Хорошо: один компонент = один файл
// components/Header.jsx
// components/Footer.jsx
// components/Navigation.jsx
// App.jsx -- только композиция
```

**Соглашение:** файлы компонентов называют с большой буквы (`Header.jsx`, `UserCard.jsx`) или используют папки (`Header/index.jsx`).

### Частые заблуждения

- "Можно экспортировать по умолчанию и именованно из одного файла" -- технически да, но это создает путаницу. Лучше выбрать один стиль.
- "Расширение `.jsx` обязательно" -- зависит от настроек сборщика. В Vite -- да, `.jsx` обязательно для файлов с JSX. В Create React App можно использовать `.js`.

## Итог

- **React** -- библиотека для создания интерфейсов. Ключевая идея: UI как функция от данных
- **Virtual DOM** -- промежуточное представление, позволяющее React обновлять только то, что изменилось
- **JSX** -- синтаксический сахар, HTML-подобная разметка внутри JavaScript, компилируется в `React.createElement()`
- **Компонент** -- функция, возвращающая JSX. Имя -- с большой буквы. Один компонент = один файл
- **Рендеринг** -- `ReactDOM.createRoot()` + `root.render()` вставляют React-дерево в DOM
- **import/export** -- модульная система для связывания файлов. `export default` (один на файл) и `export` (именованный, может быть несколько)
