# Промисы (Promise)

> [[js-fundamentals|<-- Назад к оглавлению JS]]

---

## Зачем эта тема существует?

Промисы появились в ES6 (2015) как ответ на проблемы колбэков: Callback Hell, разрозненную обработку ошибок и невозможность удобно комбинировать асинхронные операции. Promise — это **стандартный объект**, представляющий результат асинхронной операции, который может появиться сейчас, позже или никогда. Промисы стали фундаментом современного асинхронного JavaScript, на котором построены `async/await`, `fetch` и множество других API.

---

## Что такое Promise?

### Что это такое?

Promise (промис, "обещание") — это объект, который хранит **состояние** асинхронной операции и её **результат**, когда она завершится.

Аналогия: ты заказываешь кофе в кофейне. Тебе дают **чек с номером** — это промис. Чек обещает, что кофе будет готов. Ты не стоишь у стойки — ты садишься и ждёшь. Когда кофе готов, тебя зовут по номеру (fulfilled). Если кофе-машина сломалась — тебе сообщают об этом (rejected).

### Три состояния промиса

```
                    ┌─────────────────────────────┐
                    │        ПРОМИС                │
                    │                              │
                    │   ┌────────────────────┐     │
        создан ────>│   │     pending        │     │
                    │   │   (ожидание)       │     │
                    │   └─────────┬──────────┘     │
                    │             │                 │
                    │      ┌──────┴──────┐          │
                    │      ▼             ▼          │
                    │ ┌──────────┐ ┌───────────┐   │
                    │ │fulfilled │ │ rejected  │   │
                    │ │(успех)   │ │ (ошибка)  │   │
                    │ │          │ │           │   │
                    │ │ .then()  │ │ .catch()  │   │
                    │ └──────────┘ └───────────┘   │
                    │                              │
                    │    settled (завершён)         │
                    └─────────────────────────────┘
```

- **pending** — начальное состояние, операция ещё выполняется
- **fulfilled** — операция завершилась успешно, есть результат
- **rejected** — операция завершилась с ошибкой

Промис может перейти из `pending` в `fulfilled` или `rejected` **только один раз**. После этого состояние **заморожено** навсегда.

## Создание промиса

### Как работает?

```js
const промис = new Promise(function(resolve, reject) {
  // Здесь выполняется асинхронная работа

  // Если успех:
  resolve(результат);

  // Если ошибка:
  reject(ошибка);
});
```

Функция, переданная в `new Promise`, называется **executor** (исполнитель). Она получает два аргумента:
- `resolve(value)` — вызывается при успехе, переводит промис в `fulfilled`
- `reject(error)` — вызывается при ошибке, переводит промис в `rejected`

```js
// Пример: загрузка данных (имитация)
const загрузка = new Promise(function(resolve, reject) {
  setTimeout(function() {
    const успех = true;

    if (успех) {
      resolve({ name: "Игорь", age: 25 });
    } else {
      reject(new Error("Не удалось загрузить данные"));
    }
  }, 2000);
});
```

### Плохой пример -> Хороший пример

```js
// ПЛОХО — resolve и reject вызываются оба
const промис = new Promise(function(resolve, reject) {
  resolve("успех");
  reject("ошибка");  // игнорируется — промис уже resolved
  // Но это запутывает! Не делай так.
});
```

```js
// ХОРОШО — чёткая логика: или resolve, или reject
const промис = new Promise(function(resolve, reject) {
  const число = Math.random();

  if (число > 0.5) {
    resolve("Повезло: " + число);
  } else {
    reject(new Error("Не повезло: " + число));
  }
});
```

> **Частое заблуждение:** "Код внутри `new Promise` выполняется асинхронно." Нет — executor выполняется **синхронно**, немедленно. Асинхронным является только получение результата через `.then`.

```js
console.log("до промиса");

const p = new Promise(function(resolve, reject) {
  console.log("внутри executor");  // выполняется СРАЗУ
  resolve("готово");
});

console.log("после промиса");

// Результат:
// до промиса
// внутри executor     <-- синхронно!
// после промиса
```

## .then() — обработка успеха

### Как работает?

Метод `.then()` принимает функцию, которая выполнится, когда промис перейдёт в состояние `fulfilled`:

```js
const промис = new Promise(function(resolve, reject) {
  setTimeout(() => resolve(42), 1000);
});

промис.then(function(результат) {
  console.log("Получили: " + результат);  // "Получили: 42" — через 1 секунду
});
```

`.then()` может принимать **два** аргумента — обработчик успеха и обработчик ошибки:

```js
промис.then(
  function(результат) { console.log("Успех: " + результат); },
  function(ошибка)     { console.log("Ошибка: " + ошибка.message); }
);
```

Но на практике для ошибок чаще используют `.catch()`.

## .catch() — обработка ошибок

### Как работает?

`.catch()` ловит ошибки — как от `reject()`, так и исключения внутри `.then()`:

```js
const промис = new Promise(function(resolve, reject) {
  setTimeout(() => reject(new Error("Сервер не отвечает")), 1000);
});

промис.catch(function(ошибка) {
  console.log("Поймали ошибку: " + ошибка.message);
  // "Поймали ошибку: Сервер не отвечает"
});
```

`.catch()` ловит ошибки **на любом этапе** цепочки:

```js
промис
  .then(function(data) {
    throw new Error("Ошибка в обработчике!");
  })
  .catch(function(err) {
    console.log(err.message);  // "Ошибка в обработчике!"
  });
```

### Плохой пример -> Хороший пример

```js
// ПЛОХО — ошибка в .then не ловится вторым аргументом того же .then
промис.then(
  function(data) {
    throw new Error("Упс");  // эта ошибка НЕ попадёт в обработчик ниже
  },
  function(err) {
    console.log(err);  // ловит только reject промиса, не ошибки из первого аргумента
  }
);
```

```js
// ХОРОШО — .catch в конце ловит все ошибки
промис
  .then(function(data) {
    throw new Error("Упс");
  })
  .catch(function(err) {
    console.log(err.message);  // "Упс" — поймали!
  });
```

## .finally() — выполнить в любом случае

### Как работает?

`.finally()` выполняется **всегда** — и при успехе, и при ошибке. Не получает аргументов. Полезен для "уборки": скрыть индикатор загрузки, закрыть соединение.

```js
показатьЗагрузку();

загрузитьДанные()
  .then(function(данные) {
    показать(данные);
  })
  .catch(function(ошибка) {
    показатьОшибку(ошибка);
  })
  .finally(function() {
    скрытьЗагрузку();  // выполнится в обоих случаях
  });
```

## Цепочки промисов (Chaining)

### Что это такое?

Каждый `.then()` возвращает **новый промис**. Это позволяет строить цепочки — последовательные асинхронные операции без вложенности.

### Как работает?

```js
// Цепочка: каждый .then получает результат предыдущего
new Promise(function(resolve) {
  resolve(1);
})
  .then(function(результат) {
    console.log(результат); // 1
    return результат * 2;   // возвращаем 2
  })
  .then(function(результат) {
    console.log(результат); // 2
    return результат * 3;   // возвращаем 6
  })
  .then(function(результат) {
    console.log(результат); // 6
  });
```

Если `.then` возвращает **промис**, следующий `.then` ждёт его выполнения:

```js
function загрузить(url) {
  return new Promise(function(resolve) {
    setTimeout(() => resolve("данные с " + url), 1000);
  });
}

загрузить("/api/user")
  .then(function(user) {
    console.log(user);
    return загрузить("/api/posts");  // возвращаем промис
  })
  .then(function(posts) {
    console.log(posts);              // ждёт завершения предыдущего
    return загрузить("/api/comments");
  })
  .then(function(comments) {
    console.log(comments);
  })
  .catch(function(err) {
    console.error("Ошибка на любом этапе:", err);
  });
```

### Сравнение: колбэки vs промисы

```js
// КОЛБЭКИ — вложенность
загрузить("/user", function(err, user) {
  загрузить("/posts", function(err, posts) {
    загрузить("/comments", function(err, comments) {
      показать(user, posts, comments);
    });
  });
});

// ПРОМИСЫ — плоская цепочка
загрузить("/user")
  .then(user => загрузить("/posts"))
  .then(posts => загрузить("/comments"))
  .then(comments => показать(comments))
  .catch(err => обработатьОшибку(err));
```

### Плохой пример -> Хороший пример

```js
// ПЛОХО — промис внутри промиса (те же вложенности!)
загрузить("/user").then(function(user) {
  загрузить("/posts").then(function(posts) {
    загрузить("/comments").then(function(comments) {
      показать(comments);
    });
  });
});
```

```js
// ХОРОШО — возвращаем промис, строим цепочку
загрузить("/user")
  .then(function(user) {
    return загрузить("/posts");
  })
  .then(function(posts) {
    return загрузить("/comments");
  })
  .then(function(comments) {
    показать(comments);
  })
  .catch(function(err) {
    обработатьОшибку(err);
  });
```

> **Частое заблуждение:** "Если я не верну значение из `.then`, цепочка продолжит работать нормально." Технически да, но следующий `.then` получит `undefined`. Всегда возвращай результат!

## Распространение ошибок

### Как работает?

Ошибка в цепочке промисов "проскакивает" все `.then` до ближайшего `.catch`:

```
.then(...)
.then(...)    ← ошибка здесь!
.then(...)    ← пропущен
.then(...)    ← пропущен
.catch(...)   ← ошибка поймана здесь
```

```js
new Promise(function(resolve) {
  resolve("начало");
})
  .then(function(data) {
    throw new Error("Что-то сломалось");
  })
  .then(function(data) {
    console.log("Это НЕ выполнится");
  })
  .then(function(data) {
    console.log("И это НЕ выполнится");
  })
  .catch(function(err) {
    console.log("Поймали: " + err.message);  // "Поймали: Что-то сломалось"
    return "восстановились";  // цепочка продолжается!
  })
  .then(function(data) {
    console.log(data);  // "восстановились"
  });
```

> **Мини-проверка:** Что выведет этот код?
> ```js
> Promise.resolve(1)
>   .then(x => x + 1)
>   .then(x => { throw new Error("Упс"); })
>   .then(x => x + 1)
>   .catch(e => 10)
>   .then(x => console.log(x));
> ```
> Ответ: `10`. Ошибка перепрыгивает третий `.then`, `.catch` возвращает 10, последний `.then` получает его.

## Promise.resolve() и Promise.reject()

### Что это такое?

Сокращённые способы создать уже завершённый промис:

```js
// Вместо:
const p1 = new Promise(function(resolve) { resolve(42); });

// Можно:
const p1 = Promise.resolve(42);


// Вместо:
const p2 = new Promise(function(resolve, reject) { reject(new Error("нет")); });

// Можно:
const p2 = Promise.reject(new Error("нет"));
```

Полезно для тестирования и когда функция должна **всегда** возвращать промис:

```js
function получитьДанные(кэш) {
  if (кэш) {
    return Promise.resolve(кэш);  // данные уже есть — мгновенный промис
  }
  return fetch("/api/data");       // данных нет — настоящий запрос
}
```

## Итог

| Понятие | Описание |
|---------|----------|
| **Promise** | Объект, представляющий будущий результат операции |
| **pending** | Начальное состояние — ожидание |
| **fulfilled** | Операция завершилась успешно |
| **rejected** | Операция завершилась с ошибкой |
| **resolve(value)** | Переводит промис в fulfilled |
| **reject(error)** | Переводит промис в rejected |
| **.then(fn)** | Обработчик успешного результата |
| **.catch(fn)** | Обработчик ошибки |
| **.finally(fn)** | Выполняется в любом случае |
| **Цепочка** | `.then().then().catch()` — плоская последовательность |
| **Promise.resolve/reject** | Быстрое создание завершённого промиса |

**Ключевые правила:**
1. Промис меняет состояние **только один раз**
2. `.then()` **всегда** возвращает новый промис
3. Ошибка пролетает до ближайшего `.catch()`
4. Всегда ставь `.catch()` в конце цепочки
5. Возвращай значения из `.then()`, иначе цепочка получит `undefined`
