# Методы Promise: all, race, allSettled, any

> [[js-fundamentals|<-- Назад к оглавлению JS]]

---

## Зачем эта тема существует?

В реальных приложениях часто нужно работать с **несколькими** промисами одновременно: загрузить данные из трёх API, дождаться самого быстрого ответа из нескольких серверов, проверить результат всех операций. Для этого у Promise есть статические методы — `all`, `race`, `allSettled`, `any`. Каждый решает свою задачу, и выбор правильного метода — навык, который отличает осознанного разработчика от того, кто гуглит каждый раз.

---

## Promise.all

### Что это такое?

`Promise.all` принимает **массив промисов** и возвращает один промис, который:
- **fulfilled** — когда **все** промисы завершились успешно (результат — массив всех значений)
- **rejected** — когда **хотя бы один** промис завершился с ошибкой (результат — первая ошибка)

Аналогия: ты заказал три блюда в ресторане. Официант принесёт их **все сразу** на одном подносе, когда последнее будет готово. Но если одно блюдо не получилось — весь заказ считается неудачным.

### Как работает?

```js
const промис1 = Promise.resolve(1);
const промис2 = Promise.resolve(2);
const промис3 = Promise.resolve(3);

Promise.all([промис1, промис2, промис3])
  .then(function(результаты) {
    console.log(результаты);  // [1, 2, 3] — массив в том же порядке
  });
```

Практический пример — загрузка данных из нескольких источников:

```js
async function загрузитьДашборд() {
  try {
    const [пользователь, заказы, уведомления] = await Promise.all([
      fetch("/api/user").then(r => r.json()),
      fetch("/api/orders").then(r => r.json()),
      fetch("/api/notifications").then(r => r.json())
    ]);

    показатьДашборд(пользователь, заказы, уведомления);
  } catch (ошибка) {
    console.error("Один из запросов упал:", ошибка);
  }
}
```

```
Без Promise.all (последовательно):
  [user 500мс] → [orders 300мс] → [notif 200мс]  = 1000мс

С Promise.all (параллельно):
  [user    500мс]
  [orders  300мс]    = 500мс (время самого долгого)
  [notif   200мс]
```

### Поведение при ошибке

```js
Promise.all([
  Promise.resolve("OK"),
  Promise.reject(new Error("Упал!")),
  Promise.resolve("Тоже OK")
])
  .then(results => console.log(results))     // НЕ выполнится
  .catch(err => console.log(err.message));    // "Упал!"
```

```
Промис 1: ✅ "OK"
Промис 2: ❌ Error("Упал!")    ← первая ошибка
Промис 3: ✅ "Тоже OK"        ← результат игнорируется

Promise.all → ❌ rejected с Error("Упал!")
```

### Плохой пример -> Хороший пример

```js
// ПЛОХО — последовательные независимые запросы
async function загрузить() {
  const a = await fetch("/api/a");
  const b = await fetch("/api/b");
  const c = await fetch("/api/c");
  // Каждый ждёт предыдущий — медленно!
}
```

```js
// ХОРОШО — параллельно через Promise.all
async function загрузить() {
  const [a, b, c] = await Promise.all([
    fetch("/api/a"),
    fetch("/api/b"),
    fetch("/api/c")
  ]);
}
```

> **Частое заблуждение:** "Promise.all выполняет промисы." Нет — промисы запускаются в момент создания. `Promise.all` просто **ждёт** их завершения.

> **Мини-проверка:** Что вернёт `Promise.all([])` (пустой массив)?
> Ответ: `Promise.resolve([])` — мгновенно fulfilled с пустым массивом.

## Promise.race

### Что это такое?

`Promise.race` возвращает промис, который завершается с результатом **первого** завершившегося промиса (успехом или ошибкой — неважно).

Аналогия: гонка нескольких бегунов. Нам важен только тот, кто прибежал **первым**. Остальные продолжают бежать, но их результат нас уже не интересует.

### Как работает?

```js
const быстрый = new Promise(resolve => setTimeout(() => resolve("быстрый"), 100));
const средний = new Promise(resolve => setTimeout(() => resolve("средний"), 500));
const медленный = new Promise(resolve => setTimeout(() => resolve("медленный"), 1000));

Promise.race([быстрый, средний, медленный])
  .then(function(победитель) {
    console.log(победитель);  // "быстрый" — первый завершился
  });
```

**Практический пример — таймаут для запроса:**

```js
function сТаймаутом(промис, мс) {
  const таймаут = new Promise(function(resolve, reject) {
    setTimeout(() => reject(new Error("Таймаут: " + мс + "мс")), мс);
  });

  return Promise.race([промис, таймаут]);
}

// Использование
async function загрузить() {
  try {
    const данные = await сТаймаутом(fetch("/api/data"), 3000);
    console.log("Данные получены");
  } catch (err) {
    console.log(err.message);  // "Таймаут: 3000мс" — если сервер не ответил за 3 сек
  }
}
```

```
Сценарий 1 — сервер ответил быстро:
  [fetch: 1сек ✅]
  [таймаут: 3сек ⏳]    → race → ✅ данные (fetch победил)

Сценарий 2 — сервер тормозит:
  [fetch: 10сек ⏳]
  [таймаут: 3сек ❌]    → race → ❌ "Таймаут" (таймаут победил)
```

### Поведение при ошибке первого

```js
Promise.race([
  new Promise((_, reject) => setTimeout(() => reject(new Error("Ошибка")), 100)),
  new Promise(resolve => setTimeout(() => resolve("OK"), 500))
])
  .catch(err => console.log(err.message));  // "Ошибка" — первый завершился с ошибкой
```

> **Частое заблуждение:** "`Promise.race` отменяет проигравшие промисы." Нет — остальные промисы продолжают выполняться, просто их результат игнорируется. В JS нет способа отменить промис (без AbortController).

## Promise.allSettled

### Что это такое?

`Promise.allSettled` ждёт завершения **всех** промисов, независимо от того, успешно они завершились или с ошибкой. Возвращает массив объектов с информацией о каждом промисе.

Аналогия: учитель собирает контрольные работы. Он ждёт **все** работы — и правильные, и неправильные. Ему важно получить результат от каждого ученика.

Появился в ES2020.

### Как работает?

```js
Promise.allSettled([
  Promise.resolve("Успех"),
  Promise.reject(new Error("Ошибка")),
  Promise.resolve(42)
])
  .then(function(результаты) {
    console.log(результаты);
  });

// Результат — массив объектов:
// [
//   { status: "fulfilled", value: "Успех" },
//   { status: "rejected",  reason: Error("Ошибка") },
//   { status: "fulfilled", value: 42 }
// ]
```

**Практический пример — отправка данных на несколько серверов:**

```js
async function отправитьНаВсеСерверы(данные) {
  const серверы = [
    "https://server1.com/api",
    "https://server2.com/api",
    "https://server3.com/api"
  ];

  const результаты = await Promise.allSettled(
    серверы.map(url => fetch(url, {
      method: "POST",
      body: JSON.stringify(данные)
    }))
  );

  // Анализируем результаты
  const успешные = результаты.filter(r => r.status === "fulfilled");
  const неудачные = результаты.filter(r => r.status === "rejected");

  console.log("Успешно: " + успешные.length + " из " + серверы.length);

  if (неудачные.length > 0) {
    console.log("Ошибки:");
    неудачные.forEach(r => console.log("  - " + r.reason.message));
  }
}
```

### Promise.all vs Promise.allSettled

```
Promise.all:
  ✅ ✅ ❌ ✅  →  ❌ (одна ошибка = всё провалилось)

Promise.allSettled:
  ✅ ✅ ❌ ✅  →  ✅ (получаем информацию о КАЖДОМ)
```

### Плохой пример -> Хороший пример

```js
// ПЛОХО — Promise.all теряет успешные результаты при одной ошибке
async function проверитьСайты(urls) {
  try {
    const ответы = await Promise.all(urls.map(url => fetch(url)));
    // Если хоть один сайт не отвечает — мы не узнаем о других
  } catch (err) {
    console.log("Что-то упало, но что именно?");
  }
}
```

```js
// ХОРОШО — allSettled показывает статус каждого
async function проверитьСайты(urls) {
  const результаты = await Promise.allSettled(urls.map(url => fetch(url)));

  результаты.forEach(function(r, i) {
    if (r.status === "fulfilled") {
      console.log(urls[i] + " — работает");
    } else {
      console.log(urls[i] + " — не отвечает: " + r.reason.message);
    }
  });
}
```

> **Мини-проверка:** `Promise.allSettled` **никогда** не переходит в состояние rejected. Почему?
> Ответ: потому что он всегда возвращает fulfilled-промис с массивом результатов. Ошибки отдельных промисов — это просто объекты `{ status: "rejected", reason: ... }` в этом массиве.

## Promise.any

### Что это такое?

`Promise.any` возвращает результат **первого успешного** промиса. Ошибки игнорируются, пока хотя бы один не завершится успешно. Если **все** промисы rejected — возвращает `AggregateError`.

Аналогия: ты звонишь в несколько такси-сервисов одновременно. Тебе нужна **первая машина, которая приедет**. Если один сервис откажет — ничего, ждёшь другие. Только если **все** откажут — проблема.

Появился в ES2021.

### Как работает?

```js
Promise.any([
  Promise.reject(new Error("Ошибка 1")),
  new Promise(resolve => setTimeout(() => resolve("Второй"), 500)),
  new Promise(resolve => setTimeout(() => resolve("Третий"), 100))
])
  .then(function(первыйУспешный) {
    console.log(первыйУспешный);  // "Третий" — быстрее всех успешных
  });
```

```
Промис 1: ❌ (ошибка — игнорируется)
Промис 2: ✅ через 500мс
Промис 3: ✅ через 100мс ← первый успешный!

Promise.any → "Третий"
```

**Когда все промисы rejected:**

```js
Promise.any([
  Promise.reject(new Error("Ошибка 1")),
  Promise.reject(new Error("Ошибка 2")),
  Promise.reject(new Error("Ошибка 3"))
])
  .catch(function(err) {
    console.log(err instanceof AggregateError);  // true
    console.log(err.errors);
    // [Error("Ошибка 1"), Error("Ошибка 2"), Error("Ошибка 3")]
  });
```

**Практический пример — запрос к зеркалам:**

```js
async function загрузитьФайл(имяФайла) {
  const зеркала = [
    "https://mirror1.com/files/",
    "https://mirror2.com/files/",
    "https://mirror3.com/files/"
  ];

  try {
    const данные = await Promise.any(
      зеркала.map(url => fetch(url + имяФайла))
    );
    console.log("Загружено с первого доступного зеркала");
    return данные;
  } catch (err) {
    console.error("Все зеркала недоступны");
    throw err;
  }
}
```

## Promise.race vs Promise.any

```
Promise.race — первый ЗАВЕРШИВШИЙСЯ (успех или ошибка):
  ❌(100мс)  ✅(200мс)  ✅(300мс)  →  ❌ (ошибка — она первая)

Promise.any — первый УСПЕШНЫЙ:
  ❌(100мс)  ✅(200мс)  ✅(300мс)  →  ✅ (ошибка пропущена, берём успех)
```

```js
const промисы = [
  new Promise((_, reject) => setTimeout(() => reject(new Error("быстрая ошибка")), 50)),
  new Promise(resolve => setTimeout(() => resolve("медленный успех"), 200))
];

// race — первый завершившийся (ошибка)
Promise.race(промисы)
  .catch(err => console.log("race:", err.message));
// "race: быстрая ошибка"

// any — первый успешный
Promise.any(промисы)
  .then(val => console.log("any:", val));
// "any: медленный успех"
```

## Сводная таблица

| Метод | Когда fulfilled | Когда rejected | Результат |
|-------|----------------|----------------|-----------|
| **Promise.all** | Все успешны | Хотя бы один упал | Массив значений |
| **Promise.race** | Первый завершился успешно | Первый завершился с ошибкой | Одно значение |
| **Promise.allSettled** | Всегда (ждёт все) | Никогда | Массив `{status, value/reason}` |
| **Promise.any** | Хотя бы один успешен | Все провалились | Одно значение |

### Когда что использовать?

```
Нужны ВСЕ результаты, и все должны быть успешны?
  → Promise.all

Нужны ВСЕ результаты, даже если некоторые с ошибками?
  → Promise.allSettled

Нужен результат ПЕРВОГО завершившегося (неважно, успех или ошибка)?
  → Promise.race

Нужен результат ПЕРВОГО УСПЕШНОГО?
  → Promise.any
```

> **Мини-проверка:** Какой метод использовать для таймаута запроса?
> Ответ: `Promise.race` — гонка между запросом и таймером. Кто первый завершится — тот и определяет результат.

> **Мини-проверка:** Какой метод использовать, если нужно отправить аналитику на 3 сервера, и не страшно если один не ответит?
> Ответ: `Promise.allSettled` — ждём всех, но не падаем при ошибках.

## Итог

| Метод | Аналогия | Типичное применение |
|-------|----------|---------------------|
| **all** | Все блюда на одном подносе | Параллельная загрузка зависимых данных |
| **race** | Гонка: кто первый | Таймаут, выбор быстрейшего сервера |
| **allSettled** | Сбор всех контрольных | Мониторинг, аналитика, необязательные запросы |
| **any** | Первое свободное такси | Резервные серверы, зеркала |
