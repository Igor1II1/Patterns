# Event Loop — как JavaScript выполняет код

> [[js-fundamentals|<-- Назад к оглавлению JS]]

---

## Зачем эта тема существует?

JavaScript создавался для браузера — среды, где нужно одновременно реагировать на клики, загружать данные с сервера и обновлять страницу. Но движок JS **однопоточный** — он может выполнять только одну операцию за раз. Event Loop — это механизм, который позволяет JS казаться многозадачным, не будучи таковым. Без понимания Event Loop невозможно писать асинхронный код осознанно.

---

## Однопоточность JavaScript

### Что это такое?

JavaScript работает в **одном потоке** — представь одного повара на кухне. Он может делать только одно действие в каждый момент: либо резать овощи, либо мешать суп. Но он не может делать оба действия одновременно.

```
Поток JS:  [задача 1] → [задача 2] → [задача 3] → ...
                     ↑
            одна задача за раз
```

### Как работает?

Когда ты запускаешь JS-код, движок обрабатывает его **строка за строкой**, сверху вниз:

```js
console.log("Первое");   // 1 — выполняется сразу
console.log("Второе");   // 2 — выполняется после первого
console.log("Третье");   // 3 — выполняется после второго
```

**Проблема:** если одна операция занимает 5 секунд (например, запрос к серверу), весь код **замирает** — страница перестает реагировать. Это называется **блокировка потока**.

```js
// ПЛОХО — блокирующий код (представь что это занимает 5 секунд)
const data = загрузитьДанныеСинхронно("https://api.example.com");
console.log(data); // страница зависла на 5 секунд
```

Именно для решения этой проблемы существует асинхронность.

## Call Stack (стек вызовов)

### Что это такое?

Call Stack — это **стопка тарелок**. Когда вызывается функция, она кладется на верх стопки. Когда функция завершается, она снимается сверху. JS всегда выполняет ту функцию, которая лежит **наверху**.

### Как работает?

```js
function умножить(a, b) {
  return a * b;
}

function квадрат(n) {
  return умножить(n, n);
}

function показатьКвадрат(n) {
  const результат = квадрат(n);
  console.log(результат);
}

показатьКвадрат(5);
```

Стек вызовов пошагово:

```
Шаг 1:                    Шаг 2:                    Шаг 3:
┌─────────────────┐       ┌─────────────────┐       ┌─────────────────┐
│                 │       │                 │       │ умножить(5, 5)  │ <-- верх
│                 │       │ квадрат(5)      │ <--   ├─────────────────┤
│ показатьКвадрат │ <--   ├─────────────────┤       │ квадрат(5)      │
├─────────────────┤       │ показатьКвадрат │       ├─────────────────┤
│    (глобальный) │       ├─────────────────┤       │ показатьКвадрат │
└─────────────────┘       │    (глобальный) │       ├─────────────────┤
                          └─────────────────┘       │    (глобальный) │
                                                    └─────────────────┘

Шаг 4:                    Шаг 5:                    Шаг 6:
┌─────────────────┐       ┌─────────────────┐       ┌─────────────────┐
│ console.log(25) │ <--   │                 │       │                 │
├─────────────────┤       │                 │       │                 │
│ показатьКвадрат │       │ показатьКвадрат │ <--   │                 │
├─────────────────┤       ├─────────────────┤       │                 │
│    (глобальный) │       │    (глобальный) │       │    (глобальный) │ <-- пусто
└─────────────────┘       └─────────────────┘       └─────────────────┘
```

Когда стек пуст — JS ждет новых задач.

> **Частое заблуждение:** "JS может выполнять несколько функций параллельно." Нет — в стеке всегда выполняется только верхняя функция. Остальные ждут.

## Web APIs (среда выполнения)

### Что это такое?

Браузер — это не только JS-движок. Это целая среда, которая предоставляет **дополнительные возможности**: таймеры, работу с сетью, обработку событий DOM. Эти возможности называются **Web APIs**.

Аналогия: повар (JS-движок) может отправить блюдо в духовку (Web API) и продолжить готовить другое. Духовка работает **параллельно** — она не повар, у нее свои ресурсы.

### Как работает?

```js
console.log("Начало");

setTimeout(function() {
  console.log("Таймер сработал");
}, 2000);

console.log("Конец");
```

Что происходит:

```
1. console.log("Начало")     → стек → выполняется → "Начало"
2. setTimeout(fn, 2000)      → стек → передается в Web API (таймер)
3. console.log("Конец")      → стек → выполняется → "Конец"
4. ...проходит 2 секунды...
5. Web API отправляет fn     → в очередь задач
6. Event Loop берет fn       → в стек → выполняется → "Таймер сработал"
```

Результат в консоли:
```
Начало
Конец
Таймер сработал    ← через 2 секунды
```

> **Частое заблуждение:** "`setTimeout(fn, 0)` выполнит функцию мгновенно." Нет — даже с задержкой 0 функция пройдет через Web API и очередь, то есть выполнится **после** всего синхронного кода.

## Очередь задач (Callback Queue / Task Queue)

### Что это такое?

Когда Web API завершает работу (таймер истек, данные загружены), колбэк-функция попадает не в стек, а в **очередь**. Очередь — это список задач, ожидающих выполнения. Работает по принципу FIFO: **первый пришел — первый вышел** (как очередь в магазине).

### Как работает?

```
┌──────────────────────────────────────────────────────────────────┐
│                        БРАУЗЕР                                   │
│                                                                  │
│  ┌──────────┐     ┌──────────────┐     ┌───────────────────┐    │
│  │ Call Stack│     │   Web APIs   │     │  Callback Queue   │    │
│  │          │     │              │     │                   │    │
│  │  func()  │────>│  setTimeout  │────>│  cb1  cb2  cb3   │    │
│  │          │     │  fetch       │     │  →→→→→→→→→→→→→   │    │
│  │          │     │  DOM events  │     │                   │    │
│  └──────────┘     └──────────────┘     └─────────┬─────────┘    │
│       ^                                          │              │
│       │          ┌──────────────┐                │              │
│       └──────────│  EVENT LOOP  │<───────────────┘              │
│                  │  (проверяет) │                                │
│                  └──────────────┘                                │
└──────────────────────────────────────────────────────────────────┘
```

**Алгоритм Event Loop:**
1. Стек пуст?
2. Да → взять первую задачу из очереди → положить в стек
3. Нет → ждать, пока стек опустеет
4. Повторить

## Очередь микрозадач (Microtask Queue)

### Что это такое?

Помимо обычной очереди, есть **привилегированная очередь** — очередь микрозадач. Она имеет **приоритет** над обычной очередью. Сюда попадают:
- Колбэки промисов (`.then`, `.catch`, `.finally`)
- `queueMicrotask()`
- `MutationObserver`

Аналогия: представь две очереди в больнице — обычная и срочная. Врач **сначала** принимает всех срочных пациентов, и только потом обычных.

### Как работает?

```js
console.log("1");

setTimeout(() => console.log("2"), 0);

Promise.resolve().then(() => console.log("3"));

console.log("4");
```

Пошагово:

```
Синхронный код:
  console.log("1")  → выводит "1"
  setTimeout(...)    → колбэк → Callback Queue (макрозадача)
  Promise.then(...)  → колбэк → Microtask Queue (микрозадача)
  console.log("4")  → выводит "4"

Стек пуст. Event Loop проверяет:
  1. Есть микрозадачи? ДА → console.log("3") → выводит "3"
  2. Есть микрозадачи? НЕТ
  3. Есть макрозадачи? ДА → console.log("2") → выводит "2"
```

Результат: `1`, `4`, `3`, `2`

### Плохой пример -> Хороший пример

```js
// ПЛОХО — не понимаем порядок выполнения
setTimeout(() => console.log("A"), 0);
Promise.resolve().then(() => console.log("B"));
console.log("C");
// Новичок думает: C, A, B
// На самом деле:  C, B, A
```

```js
// ХОРОШО — понимаем приоритеты
console.log("C");                                // 1. Синхронно
Promise.resolve().then(() => console.log("B"));  // 2. Микрозадача (приоритет)
setTimeout(() => console.log("A"), 0);           // 3. Макрозадача (после микро)
// Результат: C, B, A — осознанно
```

> **Мини-проверка:** Что выведет этот код?
> ```js
> setTimeout(() => console.log("timeout"), 0);
> Promise.resolve().then(() => console.log("promise 1"))
>                   .then(() => console.log("promise 2"));
> console.log("sync");
> ```
> Ответ: `sync`, `promise 1`, `promise 2`, `timeout`

## Полная картина Event Loop

```
┌─────────────────────────────────────────────────────────────┐
│                      EVENT LOOP                              │
│                                                              │
│   ┌─────────────┐                                           │
│   │  Call Stack  │ <── выполняет код                        │
│   └──────┬──────┘                                           │
│          │                                                   │
│          │ стек пуст?                                        │
│          ▼                                                   │
│   ┌──────────────────┐    ДА    ┌─────────────────────┐     │
│   │ Микрозадачи есть?│─────────>│ Выполнить ВСЕ       │     │
│   │ (Promise, qMT)   │         │ микрозадачи         │     │
│   └────────┬─────────┘         └─────────────────────┘     │
│            │ НЕТ                         │                   │
│            ▼                             ▼                   │
│   ┌──────────────────┐    ДА    ┌─────────────────────┐     │
│   │ Макрозадачи есть?│─────────>│ Выполнить ОДНУ      │     │
│   │ (setTimeout, I/O)│         │ макрозадачу         │     │
│   └────────┬─────────┘         └─────────────────────┘     │
│            │ НЕТ                         │                   │
│            ▼                             │                   │
│        [ждать]            ←──────────────┘                   │
│                      (вернуться к проверке микрозадач)       │
└─────────────────────────────────────────────────────────────┘
```

Ключевое правило: **после каждой макрозадачи — выполнить ВСЕ микрозадачи.**

## Зачем вообще нужна асинхронность?

Без асинхронности:
```js
// Пользователь нажал кнопку "Загрузить"
const данные = загрузитьСинхронно("/api/users"); // 3 секунды...
// Всё это время: страница заморожена, кнопки не работают,
// анимации остановлены, пользователь думает что браузер завис
показать(данные);
```

С асинхронностью:
```js
// Пользователь нажал кнопку "Загрузить"
загрузитьАсинхронно("/api/users", function(данные) {
  показать(данные);
});
// Страница продолжает работать! Кнопки кликаются, анимации крутятся.
// Когда данные придут — колбэк выполнится автоматически.
```

## Итог

| Понятие | Описание | Аналогия |
|---------|----------|----------|
| **Call Stack** | Стек выполняемых функций | Стопка тарелок |
| **Web APIs** | Внешние возможности браузера (таймеры, сеть) | Духовка — работает параллельно |
| **Callback Queue** | Очередь макрозадач (setTimeout, события) | Обычная очередь в больнице |
| **Microtask Queue** | Очередь микрозадач (Promise) — приоритетная | Срочная очередь в больнице |
| **Event Loop** | Механизм, переносящий задачи из очередей в стек | Диспетчер / координатор |

**Правило приоритетов:**
```
Синхронный код  >  Микрозадачи (Promise)  >  Макрозадачи (setTimeout)
```
